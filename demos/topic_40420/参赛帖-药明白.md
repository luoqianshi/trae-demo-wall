# 【社会公益】药明白 — 2.97 亿老人的用药安全工具

**【正文】**

```
我是刘相，参赛组别：社会组。

说一个真实的数据：中国 65 岁以上老人超过 2 亿，其中超过 1/3 同时服用 5 种以上药物。漏服、错服、重服是常态，而子女在外打工，想关心却无从下手。

药明白，就是为解决这个问题而生的。

一句话介绍：拍照识别药盒 → 家乡话语音播报药品说明 → 用药提醒与库存管理 → 家人实时同步。

最让我自己感动的是方言播报功能：老人看不清药盒上的小字，点一下拍照，手机就用他们最熟悉的口音把药品名称、用法用量、注意事项全部读出来。目前支持 10 种家乡话：普通话、四川话、粤语、河南话、上海话、东北话、陕西话、湖南话、闽南话、山东话，自动检测用户地区推荐最适合的方言。
```

**1. Demo 简介**

- **是什么：** 药明白是一款面向中国老年人的用药安全 H5 工具，单文件无需安装，浏览器打开即用，支持 PWA 离线缓存。
- **面向谁：** 子女不在身边的独居老人；不懂药盒小字的年长患者；想远程关心父母用药情况的子女。
- **主要功能：**
  - **📷 拍药识药**：摄像头对准药盒 → AI 识别药品 → 显示名称、用法、禁忌，并自动用方言语音播报
  - **🔔 用药提醒**：自定义多时段提醒方案，每日自动重置，打卡记录用药情况
  - **📦 药库管理**：药品库存可视化进度条，库存不足时自动预警并通知家人
  - **👨‍👩‍👧 家人关怀**：家人联系方式管理，漏服/库存不足时触发 SOS 通知
  - **🌍 10 种方言播报**：普通话/四川话/粤语/河南话/上海话/东北话/陕西话/湖南话/闽南话/山东话，基于时区自动推荐
  - **👓 长辈模式**：全局字号放大，高对比度色彩，宽点击区域

**2. Demo 创作思路**

- **灵感来源：** 自己的爷爷奶奶每次吃药都要喊子女来念说明书，手机字太小根本看不清。有一次爷爷因为看不清剂量吃错了药，全家人吓了一大跳。
- **想解决的问题：**
  - 老人看不清药盒小字，没人帮忙就不敢吃
  - 子女在外打工，想关心父母的用药情况却没有任何渠道
  - 药吃完了不知道，漏服了子女也不知道
  - 子女在外地，父母手机出了问题远程讲不清楚
- **为什么做这个方向：** 中国正在快速老龄化，2.97 亿老年人的用药安全是一个被严重忽视的问题。市面上的 app 要么太复杂老人不会用，要么界面太小字根本看不清。药明白的核心理念是"用老人最熟悉的方式和他说话"——不是让老人去适应 app，而是让 app 主动适应老人。

**5. TRAE 实践过程**

```
阶段一：需求拆解与核心架构
- 与 TRAE 对话明确产品定位：拍药识药 + 方言播报 + 用药管家 + 家人关怀
- 确定 4 Tab 架构：首页 / 提醒 / 药库 / 家人
- 决定方言播报方案：Web Speech API + 10 种方言参数调优

阶段二：方言播报系统
- 构建 10 种方言配置表（pitch / rate 参数调优）
- 实现方言自动识别推荐（浏览器语言 + 时区检测）
- 首次使用弹窗推荐，localStorage 持久化选择

阶段三：拍药识药体验升级
- 接入真实摄像头（getUserMedia 后置摄像头实时预览）
- 取景框扫描线动画 + 四角对准引导
- 截图留存 + 识别中动画
- 降级方案：相册选择 / 手动搜索药库兜底

阶段四：药库与库存联动
- 打卡按剂量扣库存，计算剩余天数
- 库存可视化进度条（30 天满格，绿/红/灰三色）
```

**6. 技术亮点**

- 纯前端 H5 单文件，无后端依赖，可直接浏览器打开
- PWA Service Worker 离线缓存，无网也能用核心功能
- localStorage 全数据持久化（药库/方言/提醒/联系人）
- Web Speech API 方言语音合成，0 成本无需 TTS API
- getUserMedia 真实摄像头预览，参赛展示有真东西
