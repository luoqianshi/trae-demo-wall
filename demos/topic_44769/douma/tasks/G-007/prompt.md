# 一致性分桶器

## 背景

`Sharder` 是一个把整型键映射到固定数量分桶的分片器：业务用它把请求/数据按 key 均匀地分散到 `n` 个桶（如分库分表、负载分片）。
key 可能来自哈希、ID 差值、时间偏移等，**既可能为正，也可能为负或零**。无论 key 取何值，桶索引都必须落在合法范围内。

## 对外接口

- `NewSharder(n int) *Sharder`：创建拥有 `n` 个桶（`n >= 1`）的分片器。
- `Bucket(key int) int`：返回 `key` 所属的桶索引，**保证落在 `[0, n)` 内**。
- `Count(n int, keys []int) []int`：统计一批 `keys` 落入各桶的数量，返回长度为 `n` 的计数切片（下标即桶号）。

## 对外保证（不变量）

1. **索引合法**：对任意整型 `key`（正、负、零），`Bucket(key)` 的返回值都必须满足 `0 <= 结果 < n`。
2. **同余一致**：两个 key 若关于 `n` 同余（`a ≡ b (mod n)`），必须落入同一个桶；
   例如 `n=8` 时 `key=-1` 与 `key=7` 必须落在同一桶。
3. **统计自洽**：`Count` 的结果各桶之和等于输入 keys 总数，且每个桶计数非负、下标不越界。

## 错误现象

- 当所有 key 都是非负数时，分桶完全正常、分布均匀。
- 但一旦出现**负数 key**，桶索引会变成负数，导致按桶号建立的数组发生越界访问，或负 key 与其同余正 key 被分到不同桶。

## 你的任务

定位并修复缺陷，使上述三条不变量对任意整型 key（含负数与零）都严格成立。
