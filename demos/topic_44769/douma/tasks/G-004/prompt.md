# 配置快照

## 背景

`ConfigStore` 是一个进程内的键值配置存储（用有序的键值对表示）。
运维在变更配置前，常先调用 `snapshot()` 拍一份**当前配置的快照**留底，
随后继续通过 `set()` 修改 store。快照用于事后对比 / 回滚参考，
因此它必须是**拍摄那一刻配置的独立定格**。

## 对外接口

- `ConfigStore::new()`：创建空配置存储。
- `set(&mut self, key, val)`：写入/更新一个键的整型值（已存在则覆盖，不存在则追加）。
- `get(&self, key) -> Option<i64>`：读取某键的当前值。
- `len(&self) -> usize`：当前键的数量。
- `snapshot(&self) -> ConfigSnapshot`：拍一份当前配置快照。
- `ConfigSnapshot::get(&self, key) -> Option<i64>` / `len(&self) -> usize`：读取快照内容。

## 对外保证（不变量）

1. **快照定格**：`snapshot()` 拍下后，**无论之后对 `ConfigStore` 做多少次 `set()`**，
   该快照的内容都必须保持拍摄时的原样，不受后续修改影响。
2. **多份独立**：先后拍下的多份快照彼此独立，各自定格各自时刻。
3. **读值一致**：快照与拍摄时刻的 store 在每个键上的取值完全一致。

## 错误现象

- 只拍快照、不再改 store 时，快照内容正确。
- 但只要拍完快照后又对 store 调用了 `set()`，先前那份快照的内容竟会**跟着一起变**，无法再代表拍摄时刻。

## 你的任务

定位并修复缺陷，使上述三条不变量在任意操作顺序下都严格成立。
