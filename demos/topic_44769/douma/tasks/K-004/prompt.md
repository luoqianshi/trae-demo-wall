# 编辑距离（Levenshtein Distance）

## 背景

`edit_distance` 计算把字符串 `a` 变成字符串 `b` 所需的**最少单字符操作数**。允许的操作有三种，每次代价为 1：

- **插入**一个字符；
- **删除**一个字符；
- **替换**一个字符（把某个字符改成另一个）。

这是拼写纠错、DNA 比对、diff 等场景的基础度量。结果必须是严格的最小操作数。

## 对外接口

- `pub fn edit_distance(a: &str, b: &str) -> usize`：
  返回将 `a` 编辑为 `b` 的最小操作数（标准 Levenshtein 距离，三种操作代价均为 1）。

## 对外保证（不变量）

1. **三操作齐备**：插入、删除、**替换**都要被纳入最优决策；
   当对应位置字符不同且两侧都还有字符时，**一次替换**（代价 1）必须被考虑，而不能被迫拆成「删除 + 插入」（代价 2）。
2. **平凡边界**：`edit_distance(a, "")` 等于 `a` 的长度，`edit_distance("", b)` 等于 `b` 的长度，
   `edit_distance(a, a) == 0`。
3. **对称性**：`edit_distance(a, b) == edit_distance(b, a)`（代价对称）。

## 错误现象

- 当两个字符串的差异只是「多了/少了一些字符」（纯插入或纯删除即可解决）时，结果完全正确。
- 但当存在**需要替换**的位置（同一位置字符不同）时，结果偏大——
  本可一步替换的地方被算成了删除再插入两步，导致距离被高估。
