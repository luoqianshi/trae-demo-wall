# 配置解析管线

## 背景

`parse_config` 是一段配置解析管线：输入若干行 `key=value` 文本，
把每行解析为 `(键, 数值)`，其中 `value` 必须是 **`0..=1000` 范围内的非负整数**。
解析结果用于初始化系统参数，**任何一处非法都必须让整体解析失败**——
不能把非法值悄悄替换成默认值后当作成功，否则系统会带着错误参数静默启动。

配套的 `find_value` 用于在解析结果中按键查值：**键不存在时必须返回 `None`**，
而不是返回某个默认数值掩盖“查不到”这一事实。

## 对外接口

- `pub fn parse_config(lines: &[&str]) -> Result<Vec<(String, u32)>, String>`：
  - 每行须形如 `key=value`，`value` 为 `0..=1000` 的非负整数；
  - 全部合法 → 返回 `Ok(解析出的键值对，顺序与输入一致)`；
  - 任意一行格式非法（缺 `=`、值非数字、负数、超出范围、键为空等）→ 返回 `Err(错误描述)`。
- `pub fn find_value(config: &[(String, u32)], key: &str) -> Option<u32>`：
  - 命中键 → `Some(值)`；未命中 → `None`。

## 对外保证（不变量）

1. **错误必传播**：只要存在非法行，`parse_config` 必须返回 `Err`，
   **绝不允许**把非法值替换成 `0`（或任何默认值）后返回 `Ok`。
2. **成功即全合法**：返回 `Ok(v)` 时，`v` 中每个数值都确实是输入中对应的、落在 `0..=1000` 的合法值。
3. **查不到为 None**：`find_value` 对不存在的键必须返回 `None`，不得返回 `Some(默认值)`。

## 错误现象

- 配置全部合法时，解析结果完全正确。
- 但当某行的值非法（如 `port=abc`、`port=-1`）时，解析**没有报错**——
  非法值被悄悄当成了 `0`，函数照常返回 `Ok`，导致下游拿到一份“看似成功实则被篡改”的配置。
