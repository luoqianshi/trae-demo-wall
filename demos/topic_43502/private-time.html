关于本地存储的风险与建议

本地存储的风险（Demo 版本说明）

本工具使用浏览器 LocalStorage 仅为演示（Demo）用途。在实际生产环境中，应接入后端数据库或改造为桌面应用，以获得持久化、大容量的存储能力。
  • 浏览器清理缓存或隐私数据时，所有记忆会被永久删除
  • 切换浏览器（如从Chrome换到Safari）后，数据无法迁移
  • 卸载浏览器、重装系统、使用无痕模式，数据均会丢失
  • LocalStorage 有 5MB 左右容量限制，大量图片可能存满
  • 本工具不上传任何数据到服务器，也意味着没有云端备份

如何保护你的数据

  • 定期导出备份：进入「设置」>「导出所有数据」，将 JSON 文件保存到电脑或网盘
  • 重要时刻立即备份：写完一段珍贵记忆后，顺手导出一次
  • 多设备备份：可将 JSON 文件复制到 U 盘、移动硬盘或云盘（加密后上传更安全）
  • 避免无痕模式：无痕/隐私模式下，浏览器关闭后数据即消失
  • 谨慎清理浏览器：清理缓存前，务必先导出数据备份

数据恢复方法

进入「设置」>「导入数据」，选择之前导出的 JSON 备份文件，即可恢复所有记忆和时光胶囊。

隐私承诺

本工具为纯前端静态页面（Demo 演示版本），没有任何后端服务器，你的文字、图片、录音均只保存在你自己的浏览器中。我们不会收集、上传或分析任何数据。生产环境请自行部署后端服务。

存储容量提示

当前版本使用浏览器 LocalStorage 存储，通常限制在 5MB 左右，适合文字记录和少量图片。这只是一个演示版本（Demo）的容量限制。如果你需要存储大量图片、视频或音频，可以考虑将本工具改造为 Electron 桌面版或接入后端服务，即可突破容量限制，直接保存到本地文件夹。

突破容量限制的扩展方案

  • Electron 桌面版：将本工具打包为桌面应用，获得完整的文件系统读写权限，图片视频直接存到用户指定的本地文件夹，无容量限制
  • Node.js 本地服务：启动一个轻量本地服务器，前端通过 API 将数据写入电脑任意位置
  • IndexedDB：浏览器内置数据库，容量通常 250MB 以上，适合不想安装软件但需要更多空间的用户
  • OPFS（私有文件系统）：Chrome/Edge 支持的实验性功能，可像操作文件系统一样读写，容量更大

以上方案均可基于当前代码改造，核心功能和 UI 无需重写。

欢迎使用私藏时光

你的记忆将保存在当前浏览器的本地存储中，不会上传到任何服务器。请定期到「设置」中导出备份，以防数据丢失。

本工具为 Demo 演示版本，使用浏览器 LocalStorage 仅用于功能展示。浏览器清理缓存、切换浏览器、卸载浏览器或重装系统时，本地数据均会丢失。生产环境请接入后端数据库或改造为桌面应用。

建议每次记录重要内容后顺手导出备份。如需存储大量图片/视频，详情见「了解存储风险与备份方法」。
