📉 损失曲线

🧠 什么是神经元？
每个神经元接收多个输入，乘以各自的权重（Weight），加上偏置（Bias），再通过激活函数输出。数学上就是：output = activation(Σ wᵢ·xᵢ + b)。多层神经元叠加，就构成了神经网络。
🏋️ 什么是训练（Training）？
给模型看"输入 → 正确答案"的样本，计算预测值与真实值的误差（Loss），然后通过梯度下降反向调整所有权重，让误差越来越小。训练的本质就是：不断调整权重，让模型"猜"得越来越准。
🔮 什么是推理（Inference）？
训练完成后，用训练好的权重对新输入做前向传播（Forward Pass）得到预测结果。推理不改变权重，速度很快。大模型说到底也是"超大规模的前向传播"——只是参数从几十个变成了几千亿个。
↔️ 前向传播 vs 反向传播
前向传播：输入 → 隐藏层 → 输出，得到预测值。
反向传播：从输出层往回计算梯度，用链式法则更新每一层的权重。梯度下降 = 前向（算误差）+ 反向（调权重），循环往复。
模块 1

LLM 如何"说话"：下一词预测机制

大语言模型每次只预测一个词（token），然后把它加回输入，循环往复——就像接句子游戏。 试试输入开头，点击候选词手动续写，或点击"自动续写"看完整生成。调整"温度"观察输出变得更保守还是更有创意。

🌡 温度（Temperature）： 0.70
人工智能 正在 改变
🧩 Token（词元）
中文里大约 1-2 个字为一个 token，英文里大约 4 个字母为一个 token。GPT-4 的上下文窗口是 128k token，约等于一本中短篇小说。
🎯 概率分布 / Softmax
模型为词表中的每个词打分，再通过 Softmax 函数把分数转换为概率（总和为 1）。你看到的候选词就是概率最高的若干个。
🌡 温度（Temperature）
Temperature 越高，概率分布越"平"——模型更愿意尝试冷门词，结果更有创意但也更不可控。为 0 时模型永远只选最高概率词，结果确定但机械。
📖 试试调节温度，体会差别
  1. 把温度调到 0.1（左），多次点击"自动续写"——结果基本一致，模型"谨慎输出"。
  2. 把温度调到 1.5（右），再次点击——模型会尝试更多冷门词，每次结果不同，更容易"跑题"或出现"幻觉"。
  3. 把温度调到 0.7——这是大多数聊天模型的默认值，兼顾创意与稳定。

🔥 实战教学：好提示词 vs 差提示词
❌ 差提示词（模糊/太短）
"解释一下知识图谱。"
问题：没有定义受众、长度、语气、格式、是否需要例子。
模型可能输出一长段晦涩专业文，或太短。
✅ 好提示词（结构化）
"你是一名资深技术讲师。请向【10岁小孩】解释【知识图谱】：
1) 用一个生活中的比喻开头
2) 3 句话以内讲完
3) 结尾列出 2 个常见应用"
好处：角色清晰、受众明确、有格式控制、答案可预期。
📐 写出好提示词的 5 个维度（CRISPE 框架）
C · Capacity 角色
你希望模型扮演谁？比如：资深医生、小学老师、架构师。
R · Response 格式
想要什么输出？Markdown / JSON / 表格 / 代码 / 短问答？
I · Insight 背景
给模型上下文：这是给谁看的？有什么已有信息或约束？
S · Statement 任务
要模型做什么？一句话说清核心任务。
P · Personality 个性
希望输出什么语气？正式 / 幽默 / 简洁 / 激情？
E · Experiment 实验
可以让模型给出 3 个不同答案，并让你选择——降低犯错率。

🎯 练习：图谱能做什么推理？

点击下方问题，观察图谱上高亮的路径如何回答问题。

🟢 节点 / 实体 (Node)
图谱中的"事物"——人、地点、产品、技术名词。每个节点存储：ID、类型、属性。
🔗 关系 / 边 (Edge)
节点之间的联系，如"导演"、"产于"、"属于"。关系可以带有方向和权重。
🔍 路径 / 推理 (Path)
从节点 A 到节点 B 的序列。例如"知识图谱 → 被用于 → RAG"，就是一条推理路径。
🏛 图数据库
Neo4j、NebulaGraph、ArangoDB 等是专门存储和查询知识图谱的数据库，使用 Cypher / SPARQL 语言查询。
📚 想深入了解知识图谱的优化方案 & 主流做法？
包括：构建流水线（抽取-对齐-融合）、图嵌入（TransE/TransH/RESCAL）、 查询优化（索引+分库分图）、大规模图谱工程（NebulaGraph/Neo4j 集群）、 以及 2024~2026 年主流方向（LLM + KG 融合、动态图谱、企业知识中台）。
