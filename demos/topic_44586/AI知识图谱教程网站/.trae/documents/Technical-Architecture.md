# 手势追踪变花 - 技术架构文档

## 1. 技术架构

```
┌─────────────────────────────────────────┐
│              前端层 (Frontend)           │
│  ┌─────────────┐  ┌──────────────────┐  │
│  │   HTML/CSS  │  │  Three.js 3D渲染  │  │
│  └─────────────┘  └──────────────────┘  │
│  ┌─────────────────────────────────────┐│
│  │      MediaPipe Hands 手部追踪       ││
│  └─────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

## 2. 技术选型

- **前端框架**: 纯 HTML5 + CSS3 + JavaScript（无框架依赖）
- **3D 渲染**: Three.js
- **手部追踪**: MediaPipe Hands (通过 CDN 引入)
- **构建工具**: 无需构建，直接使用 ES6 模块

## 3. 依赖库

## 5. 核心模块设计

### 5.1 摄像头模块 (CameraModule)
- 初始化摄像头流
- 处理权限请求和错误
- 提供视频帧给手部追踪模块

### 5.2 手部追踪模块 (HandTracker)
- 初始化 MediaPipe Hands
- 处理每一帧视频
- 检测手部位置和关键点
- 发出手部位置事件

### 5.3 花朵渲染模块 (FlowerRenderer)
- 创建 3D 花朵模型
- 管理花朵实例
- 处理花朵动画（摇曳、绽放）
- 与手部位置同步

### 5.4 主控制器 (MainController)
- 协调各模块工作
- 处理模块间通信
- 管理应用状态

## 6. 数据模型

### 6.1 手部数据
```javascript
{
  palmX: number,      // 手掌 X 坐标 (归一化 0-1)
  palmY: number,      // 手掌 Y 坐标 (归一化 0-1)
  palmZ: number,      // 手掌 Z 坐标
  landmarks: [],      // 21个关键点坐标
  handedness: 'Left' | 'Right'  // 左手/右手
}
```

### 6.2 花朵数据
```javascript
{
  id: string,
  position: { x, y, z },
  scale: number,           // 生长动画 0-1
  swayPhase: number,       // 摇曳相位
  swaySpeed: number,        // 摇曳速度
  color: { r, g, b },      // 花朵颜色
  type: 'rose' | 'tulip' | 'sakura'  // 花朵类型
}
```

## 7. 动画系统

### 7.1 花朵生长动画
- 缩放从 0 到 1
- 持续时间: 800ms
- 缓动函数: easeOutElastic

### 7.2 花朵摇曳动画
- 基于正弦波的循环动画
- 摇曳幅度: ±15°
- 摇曳周期: 2-4秒（随机）

### 7.3 粒子效果
- 花朵绽放时喷射粒子
- 粒子数量: 20-30
- 粒子寿命: 1-2秒
- 粒子扩散半径: 50-100px

## 8. 性能优化

- 手部追踪使用 requestAnimationFrame 节流
- 花朵模型使用实例化渲染
- 粒子系统使用 Points 而非独立对象
- 移动端降低渲染分辨率
