AI 核心概念科普

理解 Claw 类 AI 产品的底层架构，从 Agent 到 Memory 到 MCP

AI Agent 的组成公式

大模型(思考) + Memory(记忆) + RAG(知识) + MCP(手脚) + Skills(经验) = AI Agent

● AI Agent（智能体）

核心概念 · 2026年最火方向

AI Agent 不是简单的聊天机器人，而是拥有自主决策、规划及执行能力的数字实体。 它能像人类员工一样应对复杂任务：理解需求 → 制定计划 → 调用工具 → 执行操作 → 反思结果。

五大核心组件

感知：接收用户指令和环境信息
规划：将大任务拆解为可执行的子步骤
行动：调用工具（写代码、发邮件、查数据）完成任务
记忆：记住用户偏好和历史决策
反思：评估执行结果，自我纠错优化

"2026年被称为 Agent 元年，因为技术栈终于成熟了：长上下文窗口让 Agent 能理解完整项目，工具调用协议让 Agent 能操作外部系统，记忆机制让 Agent 能记住你的偏好。"

● Memory（记忆系统）

上下文管理 · 解决"聊完就忘"

大模型本身没有记忆，每次对话都是"全新的开始"。Memory 系统就是解决这个问题的技术方案。 它让 Agent 能记住你是谁、你喜欢什么、你们之前聊过什么。

三层记忆模型（以 OpenClaw 为例）

工作记忆（Context）：当前会话中模型能"看到"的所有内容，受 token 上限限制（通常 32K-200K）

短期记忆（Compaction）：当工作记忆接近上限时，自动将旧对话总结成摘要压缩保存

长期记忆（Memory Files）：持久化的 Markdown 文件，存储决策、偏好、教训等跨会话信息

核心文件结构

MEMORY.md：策略性长期记忆，存储重要决策和用户偏好
memory/YYYY-MM-DD.md：按日期组织的操作日志
AGENTS.md：Agent 行为规范和工作流
SOUL.md：Agent 的人格与语气设定
TOOLS.md：工具使用的本地笔记

混合检索优化

向量检索（70%）：语义匹配，擅长"意思相近但措辞不同"的查询
BM25 关键词（30%）：精确 token 匹配，适合查 commit hash、代码符号
时间衰减：让近期记忆分数更高，旧记忆逐渐淡化（如 30 天后分数减半）

● MCP（模型上下文协议）

工具调用 · AI 的"USB接口"

MCP（Model Context Protocol）是一套标准化的工具调用协议。 它规定了 AI 如何与外部世界交互：查询数据库、操作文件、发送邮件、调用 API…… 本质是为 AI 安装"USB 接口"，让它能连接各种外部服务。

工作流程

1. 发现：AI 询问"有哪些工具可用？"
2. 描述：MCP Server 返回工具列表和参数定义
3. 决策：AI 判断需要调用哪个工具
4. 调用：AI 生成 JSON 格式的调用指令
5. 执行：MCP 执行操作并返回结果
6. 总结：AI 基于结果给出最终回答

产品MCP 支持说明
Cursor原生支持内置 MCP Host，可连接任意 MCP Server
ClaudeConnectorsAnthropic 自研连接器生态
Trae部分支持国内版逐步接入中
WorkBuddy兼容 OpenClaw可直接使用 OpenClaw 技能生态

● RAG（检索增强生成）

知识外挂 · 解决"知识过时"

大模型只知晓训练截止日之前的公网知识。RAG 就是为其配备一座"图书馆"： 用户提问后，系统先去数据库查找相关资料，再输送给大模型回答。 解决"知识过时"和"私有数据"两大痛点。

工作流程

1. 文档入库：将私有文档切分成片段，生成向量存入数据库
2. 查询向量化：将用户问题也转成向量
3. 相似度检索：在向量库中找到最相关的文档片段
4. 拼接生成：将检索结果 + 用户问题一起送给大模型生成答案

● Skills（技能 / SOP）

经验固化 · 从"有工具"到"会干活"

给一名大学生一把扳手，他未必能修车。Skills 就是操作手册： 明确规定在特定场景下，按照什么顺序、怎样组合使用 MCP 工具。 本质是将"如何操作"固化为一套流程。

Skill 示例：自动部署流程

1. git status 查看改动列表
2. git diff 查看关键改动内容
3. 总结改动，等待用户确认
4. 确认后执行 build
5. 部署后验证网站可访问性

WorkBuddy 内置 20+ Skills，覆盖海报生成、报表制作、行业调研等场景

● OpenClaw

开源框架 · 本地 AI Agent

OpenClaw 是一个面向本地电脑操作的 AI Agent 框架。 你在电脑上能做的操作（写代码、发邮件、交易等），它都可以接管完成。 由于运行于本地，它拥有极大权限，效率极高。

核心特点

本地运行：直接操作你的电脑，而非云端虚拟机
文件即记忆：所有持久化信息写入 Markdown 文件，模型只"记住"磁盘上的内容
多 Agent 协作：支持主 Agent、博客 Agent、工具 Agent 分工协作
Heartbeat 自动化：定期心跳检查，主动执行周期性任务
Skills 生态：兼容 OpenClaw 的技能可被 WorkBuddy 等工具直接使用

"切勿神化 OpenClaw，它并未在底层技术上实现突破，而是工程化成果的集大成者。它成功地将大模型从'陪聊伙伴'转变为真正的'数字员工'。"

一句话总结

大模型是大脑，负责思考；Memory是记忆，让它不忘事；
RAG是图书馆，提供最新知识；MCP是手脚，能操作外部世界；
Skills是经验手册，知道怎么干活；OpenClaw是把这些串起来的框架。

当这些组件组合在一起，AI 就从"聊天机器人"变成了能独立干活的"数字员工"。
