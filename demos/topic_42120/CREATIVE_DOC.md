# 墨字破妖 · InkBattle — 创意文档

> **执笔破妖，识字通关** — Wield the brush, break the curse, master the characters.

---

## 一、项目概述

**墨字破妖 (InkBattle)** 是一款以**汉字拆解描摹**为核心玩法的国风水墨 HTML5 游戏。玩家执毛笔为武器，通过手写描摹汉字部首来破除妖魔弱点，收集部首召唤并击败 BOSS，最终「识字通关」。

游戏面向 GameDistribution HTML5 游戏平台，采用纯前端 Canvas 渲染，零外部依赖，内置程序化国风音效，支持中英文双语。

---

## 二、创意理念

### 2.1 核心隐喻：写字即战斗

在中国传统文化中，文字被认为具有通灵辟邪之力——道家符箓、门神对联皆源于此。**墨字破妖**将这一文化基因转化为游戏机制：

- **毛笔是武器** — 鼠标/手指化为毛笔光标，在宣纸上书写
- **汉字是咒语** — 每写对一个部首，便击破一处妖邪弱点
- **识字是修行** — 从简单字（一、二、人）到复杂字（魔、龍、靈），难度层层递进

### 2.2 国风水墨美学

游戏视觉语言完全围绕中国传统水墨画展开：

| 元素 | 设计 |
|------|------|
| 底色 | 宣纸暖黄 (`#f5f0e8`) |
| 光标 | 竹杆毛笔，带墨迹拖尾 |
| 怪物 | 水墨晕染风格椭圆，部首/汉字标注其上 |
| 战场 | 松树、塔松、古建、山石等水墨背景障碍物 |
| 粒子 | 墨滴飞溅、消散效果 |
| 字体 | 楷体/宋体系列，保留书法韵味 |
| 音效 | 程序化合成——沙沙书写声、清脆成功叮、厚重锣鼓 BOSS 登场、五声音阶胜利旋律 |

### 2.3 识字教育内核

游戏暗含汉字教学逻辑：每个关卡对应一个汉字，将该字**拆解为部首组件**，玩家需要依次描摹每个部首。打 BOSS 时更要描摹完整汉字的所有笔画。玩家在不知不觉中学习汉字结构与书写笔顺。

## 三、核心玩法

### 3.1 游戏流程

```
┌─────────────┐
│  开始界面    │  按 Enter 或点击"开始游戏"
└──────┬──────┘
       ▼
┌─────────────┐
│  战场阶段    │  小怪在地图上移动 (AI: 闲逛/攻击/防御)
│  (PLAYING)  │  每 4 秒随机怪物暴露弱点 (金色光晕)
└──────┬──────┘
       │ 点击弱点
       ▼
┌─────────────┐
│  描摹弹窗    │  按住鼠标画线描摹部首轮廓
│ (DRAW_MODAL)│  DTW 算法匹配判定 (50% 容错)
└──────┬──────┘
       │ 描摹成功 → 小怪消灭
       │ 描摹失败 → 弹窗关闭，重新来过
       ▼
┌─────────────┐
│  收集部首    │  底部栏显示已收集/未收集部首
└──────┬──────┘
       │ 全部部首收集完毕
       ▼
┌─────────────┐
│  BOSS 合体  │  全屏黑幕 → 完整汉字浮现 → BOSS 登场
│ (BOSS_MERGE)│  铜锣音效 + 粒子爆发
└──────┬──────┘
       │ 点击 BOSS 弱点
       ▼
┌─────────────┐
│  BOSS 描摹  │  描摹完整汉字所有部首笔画
└──────┬──────┘
       │ 全部笔画描完 → BOSS 击败
       ▼
┌─────────────┐
│  通关结算    │  积分统计 + 道具掉落 + 已收集汉字展示
│  (RESULT)   │  + BOSS 击杀加分 (+100)
└──────┬──────┘
       │ 点击"下一关"
       ▼
     循环 ……
```

### 3.2 弱点机制

| 参数 | 值 |
|------|-----|
| 弱点触发间隔 | 每 4 秒 |
| 弱点持续时间 | 3 秒 |
| 弱点冷却时间 | 5 秒 |
| 弱点半径 | 30px |
| 视觉提示 | 金色扩散光环 + "弱点！"金色飘字 |
| 过期惩罚 | 弱点未点击自动消失 → 玩家 HP -1 |

### 3.3 描摹判定系统

- **算法**：简化 DTW (Dynamic Time Warping) + Douglas-Peucker 降采样
- **流程**：用户轨迹 → 降采样 → 归一化到 200×200 空间 → DTW 距离计算 → 评分
- **容错率**：50% 匹配即通过（面向全年龄段，儿童友好）
- **BOSS 模式**：笔画计数匹配——需要描摹正确的笔画数量

### 3.4 生存压力

| 压力来源 | 机制 |
|----------|------|
| 沙漏计时器 | 10 秒倒计时，归零扣 1 HP 并重置 |
| 弱点过期 | 弱点 3 秒未点击 → 扣 1 HP |
| 玩家 HP | 共 3 点「墨心」(Ink Heart) |
| 屏幕震动 | 受伤时震动反馈 |
| HP 归零 | 游戏结束，可选择看激励广告复活 |

## 四、道具系统

通关时随机获得 1 个道具（可叠加双倍道具效果获得 2 个），最多同时激活 2 个。

| 道具 | 图标 | 效果 |
|------|------|------|
| 一笔描摹 | ✒️ | 自动完成当前描摹判定 |
| 弱化妖力 | 💀 | BOSS 血量降为 1 |
| 双倍积分 | ✨ | 本关积分 ×2 |
| 双倍道具 | 🎁 | 通关获得双倍道具 |

### 5.2 状态机

```
START → PLAYING ⇄ DRAW_MODAL
                  ↓ (全部小怪消灭)
               BOSS_MERGE → PLAYING
                  ↓ (BOSS 击败)
                RESULT → PLAYING (下一关)
                  ↓ (HP 归零)
              GAME_OVER → PLAYING (重新开始 / 广告复活)
```

### 5.3 核心技术特点

- **Canvas 渲染**：全游戏在一个 Canvas 上完成，宣纸底色 + 噪点纹理
- **程序化音效**：基于 Web Audio API 合成，无需任何音频文件
- **DTW 手写识别**：简化的动态时间规整算法，50% 容错率，儿童友好
- **模块化 ES Module**：14 个 JS 模块，职责清晰
- **i18n 双语架构**：预加载词典，瞬时切换，支持 `Ctrl+L` 快捷键
- **GD SDK 双模式兼容**：同时支持 GD 平台注入式和 CDN 自托管式
- **无外部依赖**：纯原生 JavaScript + Canvas API

### 5.4 汉字数据格式

```json
{
  "char": "花",
  "difficulty": 2,
  "radicals": ["艹", "亻", "化"]
}
```

支持可选的 `strokes` 字段定义自定义描摹路径，回退到内置 16 种基本笔画映射表。

### 5.5 怪物系统

| 类型 | HP | 尺寸 | 弱点判定 | 描摹模式 |
|------|-----|------|----------|----------|
| Minion | 1 | 90×90 | 单部首路径匹配 | 单笔描摹 |
| Boss | 2* | 150×150 | 全部首笔画计数 | 多笔描摹 |

> *可通过「弱化妖力」道具降为 1

## 六、关卡难度曲线

| 关卡 | 难度 | 选字策略 |
|------|------|----------|
| 第 1~2 关 | 简单 | 仅难度=1 的汉字 (一、二、十、人等) |
| 第 3 关起 | 随机 | 全部汉字池，已用汉字移除，池空后重置 |

数据源包含 100 个汉字（已去重），难度 1~3，部首数 1~4 个。

## 七、商业化设计

通过 GameDistribution SDK 接入广告变现：

| 广告类型 | 触发时机 | 回调处理 |
|----------|----------|----------|
| 中场广告 | 关卡间隙（"下一关"按钮） | 广告完成后进入下一关 |
| 激励广告 | 游戏结束后可选择观看 | 观看完成 → HP 回满，继续游戏 |
| 广告暂停 | 广告播放期间 | 暂停游戏渲染循环 |

## 十、总结

**墨字破妖**是一款将中国汉字文化、水墨美学和休闲游戏机制有机融合的 HTML5 游戏。它以「手写描摹」为交互核心，让玩家在战斗闯关的过程中自然接触汉字的部首结构与书写方式。国风水墨的视觉风格与程序化音效营造出独特的东方美学氛围，而 DTW 手写识别算法的宽松容错设计确保了全年龄段的可玩性。

**核心金句**：*每一个汉字，都是一道有待书写的咒语。*
