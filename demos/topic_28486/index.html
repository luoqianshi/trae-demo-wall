一、Demo 简介

是什么

一款运行在 Windows 系统上的桌面悬浮窗考勤打卡应用，基于 Qt 6.10.2 框架开发，通过系统托盘和悬浮窗提供便捷的打卡体验。

面向谁

企业员工、自由职业者、学生等需要记录工作/学习时间的个人用户，尤其适合中小型团队和个人使用。后期可扩展日记功能，帮助用户清楚记录每天的工作内容和地点。

主要功能

🎯
悬浮窗打卡
双击悬浮窗即可快速打卡，支持圆角矩形/圆形/椭圆形三种形状，可自由调整大小和位置
⏰
智能提醒
上班前17分钟开始提醒，未打卡时每5分钟自动提醒一次
📋
多场景支持
正常打卡、补卡、外勤打卡全覆盖，支持自定义外勤地点
📊
统计分析
月度考勤统计、迟到/早退/外勤数据一目了然，支持CSV导出

二、Demo 创作思路

灵感来源

我观察到身边很多同事经常忘记打卡或错过打卡时间窗口，公司考勤系统不能时时查看，也记不清楚自己什么时候能打卡。有了这个系统能完整知道自己什么时候打卡，也不需要用excel表格记录，还能清楚外勤地点和工作内容。

想解决的问题

  • 操作繁琐：记录每天打卡时间过于繁琐，需要打开专门程序
  • 忘记打卡：缺乏即时提醒机制，容易错过打卡时间
  • 异常难追溯：迟到、早退等异常打卡记录分散，难以统一管理
  • 外勤难记录：外出工作时无法便捷记录外勤地点和时间
  • 统计不便：考勤数据统计需要人工汇总，效率低下

为什么做这个方向

选择桌面悬浮窗方案是因为：即时性——悬浮窗始终可见，随时可打卡；轻量化——无需打开专门程序，不占用系统资源；便捷性——双击即可打卡，操作耗时从数十秒缩短至毫秒级；私密性——数据本地存储，保障用户隐私安全。

四、TRAE 实践过程

开发流程

1. 需求分析：与 TRAE 对话，明确考勤系统的核心功能和技术选型
2. 架构设计：在 TRAE 辅助下设计系统架构，包括主窗口、悬浮窗、记录管理、设置等模块
3. 核心功能实现：悬浮窗设计、智能打卡逻辑、提醒系统、数据存储
4. UI设计：使用 Qt Designer 设计各类对话框界面
5. 测试优化：在 TRAE 帮助下排查问题、优化代码逻辑

关键步骤说明

  • 需求分析：与 TRAE 讨论考勤系统的核心功能和技术方案
  • 架构设计：确定模块划分和数据存储方案
