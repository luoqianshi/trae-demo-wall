TRAE
DEMO WALL ONE-STOP BROWSER

大赛论坛中数百个前端作品散落在不同页面，逐一翻阅效率极低。
TRAE Demo Wall 通过 Python 爬虫自动采集论坛数据，将所有网页/前端作品以卡片式画廊呈现，支持分类筛选、搜索排序，并直接在页面内 iframe 预览 Demo。

// 01 PROBLEM

WHY WE
BUILT THIS

论坛的帖子列表式浏览方式很难快速感知一个作品的真实效果。既然这些作品本质都是网页，为什么不把它们直接变成「可体验」的？

[01]

发现难

论坛帖子按时间排列，没有分类和搜索，找到感兴趣的作品全靠运气。

[02]

体验难

作品需要手动下载附件才能预览，很多用户看完标题就划走了。

[03]

学习难

想参考某个作品的实现方式，但帖子中往往没有提供在线 Demo 链接。

// 02 FEATURES

WHAT IT
DOES

从「逐帖翻阅 + 下载解压」到「打开网站 → 筛选 → 点击即体验」。每一个作品都可以直接在 iframe 中运行体验。

自动采集

Python 爬虫遍历 Discourse 论坛 JSON API，自动识别并过滤 web/前端类作品，生成结构化数据。

卡片画廊

三列响应式网格展示所有作品，每张卡片包含缩略图、类型标签、浏览量与点赞数。

搜索筛选

支持关键词搜索、标签分类筛选、按浏览量/点赞数/发布时间排序，快速定位目标作品。

即时预览

详情页以 16:9 iframe 沙盒直接运行作品 Demo，支持外部链接和本地部署双模式预览。

粒子背景

Canvas 2D 粒子系统，80 粒子 + 鼠标吸引 + 距离连线，营造沉浸式科技氛围。

懒加载

首屏仅加载轻量索引，滚动时通过 IntersectionObserver 按需分页加载，流畅无卡顿。
