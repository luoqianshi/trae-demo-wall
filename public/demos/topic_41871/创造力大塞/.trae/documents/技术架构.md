# 高中学习管理系统 - 技术架构文档

## 1. 架构设计

```
┌─────────────────────────────────────┐
│           UI 层 (HTML/CSS)          │
├─────────────────────────────────────┤
│         业务逻辑层 (JavaScript)      │
├─────────────────────────────────────┤
│           数据层 (LocalStorage)      │
└─────────────────────────────────────┘
```

**说明**：采用单文件HTML架构，无需后端服务器，数据存储于浏览器本地存储。

---

## 2. 技术选型

| 技术 | 版本 | 用途 |
|------|------|------|
| HTML5 | - | 页面结构 |
| CSS3 | - | 样式与动画 |
| JavaScript | ES6+ | 业务逻辑 |
| LocalStorage | - | 本地数据持久化 |

**优势**：
- 零依赖，无需安装任何包
- 单文件，可直接在浏览器打开
- 数据本地存储，保护隐私
- 轻量快速，适配各种设备

---

## 3. 页面路由

由于是单文件应用，采用哈希路由（#）实现页面切换：

| 路由 | 页面 | 说明 |
|------|------|------|
| #/ | 首页/选科 | 首次进入或选择学科 |
| #/overview | 学科总览 | 显示已选学科列表 |
| #/subject/:name | 学科详情 | 显示指定学科知识点 |
| #/point/:id | 知识点详情 | 显示知识点内容与评级 |

---

## 4. 数据模型

### 4.1 用户数据
```javascript
{
  grade: String,      // "高一" | "高二" | "高三"
  subjects: Array,    // ["语文", "数学", "英语", ...]
  createdAt: String   // 创建时间
}
```

### 4.2 知识点数据
```javascript
{
  id: String,         // 唯一标识 "科目-年级-序号"
  subject: String,    // 学科名称
  grade: String,      // 年级
  chapter: String,    // 章节/单元
  name: String,       // 知识点名称
  content: String,    // 核心内容（HTML格式）
  rating: Number,     // 评级 1-5
  lastReview: String  // 最近复习日期
}
```

### 4.3 学习记录
```javascript
{
  date: String,       // 日期 "YYYY-MM-DD"
  points: Array       // 当日学习/更新的知识点ID列表
}
```

## 5. 核心模块

### 5.1 Router 模块
- 监听 hashchange 事件
- 解析路由并渲染对应页面
- 管理页面切换动画

### 5.2 DataStore 模块
- 封装 LocalStorage 操作
- 提供 get/set 方法
- 自动序列化/反序列化 JSON

### 5.3 UI 组件
- **SubjectCard**：学科卡片组件
- **KnowledgePoint**：知识点条目组件
- **RatingSelector**：评级选择器组件
- **ProgressBar**：进度条组件

---

## 6. 文件结构

```
index.html          # 单文件应用（包含HTML/CSS/JS）
```

**文件内结构**：
```
├── <style>         # 所有CSS样式
├── <div id="app">  # 应用容器
└── <script>        # 所有JavaScript逻辑
    ├── 数据定义
    ├── 工具函数
    ├── UI渲染函数
    ├── 事件处理
    └── 初始化逻辑
```

---

## 7. 响应式设计

- **桌面端**：多列卡片布局，最大宽度1200px
- **平板端**：双列卡片布局
- **移动端**：单列卡片，触摸友好的点击区域

---

## 8. 性能考虑

- 使用事件委托减少事件监听器
- 按需渲染，非全量重绘
- CSS 动画使用 transform/opacity 触发GPU加速
- 图片懒加载（如有）
