### 1. Demo 简介

**是什么：**
脑衡 NeuroFocus 是一个脑电耳机式交互 Demo。它不是把脑电数据简单显示出来，而是尝试把 EEG 输入、专注音频、降噪模式、短休息音频引导、音乐恢复和外部音乐软件接入整合到一个日常耳机形态中。

通过模拟 EEG 数据，Demo 展示用户在学习、工作和情绪调节中的专注度、疲劳度、认知负荷、平静度、脑健康趋势以及声音场景使用状态，并用耳机声音反馈帮助用户调整节奏。NeuroFocus 不只是播放内置声音，而是模拟将脑状态反馈接入用户常用音乐软件（Spotify / Apple Music / 网易云音乐 / QQ 音乐），根据脑状态推荐声音场景，并在报告中记录声音场景切换与恢复效果。

**面向谁：**
- 学习工作人群：大学生、程序员、备考人群等长时间专注的用户
- 压力与情绪管理人群：需要放松与情绪调节的用户
- 音乐恢复爱好者：对脑电感知与音乐恢复结合感兴趣的用户

**主要功能：**

1. **脑电耳机式专注疲劳监测 + 音频反馈**
   模拟连接 NeuroFocus EEG Headset，通过「开始实时体验」入口（`startRealtimeExperience()`）自动完成设备连接、个人基线校准和 EEG 采集。页面显示专注度、疲劳度、平静度、认知负荷等指标。声音控制面板可切换专注音乐、White Noise、Breath Guide 和 Calm Ambient 模式，ANC 根据脑状态自动调整。

### 2. Demo 创作思路

**灵感来源：**
灵感来自脑电耳机类消费级硬件产品。传统脑电设备偏实验室化，佩戴复杂、门槛高；而脑电耳机更接近日常使用场景，可以像普通耳机一样佩戴，同时完成脑状态监测。更重要的是，普通脑电设备只能采集信号，而脑电耳机既能采集 EEG，也能通过音乐、语音、降噪和呼吸节奏给出反馈。我想借鉴这种"日常耳机 + 脑电感知 + 音频反馈"的产品形态，做一个更容易被普通用户理解和体验的 Demo。

**想解决的问题：**
在学习和工作场景中，很多人只能靠主观感受判断自己是否疲劳，但主观感受往往不准确。普通番茄钟只能按时间提醒，不能判断用户真实的脑状态。脑电耳机可以通过 EEG 检测脑状态，并用音频反馈帮助用户调整节奏，而不仅仅是提醒。

在情绪调节与放松场景中，很多人在压力下难以自我觉察和主动放松，普通白噪声 App 只能被动播放，无法根据真实脑状态调整。脑电耳机可以通过 EEG 检测疲劳与平静度，并用音乐恢复和声音场景主动引导用户进入放松或专注状态。

**为什么必须是耳机：**
1. 更自然的日常佩戴：相比脑电帽，耳机更接近学习、办公和放松场景中的真实使用习惯。
2. 输入与反馈一体：普通脑电设备只能采集信号，耳机既能采集 EEG，也能用音乐、语音和降噪回应你。
3. 不占用视觉注意力：学习工作和放松时用户需要看屏幕、看资料，耳机用声音引导，不抢眼睛。

**为什么做这个方向：**
我正在关注脑电专注疲劳检测和脑机接口方向，希望探索 EEG 输入与音频输出一体化的产品形态。因此我没有只做一个专注 App，而是把"脑电耳机式专注疲劳检测 + 音频反馈"与"音乐恢复 + 声音场景"结合起来。脑电耳机的独特价值在于：它把 EEG 输入、音频输出、降噪控制、短休息引导和音乐恢复整合到一个日常耳机形态中。

本作品选择音乐恢复方向，是因为耳机天然具备声音反馈能力。相比需要外接复杂设备的方案，音乐恢复更贴近耳机产品形态，也更能体现 EEG 输入与音频输出的一体化优势。

**关于音乐软件接入的设计理念：**
NeuroFocus 不只是播放内置声音，而是模拟将脑状态反馈接入用户常用音乐软件。耳机根据专注、疲劳和平静度变化，推荐 Focus Mix、White Noise、Breath Guide 或 Calm Ambient，并在报告中记录声音场景切换与恢复效果。NeuroFocus 不替代用户常用的音乐软件（Spotify / Apple Music / 网易云音乐 / QQ 音乐），而是连接到它们，将脑状态反馈转化为声音场景推荐。本功能为 Demo 模拟，不涉及真实账号授权。

初赛阶段不追求真实接入脑电硬件，而是先把硬件交互链路、音频反馈逻辑、用户体验流程和应用价值完整展示出来。通过模拟 EEG 数据，让评审可以在网页中体验"佩戴耳机 → EEG + 音频同步启动 → 耳机切换专注音乐/降噪 → 短休息引导 → 音乐恢复声音场景 → 接入外部音乐软件推荐 → 报告生成"的完整闭环。

### 科学严谨性设计

为了避免把脑电概念包装成伪科学，本 Demo 没有直接宣称"检测疾病"或"治疗焦虑"。页面中将 EEG 输出解释为趋势估计，并加入以下科学边界机制：

1. **信号质量 Signal Quality**：显示 Good / Medium / Low，信号差时降低指标可信度。
2. **伪迹风险 Artifact Risk**：检测眨眼、眼动、肌电等干扰，风险高时指标变灰并提示重新佩戴。
3. **判断置信度 Confidence Level**：每个指标附带 High / Medium / Low 置信度标签。
4. **个人基线**：不使用固定阈值，先建立个人基线再观察相对变化。
5. **脑电特征依据**：解释 Alpha / Theta / Beta 频段与指标的对应关系。
6. **推荐理由**：每个声音场景推荐附带具体理由和置信度。
7. **科学边界声明**：明确标注哪些结论不能过度解读。

音乐恢复部分被表述为"声音场景推荐"，而不是医学治疗。指标命名使用"估计"、"趋势"、"负荷"等谨慎措辞，避免"精准检测"、"准确识别"等过度宣称。

### 5. 作品价值与总结（按评分维度）

> **核心信息**：NeuroFocus 不是"脑电 + 音乐"的概念包装，而是完整的脑电耳机交互闭环：脑状态洞察 → 自动实时监测 → 专注短休息 → 音乐软件接入 → 音乐恢复摘要 → 科学解释边界。

#### 创新性（30%）

- **从音乐播放到脑状态反馈**：NeuroFocus 不只是播放内置声音，而是把 EEG 输入转化为脑状态反馈，再接入用户常用音乐软件（Spotify / Apple Music / 网易云音乐 / QQ 音乐），根据专注、疲劳和平静度推荐声音场景。
- **从单个分数到趋势判断**：不依赖某个瞬时分数下结论，而是基于个人基线观察专注趋势、疲劳负荷和平静度的相对变化，转化为 Brain Break 和声音场景推荐。
- **EEG 不确定性纳入交互设计**：信号质量（Signal Quality）、伪迹风险（Artifact Risk）和判断置信度（Confidence Level）不是后台隐藏参数，而是直接设计进交互流程，避免把噪声误判为脑状态变化。
- **从概念展示到硬件交互路径**：首页新增「硬件交互路径」模块，把脑电耳机的佩戴 → 采集 → 反馈 → 恢复链路可视化呈现。

#### 实用性（30%）

NeuroFocus 面向四个真实场景：

- **学习备考**：长时间复习时疲劳负荷连续升高，系统推荐 Brain Break 短休息引导，帮助恢复专注。
- **办公专注**：根据脑状态推荐 Focus Mix 或 White Noise，在疲劳出现前主动调整声音场景。
- **科研创作**：记录专注波动和恢复效果，生成每日脑状态报告辅助复盘。
- **情绪恢复**：推荐 Calm Ambient 或 Breath Guide 帮助放松（声音场景建议，非医学治疗）。

Demo 验证了核心价值链路：连接 → 监测 → 基线 → 音乐推荐 → 恢复报告，完整闭环可在网页中直接体验。

#### 完成度（20%）

- **实时体验**：点击「开始实时体验」后自动完成佩戴检测、个人基线建立并开始实时监测（`startRealtimeExperience()` 高层入口），附带进度 checklist 展示完整流程。
- **实时 EEG 趋势**：专注趋势、疲劳负荷、信号质量、伪迹风险、置信度全程可见。
- **音乐软件接入**：Spotify / Apple Music / 网易云音乐 / QQ 音乐 Demo Connector（模拟接入，不涉及真实账号授权）。
- **Brain Break**：iOS bottom sheet 短呼吸引导，疲劳升高时自动触发。
- **每日报告**：音乐恢复摘要、推荐理由、科学边界一并呈现。
- **评审 60 秒路径**：6 步完整体验（点击"开始实时体验" → 查看实时 EEG 趋势 → 连接音乐软件 → 查看推荐声音场景 → 触发 Brain Break → 生成每日脑状态报告）。

#### 设计体验（20%）

- **iOS 风格**：统一组件库（ios-card、ios-pill、ios-segmented、ios-bottom-sheet），页面过渡 is-entering / is-leaving、stagger reveal、count-up、segmented control。
- **Neurable 风格产品叙事**：首页按 Hero → 核心能力 → 仪表盘 → 专注 → 音乐 → 科学 → 规格的顺序组织，强调产品化表达。
- **科学解释折叠**：科学边界内容默认折叠，点击展开，避免铺满页面影响阅读节奏。
- **首页 5 个新模块**：「为什么需要 NeuroFocus？」「创新点」「适用场景」「60 秒评审体验路径」「硬件交互路径」，让评审在首页即可快速理解项目价值。

v29 交互优化与视觉重构：
- 页面切换状态机重构：`.page-stage` + `.page-scroll` 单容器叠层系统，彻底消除页面残留和抖动
- `navigate()` 清晰状态机：`_navLock` 防重入，离场页先移除 `is-active` 再加 `is-leaving`，340ms 后清理

- 高端产品官网视觉：Hero 产品发布页 + Product Details + Quick Facts 模块
- 专注页深蓝渐变 hero 卡 + EEG 线条绘制 + 渐变分数
- 音乐恢复页青蓝渐变推荐卡 + 呼吸圆圈动画 + 场景符号动画
- 报告页紫色渐变总览卡 + 高质量睡眠可视化
- 睡眠图：深色渐变评分卡 + 不同高度时间线 + conic-gradient 环图 + insight 卡
- 场景符号：Focus Mix 波形线 / White Noise 噪声点 / Breath Guide 呼吸圆环 / Calm Ambient 柔和光斑
- 页面切换：仅 opacity + translate3d，无 blur，inactive 暂停动画
- 支持 prefers-reduced-motion，移动端禁用 parallax
