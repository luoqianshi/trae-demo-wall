推荐算法

位次优先，分数兜底

高考录取本质是位次竞争。系统优先使用位次比（考生位次 / 院校录取位次）进行分类，当位次不可用时自动切换为分差模式。

ratio = userRank / schoolRank
ratio ≤ 0.75 → 保 ｜ ≤ 1.05 → 稳 ｜ ≤ 1.30 → 冲

每年取该院校最低投档分（最容易考上的专业组），跨年份按校名聚合，解决专业组代码逐年变化的问题。

开发过程

用 TRAE 全程驱动开发

从数据采集到前端实现，从配色设计到Bug修复，全程在 TRAE 中完成。

STEP 01

数据采集与清洗

Python 爬虫抓取 23 省教育考试院官方数据，OCR 识别图片表格，清洗去重后生成 25 万条结构化录取记录。

STEP 02

前端架构搭建

Tailwind CSS + 原生 JS 构建 SPA 应用，58MB 数据前端离线加载，无需后端即可运行全部功能。

STEP 03

智能算法实现

位次比 + 分差双重推荐算法，跨年份数据聚合，冲稳保四档自动分类与排序。

STEP 04

UI 精修与自检

蓝白配色体系设计、字体对比度优化、科类过滤、三年分数对比、全流程功能自检与 Bug 修复。
