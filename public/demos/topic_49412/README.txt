============================================================
  Kindchat（轻聊）使用说明
============================================================

一、程序简介
-----------
Kindchat 是一款轻量级匿名即时通讯工具，支持真人随机匹配和 AI 智能体
辅助对话。用户可在浏览器中直接使用，无需注册账号。

二、核心功能
-----------

1. 匿名随机匹配
   - 每位用户生成 6 位随机身份码（如 "A3XK9M"），保护隐私
   - 随机头像由背景色 + 颜文字表情组合生成，无需上传图片
   - 支持手动修改对方昵称和头像显示

2. 双端 AI 智能体
   - 用户可配置自己的大语言模型 API（支持 OpenAI 兼容接口）
   - 设置触发词后，对话中说出触发词即可唤醒 AI 智能体
   - 双 AI 模式：双方都开启 AI 时，各自智能体独立响应
   - 单 AI 模式：仅一方开启时，该方 AI 响应双方的触发词呼叫
   - AI 会自动分析对话上下文，判断是否需要介入调解
   - 违规内容检测：多级敏感词过滤 + AI 情感分析

3. 对话管理
   - 真人对话：结束前弹出确认框，确认后进入评分页面
   - AI 对话：直接结束，自动生成标题并保留在历史记录中
   - 历史记录：右键可删除对话，点击可重新进入（对方在线时）
   - 配色方案：服务器统一分配 AI 颜色，保证双方视角一致

4. 一键部署
   - 双击 "打开Kindchat.hta" 自动完成依赖安装、防火墙配置、服务启动
   - 支持 Windows 系统（.bat 脚本）和 Linux/Mac 系统（.sh 脚本）
   - 自动检测本机局域网 IP，访客在浏览器输入地址即可加入

三、技术实现
-----------

1. 前端（React + TypeScript + Tailwind CSS）
   - 状态管理：Zustand 轻量状态库
   - WebSocket 通信：实时双向消息推送
   - 响应式布局：适配桌面和移动端浏览器
   - 组件：ChatScreen（聊天界面）、MenuScreen（菜单/历史）、
     AvatarDisplay（头像）、HistoryView（历史记录）

2. 后端（Node.js + Express + WebSocket）
   - WebSocket 服务器：ws 库，路径 /ws
   - HTTP 服务：Express，提供静态文件 + 健康检查 + IP 查询
   - 匹配系统：匹配池管理，至少一方开启 AI 才可匹配
   - AI 调解者：调用外部大模型 API 分析对话，适时介入
   - 会话管理：内存存储对话数据，支持历史恢复

3. 网络部署
   - 服务器监听 0.0.0.0:12345，局域网内设备均可访问
   - 自动配置 Windows 防火墙放行端口
   - IP 检测：Node.js os.networkInterfaces() 按网卡优先级排序
     （WLAN/以太网 优先，排除 VPN/Virtual/Hyper-V 等虚拟网卡）
   - HTA 启动器：通过 .local_ip 文件 + /api/ip 接口获取本机 IP

4. AI 智能体逻辑
   - 触发词检测：每个 AI 各自检查消息中是否包含自己的触发词
   - 双 AI 模式：各自独立调用，绝对侧标识（left/right）防止混淆
   - 单 AI 模式：仅 AI 提供者响应，非提供者消息不触发
   - PASS 机制：AI 判断无需介入时回复 "PASS"，静默跳过
   - 系统提示词：明确告知 AI 只响应对应的触发词，避免身份混淆

四、文件结构
-----------
deploy/
├── 打开Kindchat.hta      # Windows 一键启动器（安装依赖+启动服务+打开浏览器）
├── 打开Kindchat.html     # 服务启动后快速访问入口
├── start.bat             # Windows 手动启动脚本
├── start.sh              # Linux/Mac 手动启动脚本
├── README.txt            # 本说明文件
├── Dockerfile            # Docker 容器化部署
├── render.yaml           # Render.com 云端部署配置
├── client/dist/          # 前端构建产物（HTML+JS+CSS）
├── server/dist/          # 后端编译产物（Node.js）
└── server/package.json   # 后端依赖清单

五、使用步骤
-----------
1. 确保已安装 Node.js（v16+）
2. 双击 "打开Kindchat.hta"（推荐）或运行 "start.bat"
3. 等待服务启动，浏览器自动打开页面
4. 将显示的地址（如 http://192.168.1.7:12345）发给对方
5. 对方在浏览器打开该地址即可进入
6. 如需开启 AI 智能体，在左侧菜单中配置 API 信息

六、API 配置说明
---------------
支持的模型：兼容 OpenAI API 格式的大语言模型
- 提供商：OpenAI / 自定义
- 接口地址：如 https://api.openai.com/v1/chat/completions
- API Key：你的 API 密钥
- 模型名称：如 gpt-4o / gpt-3.5-turbo / deepseek-chat
- 触发词：在对话中说出该词即可唤醒 AI（默认"小助手"）

七、注意事项
-----------
- 主机和访客需在同一局域网内
- 主机关闭服务后，所有连接中断
- 对话数据存储在内存中，服务重启后丢失
- AI 对话记录保存在浏览器本地存储中
- 防火墙需放行 12345 端口（脚本自动配置）
