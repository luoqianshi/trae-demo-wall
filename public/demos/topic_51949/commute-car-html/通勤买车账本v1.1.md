## 一、项目概述

### 1.1 一句话定位

**通勤买车账本**是一个面向跨城通勤和高频用车人群的**纯前端购车成本测算工具**。用户输入通勤里程、车辆价格、能源价格、保险保养等参数后，工具基于默认数据和固定公式，自动算出燃油车与纯电动车在全生命周期内的总成本、每公里成本、日均/月均成本，并给出可视化对比与智能建议。

### 1.2 核心特点

- **纯前端实现**：无需后端服务、数据库或服务器，直接打开 HTML 文件即可使用。
- **默认数据驱动**：所有计算参数均有合理默认值，用户无需选择具体车型，只需调整参数即可测算。
- **实时计算**：任何输入项修改后，结果、图表、对比卡片和 AI 建议都会即时更新。
- **双车型对比**：同时测算燃油车和纯电动车的成本，直观展示差距。
- **本地历史记录**：测算结果可保存到浏览器本地，支持导出、导入和清空。
- **数据可视化**：使用 ECharts 渲染成本构成柱状图、10 年堆叠柱状图、成本构成环形图。

### 1.3 与 AI 的关系

当前版本已接入**智谱 AI（GLM-4-Flash）**，基于用户输入的真实数据生成个性化购车建议。AI 输出内容严格基于计算结果，用具体数字说话，包括：

- 一句话结论（油车/电车二选一）
- 两种车型的驾驶风格与适用场景
- 针对用户里程、出险率等具体数据的风险提示

如果网络不可用或 AI 接口异常，工具本身的核心计算功能不受影响。

## 二、产品定位

### 2.1 目标用户

| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| 新手司机 | 第一次买车，不懂车 | 算清买车后每月、每年花多少钱 |
| 跨城通勤族 | 每天往返 50km+，高速占比高 | 油车还是电车更省 |
| 异地上班族 | 周末长途往返 | 长途用车成本对比 |
| 换车用户 | 从油车换电车，或反之 | 总成本差异量化 |
| 预算敏感型 | 10-30 万预算，精打细算 | 同预算下哪种动力更划算 |
| 家庭用户 | 注重空间和舒适 | 成本与用车场景匹配 |

### 2.2 核心使用场景

```
场景1：小李每天通勤 50km（往返），高速占 60%，预算 15 万
→ 输入参数后，对比燃油车和纯电车 10 年总成本
→ 关注：油费/电费、保养、保险、折旧

场景2：小王预算 30 万，追求驾驶感，偶尔商务接待
→ 调整购车价、油耗/电耗、残值率等参数
→ 关注：总持有成本、每公里成本、月均支出

场景3：老张从油车换电车，有家充桩
→ 设置低电价、高年里程、电车低保养费
→ 关注：电车每年省多少、保险涨多少、维修贵多少
```

## 六、成本计算模型

### 6.1 基础里程计算

```javascript
// 年通勤里程
annualCommuteMileage = oneWayDistance * 2 * commuteDaysPerMonth * 12;

// 年总里程
annualTotalMileage = annualCommuteMileage + extraAnnualMileage;

// 使用周期总里程
totalMileage = annualTotalMileage * useYears;
```

### 6.2 能源费用计算

```javascript
// 燃油车年能源费
annualFuelCost = (annualTotalMileage / 100) * fuelConsumption * fuelPrice;

// 纯电车年能源费
annualElectricityCost = (annualTotalMileage / 100) * electricConsumption * electricityPrice;
```

### 6.3 保险费用计算

```javascript
insuranceCoefficients = {
  "basic": 0.75,    // 基础保险
  "std": 1.0,       // 标准保险
  "full": 1.5       // 全保
};

annualInsuranceCost = baseInsurance * insuranceCoefficients[insuranceLevel];
```

### 6.4 保养费用计算

```javascript
// 基础保养费
annualMaintenanceCost = annualMaintenanceCount * singleMaintenanceCost;

// 纯电车额外增加电池/三电系统检测费用
if (carType === 'ev') {
  annualMaintenanceCost += 600;
}
```

### 6.5 维修费用计算

```javascript
annualRepairCost = annualAccidents * singleRepairCost;
```

### 6.6 折旧费用计算

```javascript
// 总折旧额（直线法）
totalDepreciation = carPrice * (1 - residualValueRate / 100);

// 年均折旧
annualDepreciation = totalDepreciation / useYears;
```

### 6.7 贷款利息计算

```javascript
// 贷款本金
loanPrincipal = carPrice * (1 - downPaymentRatio / 100);

// 等额本息总利息
monthlyRate = loanRate / 100 / 12;
numPayments = loanYears * 12;

if (monthlyRate === 0) {
  monthlyPayment = loanPrincipal / numPayments;
} else {
  monthlyPayment = loanPrincipal * monthlyRate / (1 - Math.pow(1 + monthlyRate, -numPayments));
}

totalInterest = monthlyPayment * numPayments - loanPrincipal;
annualInterest = loanYears > 0 ? totalInterest / loanYears : 0;
```

### 6.8 年总成本计算

```javascript
annualTotalCost = annualEnergyCost
                + annualInsuranceCost
                + annualMaintenanceCost
                + annualRepairCost
                + annualDepreciation
                + (homeParkingFee + workParkingFee + otherParkingFee) * 12
                + monthlyToll * 12
                + monthlyCarWash * 12
                + annualTax
                + annualInterest;
```

### 6.9 全周期总成本与单位成本

```javascript
// 全生命周期总支出（结果区展示）
totalCost = (annualTotalCost - annualInterest) * useYears + totalInterest;

// 每公里成本
costPerKm = totalCost / totalMileage;

// 日均成本
costPerDay = totalCost / (useYears * 365);

// 月均成本
costPerMonth = totalCost / (useYears * 12);
```

### 6.10 双车对比差额

```javascript
// 燃油车 vs 纯电车差额
gap = gasTotalCost - evTotalCost;

// 差额百分比
gapPct = Math.abs(gap) / Math.max(gasTotalCost, evTotalCost) * 100;

// 月均差额
monthlyGap = gap / useYears / 12;
```

## 八、AI 智能顾问

### 8.1 功能说明

AI 顾问模块基于当前测算数据，调用智谱 AI GLM-4-Flash 模型生成购车建议。建议内容严格基于用户输入的具体数字，避免空泛结论。

### 8.2 输出结构

AI 输出采用 Markdown 格式，包含以下章节：

1. **一句话结论**：直接给出燃油车和纯电车二选一建议。
2. **两种车的驾驶风格**：
   - 燃油车：适合追求机械质感、长途自驾的用户
   - 纯电车：适合追求舒适科技、城市通勤的用户
3. **风险提示**：针对年里程、出险率、家充条件等给出具体建议。

### 8.3 关键数据引用

AI 建议中必须引用的核心数据包括：

- 年通勤里程
- 当前选择车型
- 10 年总成本
- 年均出险次数
- 燃油车和纯电车各自的 10 年总成本
- 年能源费差距
- 每月成本差额

### 8.4 标签展示

AI 建议下方会生成若干标签，例如：

- `年里程较高`
- `建议家充桩`
- `电车更省`
- `出险率偏高`

## 十、数据存储

### 10.1 存储方式

当前版本为纯前端工具，所有数据均存储在浏览器端：

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 用户输入参数 | 页面状态 / localStorage（历史记录） | 不保存到服务器 |
| 测算结果 | 实时计算 / localStorage（历史记录） | 不保存到服务器 |
| 历史记录 | `localStorage` | 键名：`commute_car_history` |
| 主题/偏好 | 暂不支持 | 后续可扩展 |

### 10.2 隐私说明

- 所有数据均保存在用户本地浏览器中。
- 除非用户主动导出并分享，否则任何第三方（包括开发者）无法获取用户输入数据。
- AI 建议调用智谱 AI 接口时，仅发送必要的测算数据文本，不发送用户身份信息。

## 十三、使用说明

### 13.1 本地运行

方式一：直接打开

```
双击 commute-car-calculator.html 即可在浏览器中打开
```

方式二：本地服务器（推荐，避免部分浏览器安全限制）

```bash
# 进入项目目录
cd commute-car-html

# Python 3
python -m http.server 8080

# 浏览器访问
http://localhost:8080/commute-car-calculator.html
```

### 13.2 使用流程

1. 打开页面，查看默认测算结果。
2. 在"通勤基础"Tab 中修改自己的通勤距离、天数和路况占比。
3. 在"车辆配置"Tab 中设置购车价、首付、贷款年限等。
4. 在"日常成本"Tab 中调整油价/电价、停车费等。
5. 在"风险维护"Tab 中分别为燃油车和纯电车设置保险、保养、维修、残值率。
6. 查看结果卡片、图表、双车对比和 AI 建议。
7. 点击"保存当前测算到历史记录"保存当前方案。
8. 切换到"历史记录"视图查看、载入、导出或删除方案。

### 13.3 注意事项

- 工具基于默认参数和公式计算，结果仅供参考，不构成实际购车建议。
- 实际成交价、保险费用、维修费用等因地区和时间差异较大，建议用户根据当地实际情况调整。
- 历史记录保存在当前浏览器中，清除浏览器数据会丢失。
