💡 创意名称与介绍

创意名称

心灯 AI —— 校园心理健康智能守护平台

想解决什么问题

高校学生心理困扰日益普遍，但求助门槛高、发现不及时、资源有限：学生不敢开口、老师难以及时察觉、心理中心人手不足，导致小问题拖成大危机。

为什么会想到做这个

身边同学因学业压力、人际关系产生焦虑却不愿去心理中心；学校心理老师反映「等学生主动来，往往已经晚了」。我们希望用 AI 降低倾诉门槛，用数据辅助老师早发现、早介入，让守护像一盏温暖的灯，始终亮着。

大概是什么产品

一套面向高校的心理健康智能守护系统，包含 Web 管理端 + 学生端小程序，集成 AI 对话、情绪追踪、心理测评、匿名树洞与危机预警后台。

👥 目标用户及痛点

核心用户

高校在校学生

18–25 岁，面临学业、就业、情感等多重压力，需要私密、低门槛的心理支持渠道。

服务提供者

心理老师 & 管理员

负责日常咨询、危机干预与校园心理健康数据管理，需要高效的学生状态洞察工具。

使用场景

何时使用

深夜失眠焦虑时与 AI 倾诉；每日心情打卡；考前做压力测评；匿名发布树洞；老师收到预警后主动关怀。

当前痛点

没有产品时

学生独自硬扛或只在崩溃后才求助；老师无法掌握学生整体情绪趋势；危机信号分散在聊天、日记中难以汇总。

🎯 价值与意义

社会价值

降低心理健康求助羞耻感，让「开口」变成日常；辅助学校构建「预防—发现—干预」体系，减少校园心理危机事件，守护青年成长。

效率提升

AI 承担 7×24 初筛与陪伴，释放心理老师精力聚焦高危个案；数据大屏与预警溯源让干预决策有据可依，管理效率显著提升。

⚙️ 技术实现

Vue 2 + Element UI Spring Boot 2.7 MyBatis + MySQL JWT 鉴权 DeepSeek API uni-app 小程序 ECharts 可视化
【学生端】 Web / 小程序 → AI 聊天 · 打卡 · 树洞 · 测评
【API 层】 Spring Boot REST · JWT · 角色权限控制
【智能层】 DeepSeek 对话 · 情绪分析 · 预警触发引擎
【管理端】 心理老师预警后台 · 管理员数据大屏
【数据层】 MySQL · 用户/聊天/打卡/预警/树洞 等 10+ 张表

预警触发逻辑（已实现）

  • 极端词汇检测（聊天/树洞）→ 高危预警
  • 中度负面词 + 低落/焦虑分值 → 轻/中度预警
  • 近 7 天多条负面情绪或连续 3 天负面打卡 → 趋势预警
  • 心理测试中度/重度结果 → 自动预警并提升风险等级
