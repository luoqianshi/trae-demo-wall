创意名称 & 创意介绍
用 AI 解决嵌入式开发的信息碎片化问题

想解决什么问题

嵌入式开发入门门槛高，初学者面对琳琅满目的开发板（Arduino、ESP32、STM32、树莓派等）常常不知道如何选择。查阅规格参数、引脚图、通信接口等信息需要在多个数据手册和网页间来回切换，效率低下。不同板子的编程框架和引脚编号差异大，新手容易接错线、写错代码。

为什么会想到做这个

在学习嵌入式开发的过程中，经常需要查板子规格、对比不同开发板的性能、确认引脚接线方式，还要根据板子型号编写对应的初始化代码。这些信息分散在各种 datasheet 和论坛帖子里，查找和整理非常耗时。于是想到用 AI + MCP 协议把这些信息整合起来，让 AI 直接变成嵌入式开发顾问。

大概是什么产品

一个基于 MCP（Model Context Protocol）协议的 AI 嵌入式开发助手，通过标准化的工具接口让 AI 能够：

  • 查询 19 种主流开发板的完整规格参数和引脚图
  • 自动识别 USB 连接的开发板型号
  • 通过 STM32 Bootloader 协议探测芯片硬件信息
  • 一键生成 9 种常用代码模板（Blink、串口、WiFi、I2C、SPI、PWM、ADC、中断等）
  • 多板对比，辅助选型决策

目标用户及痛点
面向嵌入式初学者和创客爱好者

面向哪些用户

  • 嵌入式开发初学者
  • 高校电子/自动化/计算机专业学生
  • 创客 / DIY 爱好者
  • STEM 教育工作者

在什么场景下使用

  • 刚接触嵌入式，不知道选哪块板子入门
  • 做项目时需要快速查阅引脚定义和接口参数
  • 需要在多块开发板之间做选型对比
  • 板子插上电脑后不确定是否被正确识别
  • 需要快速生成某个功能的基础代码模板

当前痛点

  • 信息碎片化：开发板规格散落在不同 datasheet 和网页中，查找一块板子的完整参数需要打开 5-6 个网页
  • 引脚图不直观：需要翻阅原理图 PDF 才能确认引脚定义，新手经常接错线导致硬件损坏
  • 选型困难：面对 Arduino、ESP32、STM32 等不同平台，缺乏直观的对比工具辅助决策
  • 设备识别麻烦：新手不知道 USB 连接后电脑有没有正确识别板子，排查驱动问题费时费力
  • 重复编码：每次新建项目都要从零写初始化代码，Blink、串口、I2C 这些基础代码反复编写

价值与意义
从效率提升和社会价值两个角度阐述

⚡ 效率提升

将原本需要翻阅多份 datasheet、对比多个网页的开发板选型工作，压缩到一句话查询即可完成。USB 自动识别功能让开发者插上板子就能知道是什么型号、支持什么接口，省去手动排查的时间。9 种代码模板覆盖了嵌入式开发最常用的场景，从 Blink 到 WiFi 扫描一键生成，大幅减少重复编码工作。

🌎 社会价值

降低嵌入式开发的入门门槛，让更多初学者和创客能快速上手硬件开发。在教育场景中，老师可以用这个工具快速演示不同开发板的差异，学生可以专注于学习编程逻辑而不是纠结于硬件细节。这对于推广 STEM 教育、培养硬件开发人才具有积极意义。

代码模板生成示例

AI 根据板子型号自动选择正确的引脚和编程框架，生成可直接编译的代码：

Arduino - Blink // ESP32-DevKitC V4 — Blink 示例 // 框架: Arduino (ESP32 Core) // LED 引脚: GPIO 2 void setup() { pinMode(2, OUTPUT); Serial.begin(115200); Serial.println("ESP32-DevKitC V4 Blink!"); } void loop() { digitalWrite(2, HIGH); delay(500); digitalWrite(2, LOW); delay(500); }
Arduino - WiFi Scan // ESP32-DevKitC V4 — WiFi 扫描示例 // 框架: Arduino (ESP32 Core) #include <WiFi.h> void setup() { Serial.begin(115200); Serial.println("WiFi Scanner..."); WiFi.mode(WIFI_STA); WiFi.disconnect(); } void loop() { int n = WiFi.scanNetworks(); for (int i = 0; i < n; i++) { Serial.print(WiFi.SSID(i)); Serial.print(" ("); Serial.print(WiFi.RSSI(i)); Serial.println(" dBm)"); } delay(10000); }

技术架构
基于 MCP 标准协议，兼容所有支持 MCP 的 AI 客户端

技术栈

  • 协议层：MCP (Model Context Protocol) stdio 传输
  • 运行环境：Python 3.10+
  • 依赖库：mcp、pyserial
  • 硬件通信：USB VID/PID 识别 + 串口 Bootloader 协议
  • 数据库：JSON 格式开发板规格库 (19 块板)

兼容客户端

  • TRAE IDE — 原生 MCP 支持
  • Claude Desktop — Anthropic 官方客户端
  • Cursor — AI 代码编辑器
  • 任何支持 MCP 协议的 AI 工具
