# 情感树洞 AI Demo - 系统功能说明书

> 版本：v2.0.0（前后端分离 + 数据库持久化改造）
> 更新日期：2026-06-26
> 文档范围：本说明书完整记录 Demo 当前的功能逻辑、交互细节、技术架构与本次优化点。

## 一、系统概述

情感树洞 AI 是一款面向青少年心理健康陪伴的应用，以"情感树洞"为隐喻，提供 AI 对话倾诉、情绪记录、AI 绘画疗愈、个人成长档案四大核心功能。本次 v2.0.0 升级将原纯前端 Demo 改造为**前后端分离 + MySQL 数据库持久化**架构，所有硬编码数据（AI 回复库、画作模板、用户见证、成就、统计数据）全部迁移到数据库存储，前端通过 RESTful API 动态获取。

> 说明：本版本未接入真实大模型 API，AI 回复仍采用规则匹配引擎，但匹配逻辑、回复选择、避免重复、追问机制等全部由后端服务完成，数据存储于 MySQL 数据库的 `ai_scenes` + `ai_responses` 表中。

## 二、技术架构

### 2.1 整体架构（v2.0.0 升级）

- **前端**：HTML + CSS + JavaScript 单页应用（demo.html）+ 产品介绍页（emotion-tree-hole.html）
- **后端**：Node.js + Express，提供 RESTful API（服务地址 `http://localhost:3000/api`）
- **数据库**：MySQL，共 13 张表，覆盖用户、AI 对话、情绪记录、绘画、成就、用户见证、统计等
- **AI 引擎**：后端基于正则表达式 + 关键词匹配的规则引擎（场景匹配 + 优先级排序 + 避免重复 + 追问机制）
- **数据存储**：所有业务数据持久化到 MySQL，前端通过 API 增删改查
- **资源依赖**：本地静态图片资源（assets 目录，由后端 `/assets` 静态服务提供）

### 2.2 后端服务结构


## 三、功能模块说明

### 3.1 首页（Home）

#### 3.1.1 问候与情绪选择
- 根据系统时间显示动态问候语（早上好/下午好/晚上好）
- 提供 6 种情绪快速选择：开心、平静、焦虑、难过、生气、疲惫
- 选择情绪后显示对应的情绪解读卡片

#### 3.1.2 快速体验入口（本次新增）
- **位置**：情绪选择卡片下方
- **样式**：粉紫青三色渐变按钮，带流动动画
- **交互**：点击后自动跳转至 AI 对话页，并自动填充示例消息"我今天考试没考好，感觉很难过，不知道该怎么办"，800ms 后自动发送
- **价值**：降低首次使用门槛，让评委/用户 3 秒内体验到核心价值

#### 3.1.3 新手引导（本次新增）
- **触发条件**：首次登录后自动触发，通过 `localStorage.emotree_onboarding_done` 标记是否已展示
- **引导步骤**：3 步分步引导
  1. 第 1 步：高亮情绪选择卡片，提示"先选择你现在的情绪状态"
  2. 第 2 步：高亮 AI 树洞倾诉入口，提示"点击这里进入 AI 树洞倾诉"
  3. 第 3 步：高亮底部绘画 Tab，提示"或者试试 AI 绘画疗愈"
- **交互方式**：
  - 遮罩层覆盖非高亮区域，聚焦当前引导目标
  - 渐变色提示气泡（粉紫渐变），显示引导文字 + 步骤指示（如"1/3"）+ "下一步"按钮
  - 最后一步按钮文案变为"开始体验 ✓"
  - 右上角提供"跳过引导"按钮，可随时退出
  - 点击遮罩空白处也可跳过引导
- **价值**：帮助首次使用的评委/用户快速了解核心功能入口，降低认知成本

#### 3.1.4 功能入口宫格
- AI 树洞倾诉、AI 情绪绘画、情绪日记、冥想放松、心理测评、求助资源

### 3.2 AI 树洞对话（Chat）

#### 3.2.1 AI 回复库（v2.0.0 迁移至数据库）
- **存储位置**：MySQL `ai_scenes`（场景表）+ `ai_responses`（回复表）
- **场景扩展**：从原 4 个场景扩展至 12 个场景
  1. 学习压力大（6 条回复）
  2. 家庭关系（5 条回复）
  3. 孤独（4 条回复）
  4. 失眠（4 条回复）
  5. 人际关系（3 条回复）
  6. 自我怀疑（3 条回复）
  7. 情绪低落（3 条回复，含心理援助热线 400-161-9995）
  8. 焦虑（3 条回复）
  9. 愤怒（3 条回复）
  10. 迷茫（3 条回复）
  11. 感情（3 条回复）
  12. 积极情绪（3 条回复）
- **默认回复**：扩展至 6 条，避免重复感
- **回复总数**：60+ 条，覆盖青少年常见心理场景
- **数据维护**：回复内容可在数据库中直接增删改，无需修改前端代码

#### 3.2.2 匹配引擎（v2.0.0 迁移至后端）
- **接口**：`POST /api/chat/reply`
- **匹配方式**：后端正则表达式 + 关键词集合（替代原前端 includes 简单匹配）
- **优先级策略**：高安全风险场景（情绪低落/抑郁）优先匹配，确保涉及自伤倾向的关键词能触发专业引导
- **场景排除**：人际关系场景会排除家庭关键词，避免误判
- **避免重复**：后端通过会话维度记录每个场景上次使用的回复索引，同一场景连续触发时跳过上次回复
- **上下文感知**：后端保留当前会话最近 6 轮对话，当对话轮数 ≥ 2 时，50% 概率追加追问句（如"这种情况持续多久了？"），引导深入交流
- **数据持久化**：每轮对话（用户消息 + AI 回复）实时写入 `chat_messages` 表，会话信息写入 `chat_sessions` 表

#### 3.2.3 交互细节
- 打字指示器：AI 回复前显示"正在思考..."动画
- 消息气泡：用户右侧粉红渐变，AI 左侧深色卡片配头像
- 快捷话题：提供"学习压力大""和父母吵架""感觉很孤独""总是失眠"4 个快捷按钮
- **输入校验**：
  - 空输入提示"请输入你想说的话"
  - 超过 500 字提示"内容太长啦，请精简一下"
- **动态思考时间**：AI 思考时间根据输入文本长度动态计算（800ms + 每字15ms，上限 2500ms），更接近真实 AI 响应节奏
- **防重复发送**：发送期间禁用发送按钮并降低透明度，回复完成后恢复
- **错误降级处理**：try-catch 包裹 API 调用，失败时返回友好提示"我刚才走神了一下，能再跟我说一遍吗？"
- **对话历史记录**：每轮对话同步写入后端 `chat_messages` 表，支持后续上下文分析与历史查询

### 3.3 AI 情绪绘画（Art）

#### 3.3.1 绘画引擎（v2.0.0 迁移至数据库）
- **存储位置**：MySQL `art_templates`（画作模板表）+ `art_keywords`（情绪关键词表）
- **接口**：`POST /api/art/generate`
- **情绪类别匹配**：从原 3 张图片简单匹配，扩展为 6 种情绪类别 + 默认类别
  - positive（积极）：阳光/温暖/希望等 13 个关键词
  - lonely（孤独）：孤独/寂寞/一个人等 10 个关键词
  - calm（平静）：平静/安宁/放松等 11 个关键词
  - anxious（焦虑）：焦虑/紧张/害怕等 11 个关键词
  - sad（悲伤）：难过/伤心/失落等 10 个关键词
  - default（默认）：3 张通用图片轮换
- **匹配算法**：后端统计用户输入中各类别关键词命中次数，选择命中次数最多的类别（最高匹配优先）
- **数据持久化**：每次生成的画作记录（含图片URL、标题、解读、情绪类别）写入 `artworks` 表

#### 3.3.2 动态进度提示
- 生成过程中分阶段显示不同提示文字：
  - 0-30%：AI 正在理解你的情绪...
  - 30-60%：正在构思画面构图...
  - 60-90%：正在渲染色彩与细节...
  - 90-100%：即将完成...
- 进度条增长速度调整为更接近真实生成节奏（每 350ms 增长 2-10%）

#### 3.3.3 画作解读
- 每幅画作配有标题（如《希望的曙光》《月下独白》），数据来源于 `art_templates` 表
- 每幅画作配有 1-2 句解读文字，说明画面意境与情绪对应关系
- 解读区域采用青色半透明背景卡片，与品牌色一致

#### 3.3.4 画廊展示
- 横向滚动展示历史画作缩略图（数据来源于 `artworks` 表，通过 `GET /api/art/history` 获取）

#### 3.3.5 错误处理
- 输入校验：空输入提示"请先描述你的心情"；超过 200 字提示精简
- 生成失败降级：try-catch 包裹 API 调用，失败时隐藏进度区、恢复按钮、Toast 提示"生成失败，请稍后再试"
- 按钮状态管理：生成期间禁用按钮，防止重复触发

### 3.4 产品介绍页（emotion-tree-hole.html）

#### 3.4.1 用户见证模块（v2.0.0 迁移至数据库）
- **存储位置**：MySQL `testimonials` 表
- **接口**：`GET /api/testimonials?limit=4`
- **位置**：产品理念与未来展望之间
- **内容**：4 张用户见证卡片 + 1 个数据汇总区，均由后端 API 动态加载

  1. 小雨（高一学生）：AI 对话倾诉带来的陪伴感
  2. 阿明（初三学生）：放松技巧与呼吸练习的帮助

  4. 陈老师（中学心理教师）：作为专业辅助工具的推荐
- **卡片交互**：鼠标悬停时上移 4px + 阴影增强，提升立体感
- **降级处理**：API 调用失败时使用内置默认数据，保证页面可访问
- **数据汇总**：4 项核心数据，由 `GET /api/stats/summary` 接口返回
  - 98% 用户满意度
  - 5,200+ 内测用户
  - 12,000+ 倾诉对话
  - 3,500+ 治愈画作
- **价值**：增强产品说服力，让评委感受到真实用户反馈与数据支撑

### 3.5 我的页面（Profile）

#### 3.5.1 数据统计（v2.0.0 迁移至后端）
- **接口**：`GET /api/dashboard?user_id=1`
- 统计指标：累计对话次数、情绪记录天数、治愈画作数量、已解锁成就数
- 数据来源于 `chat_messages`、`mood_records`、`artworks`、`user_achievements` 表的实时聚合
- 切换到"我的"Tab 时自动触发 API 调用，刷新统计数据

#### 3.5.2 情绪趋势图表（v2.0.0 迁移至后端）
- **接口**：`GET /api/mood/trend?user_id=1&days=7`
- 7 天柱状图，每日柱高代表情绪指数，数据来源于 `mood_records` 表
- 颜色区分：成功（绿）、主色（青）、警告（黄）、危险（红）、强调（粉）
- 图表下方显示"本周情绪稳定性提升 23%"对比文案

#### 3.5.3 成就系统（v2.0.0 迁移至后端，动态计算）
- **接口**：`GET /api/achievements?user_id=1`
- **存储位置**：`achievements`（成就定义）+ `user_achievements`（用户进度）+ `achievement_rules`（判定规则）
- 成就列表（初始）：
  - 倾诉达人：完成 5 次 AI 对话
  - 艺术疗愈师：生成 3 幅情绪画作
  - 情绪记录者：连续 7 天记录情绪
- **动态计算**：后端根据用户实际行为数据（对话数、画作数、情绪记录天数）实时计算进度与解锁状态
- 每个成就显示图标、名称、达成条件、当前进度、状态标识（已解锁 ✓ / 进行中 / 未解锁）

#### 3.5.4 功能菜单
- 情绪档案、隐私设置、关于我们、退出登录

## 五、交互细节约定

- **Toast 提示**：所有轻量操作反馈使用顶部 Toast（2 秒自动消失）
- **页面切换**：底部 Tab 切换，无刷新动画；切换到"我的"Tab 自动加载后端数据
- **启动页**：2 秒后自动隐藏
- **空状态处理**：聊天页首次进入显示欢迎语和快捷话题
- **错误处理**：输入为空时 Toast 提示，不阻塞流程；API 调用失败时控制台记录错误并降级处理
- **会话管理**：前端生成 session_id（格式 `session_时间戳`），贯穿整个对话流程

## 六、已知限制（v2.0.0）

1. AI 回复为规则匹配，非真实大模型生成，无法处理极复杂语境（计划决赛阶段接入真实大模型 API）
2. 绘画作品为预设静态图片，非真实 AI 生成
3. 用户体系简化：默认 user_id=1（Demo 用户），未实现完整登录注册流程
4. 会话上下文存储在后端内存中，服务重启后丢失（对话历史已持久化到数据库）
