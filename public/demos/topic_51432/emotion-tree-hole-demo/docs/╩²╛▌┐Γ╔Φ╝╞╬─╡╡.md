## 一、数据库概述

情感树洞 AI 后端采用 MySQL 关系型数据库，共设计 13 张数据表，覆盖用户管理、AI 对话引擎、情绪记录、AI 绘画疗愈、成就系统、用户见证、统计指标等核心业务领域。

### 设计原则
1. **分层存储**：配置数据（场景、回复、模板）与业务数据（对话、情绪、画作）分离
2. **外键约束**：核心关联关系使用外键保证数据完整性，级联删除用户数据
3. **软删除预留**：关键字段使用 `is_active` 标记启用状态，支持逻辑删除
4. **JSON 关键词**：AI 场景关键词存储为 JSON 数组，便于灵活扩展
5. **优先级机制**：AI 场景使用 `priority` 字段控制匹配顺序，安全场景优先级最高

## 二、ER 关系图

```
users (用户)
  ├── chat_messages (对话记录) —— user_id
  ├── mood_records (情绪记录) —— user_id
  ├── artworks (用户画作) —— user_id
  └── user_achievements (用户成就) —— user_id
                                    └── achievement_id → achievements

ai_scenes (AI场景)
  └── ai_responses (AI回复) —— scene_id

art_templates (画作模板) ←—— art_keywords (画作关键词) via category
```

## 三、表结构详解

### 3.1 用户表 users

存储系统用户信息，Demo 阶段支持免密登录。

| 字段名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|------|--------|--------|------|
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 用户ID，主键 |
| username | VARCHAR(50) | 否 | - | 登录用户名，唯一 |
| nickname | VARCHAR(50) | 否 | - | 用户昵称 |
| avatar | VARCHAR(255) | 是 | NULL | 头像URL |
| password_hash | VARCHAR(255) | 是 | NULL | 密码哈希（Demo阶段允许为空） |
| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 注册时间 |
| last_login_at | DATETIME | 是 | NULL | 最后登录时间 |

**索引**：
- 主键：`id`
- 唯一索引：`uk_username (username)`

---

### 3.2 AI对话场景表 ai_scenes

存储 12 个情绪场景 + 1 个默认场景，每个场景包含匹配关键词和优先级。

| 字段名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|------|--------|--------|------|
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 场景ID，主键 |
| scene_key | VARCHAR(50) | 否 | - | 场景标识（如 study_pressure），唯一 |
| scene_name | VARCHAR(50) | 否 | - | 场景中文名（如 学习压力大） |
| priority | INT | 否 | 0 | 匹配优先级（数字越大越优先） |
| keywords | TEXT | 否 | - | 匹配关键词JSON数组 |
| exclude_keywords | TEXT | 是 | NULL | 排除关键词JSON数组 |
| description | VARCHAR(255) | 是 | NULL | 场景描述 |
| is_active | TINYINT | 否 | 1 | 是否启用：1是 0否 |
| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引**：
- 主键：`id`
- 唯一索引：`uk_scene_key (scene_key)`

**场景优先级说明**：
- 100：情绪低落（depression）- 安全风险最高，优先匹配
- 90：学习压力大（study_pressure）
- 80：家庭关系（family_relation）
- 70：孤独（lonely）
- 60：失眠（insomnia）
- 50：人际关系（relationship）
- 40：自我怀疑（self_doubt）
- 30：焦虑（anxiety）
- 20：愤怒（anger）
- 10：迷茫（confusion）
- 5：感情（love）
- 1：积极情绪（positive）
- 0：默认（default）- 兜底场景

---

### 3.3 AI回复库表 ai_responses

每个场景对应多条回复，支持避免重复机制。

| 字段名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|------|--------|--------|------|
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 回复ID，主键 |
| scene_id | BIGINT UNSIGNED | 否 | - | 关联场景ID，外键 |
| content | TEXT | 否 | - | 回复内容（支持HTML） |
| sort_order | INT | 否 | 0 | 排序序号 |
| is_active | TINYINT | 否 | 1 | 是否启用 |
| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引**：
- 主键：`id`
- 普通索引：`idx_scene_id (scene_id)`
- 外键约束：`fk_responses_scene` → `ai_scenes.id`，级联删除

---

### 3.4 AI追问话术表 ai_followups

对话上下文达到一定轮数后追加的追问句。

| 字段名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|------|--------|--------|------|
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 追问ID，主键 |
| content | VARCHAR(255) | 否 | - | 追问内容 |
| trigger_rounds | INT | 否 | 4 | 触发所需的最小对话轮数 |
| probability | DECIMAL(3,2) | 否 | 0.50 | 触发概率（0.00-1.00） |
| is_active | TINYINT | 否 | 1 | 是否启用 |
| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |

---

### 3.5 对话记录表 chat_messages

存储用户与AI的完整对话历史。

| 字段名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|------|--------|--------|------|
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 消息ID，主键 |
| user_id | BIGINT UNSIGNED | 否 | - | 用户ID，外键 |
| session_id | VARCHAR(64) | 否 | - | 会话ID |
| sender | ENUM('user','ai') | 否 | - | 发送方 |
| content | TEXT | 否 | - | 消息内容 |
| matched_scene | VARCHAR(50) | 是 | NULL | AI回复匹配的场景标识 |
| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 发送时间 |

**索引**：
- 主键：`id`
- 普通索引：`idx_user_id`、`idx_session_id`
- 外键约束：`fk_messages_user` → `users.id`，级联删除

### 3.6 情绪记录表 mood_records

| 字段名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|------|--------|--------|------|
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 记录ID，主键 |
| user_id | BIGINT UNSIGNED | 否 | - | 用户ID，外键 |

| mood_score | INT | 否 | 50 | 情绪指数（0-100） |
| note | VARCHAR(500) | 是 | NULL | 情绪备注 |
| record_date | DATE | 否 | - | 记录日期 |

**索引**：

### 3.7 画作模板表 art_templates

替代前端硬编码的 artImageMap，存储 6 种情绪类别 + 默认类别。

| 字段名 | 类型 | 允许空 | 默认值 | 说明 |
|--------|------|--------|--------|------|
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 模板ID，主键 |
| category | VARCHAR(30) | 否 | - | 情绪类别 |
| image_url | VARCHAR(255) | 否 | - | 图片路径 |
| title | VARCHAR(100) | 否 | - | 画作标题 |
| description | TEXT | 否 | - | 画作解读 |
| sort_order | INT | 否 | 0 | 排序序号 |
| is_active | TINYINT | 否 | 1 | 是否启用 |
| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引**：
- 主键：`id`
- 普通索引：`idx_category (category)`

**类别说明**：
- positive（积极）：阳光、温暖、希望相关
- lonely（孤独）：孤独、寂寞相关
- calm（平静）：平静、安宁相关
- anxious（焦虑）：焦虑、紧张相关
- sad（悲伤）：难过、伤心相关
- default（默认）：无法匹配时的通用模板

### 3.11 用户成就关联表 user_achievements

| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |

## 五、执行说明

1. 执行 SQL 文件前确保 MySQL 8.0+ 已安装并运行
2. 直接执行 `db/emotree.sql` 文件，脚本会自动创建数据库 `emotree`
3. 执行顺序：建库 → 建表 → 插入初始化数据
4. 外键约束已设置级联删除，删除用户会自动清理其相关数据
