## 三、优化计划（执行顺序）

### Phase 1：LLM 管道打通（让产品"活"起来）
1. 接入 DeepSeek API（deepseek-v4-flash）
2. 实现基础对话管道（用户输入 → LLM → 回复）
3. 实现 System Prompt 组装（注入用户画像摘要）
4. 实现 RAG 检索（从 memories/persons 召回相关记录）

### Phase 2：多轮审核真实化
5. 实现方案写手 Agent（调用 LLM 生成初稿）
6. 实现审核 Agent 链（关系顾问→心理顾问→生活顾问→主持人）
7. 审核过程可视化与真实状态同步

### Phase 3：记忆飞轮
8. 对话结束后自动提取记忆（调用 LLM）
9. 低置信度记忆进入"待确认"队列
10. 用户确认后写入 Dexie 数据库

### Phase 4：场景功能补齐
11. 日记录入界面
12. 人物档案编辑
13. 每日简报生成
14. 见前简报（基于日历/提及的人物）

### Phase 5：体验优化
15. 语音输入
16. 移动端适配
17. 数据导出
