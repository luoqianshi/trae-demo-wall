# 事件缓冲窗口快照

## 背景

`EventBuffer` 是一个进程内的事件缓冲：业务持续把事件（用整型表示）追加进来，
监控/上报模块则随时取**最近 n 条**或**全部**事件的快照，
拿去做本地加工——排序、改写、追加哨兵值等，**互不影响缓冲本身**。

## 对外接口

- `NewEventBuffer() *EventBuffer`：创建空缓冲。
- `Add(v int)`：向缓冲追加一个事件。
- `Len() int`：当前缓冲中的事件数量。
- `Recent(n int) []int`：返回最近 `n` 条事件（不足 `n` 条则返回全部），顺序与加入顺序一致。
- `Snapshot() []int`：返回当前全部事件。

## 对外保证（不变量）

1. **快照独立**：调用方对 `Recent(n)` / `Snapshot()` 的返回值做任何修改（改元素、`append`、排序等），
   **都不得影响缓冲内部状态**，也不得影响之前或之后取出的其它快照。
2. **内容正确**：`Recent(n)` 返回的是按加入顺序的最近 n 条；`Snapshot()` 返回全部。
3. **缓冲自洽**：缓冲自身的 `Add` / `Len` / 后续取快照不受调用方对历史快照加工的干扰。

## 错误现象

- 只读取快照、从不修改时，一切正常。
- 但只要对取回的快照做了**就地修改或 `append`**，缓冲内部的事件数据竟会**跟着被改动**，
  导致后续读到的快照出现莫名其妙的脏值。
