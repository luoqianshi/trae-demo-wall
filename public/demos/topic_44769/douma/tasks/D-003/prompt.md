# 节点图注册表

## 背景

`NodeGraph` 是一个进程内的节点图：调用方通过 `add` 创建节点并拿到节点的**所有权句柄**
（`Rc<RefCell<Node>>`），节点的真正生命周期由调用方手中的句柄决定——
当调用方不再持有某节点的任何句柄时，该节点即视为**已释放**。

图自身还维护一个按 `id` 的**查找注册表**，方便用 `get(id)` 找回某个**仍然存活**的节点。
但这个注册表只是辅助索引，**不应延长节点寿命**：它不能仅仅因为自己记了一笔，
就让一个调用方已经放手的节点继续“赖着不走”。

## 对外接口

- `NodeGraph::new()`：创建空图。
- `add(&mut self, id, value) -> Rc<RefCell<Node>>`：创建节点并返回其句柄；同时登记进查找注册表。
- `get(&self, id) -> Option<Rc<RefCell<Node>>>`：按 id 找回**仍存活**的节点句柄；若该节点已被调用方释放或不存在，返回 `None`。
- `live_count(&self) -> usize`：返回当前**仍被调用方持有**的节点数量。
- `Node` 含公开字段 `id: usize` 与 `value: i64`。

## 对外保证（不变量）

1. **随手放随回收**：当调用方丢弃（drop）某节点的全部句柄后，该节点即视为已释放——
   `live_count()` 不再计入它，`get(id)` 对它返回 `None`。
2. **存活可寻**：只要调用方还持有某节点句柄，`get(id)` 必须能找回**同一个**节点（共享同一份内部状态）。
3. **计数精确**：`live_count()` 始终等于此刻调用方仍持有句柄的节点数量。

## 错误现象

- 创建节点后一直持有句柄时，`get` 与 `live_count` 都正确。
- 但一旦调用方 drop 掉某些节点句柄，本应随之释放的节点却**仍被算作存活**：
  `live_count()` 偏高，`get(id)` 还能“找回”一个早该消失的节点。
