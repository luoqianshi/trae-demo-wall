# 购物车构建器

## 背景

`CartBuilder` 是一个购物车构建工具：业务每次为一位用户新建一个购物车（可带初始商品，也可空车起步），
之后往各自的购物车里加商品、算总数。不同用户、不同次构建出来的购物车，**彼此应当完全独立**。

## 对外接口

- `CartBuilder()`：创建一个构建器。
- `new_cart(self, items=...) -> Cart`：新建一个购物车。可选地传入初始商品列表；不传则得到一个**空车**。
- `Cart.add(self, item)`：向该购物车加入一件商品。
- `Cart.items(self) -> list`：返回该车当前的商品列表（顺序与加入顺序一致）。
- `Cart.count(self) -> int`：返回该车的商品件数。

## 对外保证（不变量）

1. **实例独立**：通过 `new_cart()` 得到的每个购物车互不影响——
   向一个车 `add` 商品，**绝不能**改变任何其它车（包括之前或之后新建的、以及不带参数新建的空车）。
2. **空车真空**：不带初始商品调用 `new_cart()` 得到的车，初始件数必须为 0，
   且这一点**不因之前创建过多少购物车、加过多少商品而改变**。
3. **计数一致**：`count()` 与 `items()` 始终反映该车自身的真实内容。

## 错误现象

- 单独创建一个购物车并使用时，一切正常。
- 但连续创建多个购物车、并往其中一些车里加商品后，再新建的“空车”竟然**不是空的**，
  或不同车之间的商品互相串味，仿佛它们共用了同一份商品清单。

## 你的任务

定位并修复缺陷，使上述三条不变量在任意创建/加入顺序下都严格成立。
