# 命名计数器注册表

## 背景

`MetricRegistry` 是一个进程内的命名计数器注册表，类似监控库里的 counter handle 机制：
业务代码在启动或运行期向它登记一个命名指标，拿到一个**长期持有的句柄**，
之后在热路径上直接通过句柄累加，避免每次按名字查找。

## 对外接口

- `long* counter(const std::string& name)`：登记或获取名为 `name` 的计数器，返回指向其计数值的句柄。同名重复调用返回**同一个**计数器。
- `long value(const std::string& name) const`：读取该计数器当前值；不存在返回 0。
- `size_t size() const`：返回已登记的计数器数量。

## 对外保证（不变量）

1. **句柄长期有效**：`counter()` 返回的句柄，在 `MetricRegistry` 对象存活期间始终指向同一个计数器——**无论之后是否又登记了更多计数器**。
2. **累加可见**：通过句柄对计数值做的任何修改，随后用 `value(name)` 读取都必须一致。
3. **同名同体**：同名 `counter()` 返回的句柄与 `value(name)` 始终对应同一份计数值。

## 错误现象

- 先把所有指标一次性登记完，再取句柄使用，一切正常。
- 但若**先取到句柄、之后又继续登记新的计数器**，再通过早先的句柄累加时，
  `value()` 读到的值与累加量对不上：累加仿佛“写丢了”。
