# 会话池过期清理

## 背景

`SessionPool` 是一个进程内会话池：每个会话有一个整型 `id` 和一个 `ttl`（剩余存活计数）。
系统周期性调用一次清理，把所有**已过期（`ttl == 0`）**的会话从池中移除，
保留仍然有效的会话，且保持它们原有的相对先后顺序。

## 对外接口

- `void add(int id, int ttl)`：向池尾部追加一个会话。
- `void purge_expired()`：移除池中**所有** `ttl == 0` 的会话，保留其余会话及其相对顺序。
- `size_t size() const`：返回当前池中会话数量。
- `std::vector<int> ids() const`：按当前池中顺序返回所有会话的 `id`。

## 对外保证（不变量）

1. **彻底清除**：`purge_expired()` 之后，池中**不得残留任何** `ttl == 0` 的会话——无论过期会话出现在何处、是否相邻、是否位于末尾。
2. **有效保留**：所有 `ttl != 0` 的会话必须全部保留，且相对顺序不变。
3. **计数一致**：`size()` 与 `ids()` 始终反映清理后的真实内容。

## 错误现象

- 池中过期会话彼此分散（互不相邻）时，清理结果正常。
- 但当**两个及以上过期会话相邻**、或**过期会话恰好落在池末尾**时，清理后偶有过期会话残留，甚至程序行为异常。
