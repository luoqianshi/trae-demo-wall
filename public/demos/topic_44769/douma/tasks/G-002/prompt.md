# 列表分页器

## 背景

`Paginator` 是后台列表展示用的分页工具：按固定的每页条数 `page_size`，把一份总量为 `total` 的数据集切成若干页，
供前端逐页拉取。前端会先问“一共多少页”，再按 1 起始的页号逐页取数据，因此**页数必须把最后不足一页的零头也算进去**，
否则尾部数据将永远无法被翻到。

## 对外接口

- `Paginator(page_size)`：创建分页器，`page_size` 为每页条数（正整数）。
- `page_count(self, total) -> int`：给定总条数 `total`，返回总页数；`total <= 0` 时返回 `0`。
- `slice(self, items, page) -> list`：返回第 `page` 页（**1 起始**）的数据切片；页号越界（小于 1 或超出末页）时返回空列表。
- `last_page_size(self, total) -> int`：返回最后一页实际包含的条数；`total <= 0` 时返回 `0`。

## 对外保证（不变量）

1. **不丢尾页**：只要 `total > 0`，第 `page_count(total)` 页（即最后一页）一定能取到非空数据；
   `total` 不能被 `page_size` 整除时，零头单独占一页。
2. **页数自洽**：前 `page_count(total) - 1` 页都为满页（`page_size` 条），最后一页为 `last_page_size(total)` 条。
3. **整除一致**：当 `total` 恰好是 `page_size` 的整数倍时，最后一页也是满页。

## 错误现象

- 当总条数恰好是每页条数的整数倍时，分页一切正常。
- 但只要总条数除不尽（末尾有零头），前端拿到的总页数就**比实际少一页**，最后那批零头数据怎么都翻不到。
