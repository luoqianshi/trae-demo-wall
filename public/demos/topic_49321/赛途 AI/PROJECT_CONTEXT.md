# 赛途 AI 项目上下文文档

## 项目名称
赛途 AI

## 所属赛道
学习工作

## 一句话介绍
面向大学生的智能竞赛规划与组队角色匹配助手，从找比赛到组队准备，一页生成参赛方案。

## 目标用户
- 第一次参加竞赛、想找方向的大学生
- 想提升作品集、找实习方向的在校学生
- 需要找互补队友的竞赛参与者
- 对创新创业、数学建模、开发设计等竞赛感兴趣的本科生和研究生

## 核心痛点
1. **信息分散**：比赛通知散落在公众号、学校群、官网和经验帖里，筛选成本高，容易错过报名时间
2. **适合度不清**：不知道自己的专业、技能和时间更适合哪类竞赛
3. **角色不清**：组队时只看熟人关系，缺少对产品、技术、运营、答辩、文档等角色的能力匹配
4. **准备路径不清**：报名材料、时间安排、技能补齐和作品推进没有清晰节奏

---

## Demo 必须完成的 P0 功能
1. **能力画像表单**：支持选择年级、专业、技能、每周投入时间、参赛目标和组队情况
2. **竞赛方向推荐**：根据专业动态生成 3 个推荐比赛，展示匹配度、类型、理由、推荐角色和准备建议
3. **参赛规划展示**：包含推荐方向、角色建议、能力缺口、7 天准备计划和报名材料清单
4. **单页可交互**：点击按钮即可生成模拟推荐结果，无需后端支持
5. **响应式布局**：适配桌面端和移动端展示

## P1 增强功能
1. 更多专业方向的推荐规则扩展
2. 技能标签多选和更细粒度的画像维度
3. 队伍角色匹配模拟（根据已有队友推荐互补角色）
4. 竞赛日历和报名时间提醒展示
5. 可分享的个人参赛方案卡片

## P2 暂不开发功能
1. 用户注册登录系统
2. 真实竞赛数据库接入
3. 队友匹配和社交功能
4. 作品提交和评审系统
5. 多学校/多机构后台管理
6. 付费功能和商业化模块

---

## 技术边界
- **前端技术**：纯 HTML + CSS + 原生 JavaScript，单文件实现
- **禁止引入**：Vue、React 等前端框架，后端服务，数据库，外部 API
- **数据来源**：使用内置模拟数据，不采集真实用户隐私
- **部署方式**：单 HTML 文件即可运行，无需构建工具
- **浏览器兼容**：现代主流浏览器（Chrome、Safari、Firefox、Edge）

---

## TRAE 开发规则
1. **不删除或重写当前 HTML**：在现有 saitu-ai.html 基础上迭代
2. **纯前端实现**：不引入任何后端、数据库、API
3. **不新建复杂项目**：保持单文件或极简结构
4. **可演示优先**：黑客松初赛 Demo 以可交互、可体验、可讲述为核心目标
5. **每次开发记录**：使用 TRAE_DEV_LOG.md 记录每次开发的任务、修改、验收结果
