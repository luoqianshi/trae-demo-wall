一、高严重度问题（High）
H1. clearCloudToken 函数未定义 — 运行时错误
行号：3864-3869（调用处 3866）
类别：错误处理缺失 / 功能 bug
描述：cloudApi._handleAuthExpired 调用了 clearCloudToken()，但整个文件只定义了 logout()（3752-3755）和 setCloudToken/getCloudToken，从未定义 clearCloudToken。当云端返回 401 且 data.code === 'TOKEN_EXPIRED' 时，会抛出 ReferenceError: clearCloudToken is not defined，导致登录过期处理失败，用户卡在已过期会话中。
修复建议：将 clearCloudToken() 改为 logout()，或新增 const clearCloudToken = logout;。
H2. 云服务器使用 HTTP 明文传输 — 安全漏洞
行号：3743
类别：安全 / 敏感信息泄露
描述：const CLOUD_SERVER = 'http://8.137.196.81:3000'; 使用 HTTP 协议且为裸 IP。登录（3782-3799）、注册（3764-3781）的密码、所有对话同步的 Bearer Token、对话内容均以明文传输，易被中间人攻击窃取。同时硬编码 IP 不利于维护。
修复建议：改为 HTTPS 域名，强制服务端启用 TLS；密码应使用 bcrypt 等单向哈希后传输（即便 HTTPS 也建议）。
H3. API Key 通过公共文本数据库分发且明文缓存 — 安全漏洞
行号：7287-7298、9850
类别：安全 / 敏感信息泄露
描述：fetchKey 从 https://api.textdb.online/maoziaikey 和 maoziapikey 拉取 DeepSeek / 阿里云 API Key，并明文写入 localStorage（hatter_ds_apikey、hatter_aliyun_apikey、dreamhat_aliyun_apikey）。任何人访问该 URL 都能拿到 Key，且 localStorage 可被任意同源脚本读取，存在严重 Key 泄露与盗刷风险。
修复建议：改为后端代理调用，Key 仅保留在服务端；前端不接触原始 Key。
H4. currentSessionId 空指针解引用
行号：5618
类别：空指针 / 错误处理
描述：saveCurrentSession 中 if (isPrivateMode || currentSessionId.startsWith('private_')) return;，但 currentSessionId 初始值为 null（4021）。在 loadApiKeys 完成前若任何路径触发 saveCurrentSession，会抛 Cannot read properties of null (reading 'startsWith')。
修复建议：加 if (!currentSessionId) return; 守卫。
H5. 全局 Token 计数存在竞态与丢更新
行号：4588-4595
类别：竞态条件
描述：pushGlobalTokens 先本地 globalTotalTokens += addAmount，再 fetch 写回 textdb.online。这是典型的 read-modify-write 竞态：多端并发会互相覆盖。且 fetch(url) 无响应处理、无错误重试，写入失败仅 console.warn。
修复建议：改为原子递增接口（如 POST /increment?key=...&by=N）；或用后端计数。
H6. 登录角色动画的 mousemove/touchmove 监听永不移除 — 性能/内存泄漏
行号：8799-8810
类别：事件监听泄漏 / 性能
描述：initLoginCharacters 在 document 上注册了 mousemove 和 touchmove 监听且从不 removeEventListener。即使登录弹窗关闭，每次鼠标移动都会执行 updatePupils（含 4 次 getBoundingClientRect，触发强制布局）。在低端设备上会显著拖慢主线程。
修复建议：弹窗关闭时移除监听；或仅在 cloudLoginOverlay 打开时注册，关闭时移除。
H7. syncFromCloud 双向同步存在数据丢失竞态
行号：3897-3940
类别：竞态条件 / 数据持久化
描述：流程为「上传全部本地 → 拉取全部云端 → 用云端覆盖本地」。若两台设备同时同步，A 设备上传后 B 设备上传会覆盖 A 的版本，A 拉取时拿到 B 的旧数据再覆盖本地，导致 A 的新会话丢失。无锁、无版本号、无冲突解决。
修复建议：引入服务端版本号 / 时间戳冲突合并；或限制单设备在线。
H8. sendMessage 中 isProcessing 检查与设置之间存在窗口
行号：6730-6835
类别：竞态条件
描述：if (isProcessing) return; 在 6730 行，但 isProcessing = true 在 6835 行。中间有大量异步等待（如 showTruncateChoice 6786、checkContentViolation 6858 之前的同步流程）。用户在等待截断滑块期间点击建议 chip（5540-5544 直接调 sendMessage）可绕过检查，触发并发发送。
修复建议：在函数入口立即 isProcessing = true，所有 early return 路径重置为 false。
H9. 私密模式下仍向 AI 上报违规检测
行号：6858、7163-7201
类别：隐私 / 安全
描述：checkContentViolation(finalUserMessage) 在 sendMessage 中无条件调用（6858），将私密模式下的用户消息发送到 DeepSeek 做违规判断。私密模式承诺"对话不会被保存"，但内容仍外发至第三方。
修复建议：if (isPrivateMode) return; 跳过违规检测，或在隐私模式下仅本地规则匹配。
H10. printWin 可能被浏览器拦截导致后续 .document.write 抛错
行号：6480-6483
类别：错误处理缺失 / 空指针
描述：const printWin = window.open('', '_blank'); 在弹窗被拦截时返回 null，随后 printWin.document.write(...) 抛 Cannot read properties of null。Agent 流程会进入 catch 但用户无任何提示。
修复建议：if (!printWin) { showToast('弹窗被拦截，请允许弹窗后重试'); return; }。
二、中严重度问题（Medium）
M1. syncToCloud 在 saveCurrentSession 中未 await
行号：5625
类别：异步逻辑
描述：saveCurrentSession 是同步函数，末尾调用 syncToCloud()（async）但不 await。云端保存为 fire-and-forget，若用户快速连续发消息，云端写入顺序不可控，可能覆盖最新数据。
修复建议：将 saveCurrentSession 改为 async 并 await；或用队列串行化云端写入。
M2. loadApiKeys().then(...) 中 createNewSession 可能覆盖已有会话
行号：6661-6674
类别：状态管理 / 竞态
描述：初始化流程 loadApiKeys().then(() => { loadSessions(); createNewSession(); ... })。createNewSession 会生成新 currentSessionId 并清空历史。若用户在 loadSessions 后、createNewSession 前点击侧边栏某会话，loadSession 设置 currentSessionId，随后 createNewSession 又覆盖之，导致用户选中的会话被丢弃。
修复建议：初始化完成前禁用所有交互；或 createNewSession 仅在无当前会话时执行。
M3. welcomeRAF 永不取消 — 持续 RAF 循环
行号：9599-9639、9659
类别：性能 / 内存泄漏
描述：startWelcomeTypewriter 启动的 requestAnimationFrame(welcomeAnimate) 循环一旦开始永不停止，即使欢迎页已隐藏（dom.welcomeGuide.style.display = 'none'）。每帧都在操作 DOM。
修复建议：在 updateWelcomeGuide 隐藏欢迎页时 cancelAnimationFrame(welcomeRAF)；显示时再启动。
M4. 多个 setInterval/setTimeout 未在卸载时清理
行号：4514（agentElapsedTimer）、7885（speedUpdateTimer）、7850（diCollapseTimer）、4131（toast._timeout）、4830（stopBtnTimer）
类别：内存泄漏 / 定时器堆积
描述：除绘画模块在 beforeunload 清理（9918）外，主流程的定时器均未在 beforeunload 清理。speedUpdateTimer 在 stopSpeedDisplay 中清理，但若页面在流式中途关闭则泄漏。
修复建议：统一在 beforeunload 清理所有定时器。
M5. document.addEventListener('click', ...) 多次注册未去重
行号：7758、7811、7968、8694
类别：事件监听堆积
描述：四处独立的 document.addEventListener('click', ...) 用于关闭不同下拉菜单。虽不会重复触发（每处逻辑独立），但每次页面初始化都新增 4 个全局 click 监听，且无法移除。
修复建议：合并为单一 document click 处理器，统一调度。

M7. extraHtml 中图片 src 未严格校验

M8. appendMessage 中 agentBadgeHtml 直接拼入 innerHTML
行号：5206、5212、5214、5216
类别：XSS
描述：agentBadgeHtml 包含 getAgentIcon(info.icon, info.color) 的 SVG 字符串，info 来自硬编码 AGENT_INFO，目前安全。但 timeHtml 中 timeStr 来自 new Date(timestamp).toLocaleString(...)，理论上安全。整体依赖数据源可信，缺乏纵深防御。
修复建议：所有动态拼接 innerHTML 处统一过 DOMPurify.sanitize。
M9. callDeepSeekAI 中 messages 拼接遗漏 agent 消息内容
行号：6655-6661
类别：逻辑问题
描述：messages = [...conversationHistory.filter(m => m.role !== 'system' && !m.agent)]，过滤掉所有 agent 消息，然后单独把 agent 消息摘要塞进一个 system 消息（6660）。但 conversationHistory 中 user 消息可能包含 [补充信息]（6257）等 agent 流程产生的用户消息，这些被保留并发送，可能导致上下文混乱。
修复建议：明确区分 agent 流程产生的消息与正常用户消息。
M10. retryLastMessage 中 agent 路径未重置 isProcessing
行号：6996-6999
类别：状态不一致
描述：if (agentModeEnabled) { await runAgentPipeline(...); return; } 直接 return，未进入下面的 isProcessing = true 分支。runAgentPipeline 内部会设置 isProcessing，但若 runAgentPipeline 因 if (isProcessing) return;（6172）提前返回，则 isProcessing 永远不会被设置，按钮状态可能错乱。
修复建议：检查 runAgentPipeline 的 early return 条件，确保状态一致。

M13. extractMemoryEntries 中 replace_id 与 replace_ids 顺序处理可能导致重复
行号：7351-7355
类别：逻辑问题
描述：先按 replace_id 删除一条，再按 replace_ids 删除多条。若 AI 同时返回两者且指向同一 id，第二次 filter 不会报错但会做无用功。更严重的是，删除后未检查 duplicate（7356），若新条目与已存在条目重复，会被跳过，但旧条目已被删除，导致信息丢失。
修复建议：先收集所有待删除 id，统一过滤，再判重。
M14. maybeExtractMemory 是 fire-and-forget

类别：竞态条件

四、其他观察
O1. 代码组织
单文件 9933 行，含 HTML、CSS、JS。维护困难。建议拆分为 index.html、styles.css、app.js、agent.js、cloud.js、draw.js 等模块。
O2. 无单元测试
复杂逻辑如 tryParsePlan、tryParseClarify、estimateTokens、generateMathQuestion 等无测试覆盖。
O3. 无构建工具
直接 <script src="cdn"> 加载所有依赖，无打包、无 tree-shaking、无版本锁定（除 marked 等指定版本）。
O4. 可访问性（a11y）不足
多处 <button> 无 aria-label（如 5428 的 action-capsule）。
<div class="session-item" ...> 用 div 模拟按钮，无 role="button"、tabindex。
颜色对比度未校验（如 --text-tertiary: #aeaeb2 在白底上对比度低）。
O5. CSP 缺失
无 Content-Security-Policy 头，允许任意内联脚本和外部资源加载。
O6. 重试与超时策略不统一
有的 fetch 用 AbortSignal.timeout，有的用 AbortController + setTimeout，有的无超时。
