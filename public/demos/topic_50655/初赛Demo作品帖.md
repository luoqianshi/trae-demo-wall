## 1. Demo 简介

**是什么：**
AI 砍价助手是一个基于网页的智能砍价工具（H5 应用），用户输入商品信息和商家报价后，AI 自动分析底价区间、生成个性化砍价策略和话术，并接入大语言模型进行多轮自然对话，帮助用户在购物中省下开支。支持 12 个国产大模型可选、数据可视化洞察、砍价战绩海报分享等功能。

**面向谁：**
- 网购重度用户（拼多多、闲鱼等平台消费者）
- 线下购物不好意思砍价的"社恐"人群
- 想要精明消费的年轻上班族和家庭采购者
- 闲鱼二手交易买卖双方

**主要功能（五大模块）：**

### 功能一：🔍 智能砍价分析
用户输入商品名称、类别（8 大类）、商家报价、购买渠道（线上/线下/二手），AI 自动计算：
- 预估底价区间与建议目标价
- 砍价成功率预估
- 预计可省金额
- 13 种砍价策略推荐（换季清仓、比价、挑毛病、速决等）
- 三阶段砍价话术生成（开场/进攻/收尾），可一键复制
- **砍价反馈闭环**：输入实际成交价，AI 对比预测价计算准确度

![砍价分析界面](截图位置1)

### 功能二：💬 多轮对话式砍价模拟（接入大模型）
模拟和商家对话的真实场景，升级为 **LLM 驱动 + 状态机辅助** 的混合架构：
- **接入 12 个国产大模型**：DeepSeek-V3/R1、通义千问 Qwen2.5/QwQ、智谱 GLM-4/GLM-Z1 等，按品牌分组可选
- **对话阶段追踪**：初步接触 → 价格交锋 → 拉锯谈判 → 接近成交，实时显示当前阶段
- **情绪感知**：识别商家态度（强硬/松动/友好/拖延/威胁离开/让步/成交），调整话术语气
- **上下文记忆**：记住对话历史和价格变化轨迹
- **动态话术生成**：LLM 基于阶段+情绪+价格差距生成针对性应对策略
- **规则引擎兜底**：网络异常或未配置 Key 时，自动回退到本地规则引擎，保证可用性
- 支持 8+ 种商家回复场景识别

![对话模拟界面](截图位置2)

### 功能三：📊 数据洞察（数据可视化）
纯 Canvas 绘制的三大数据图表（无外部依赖）：
- **月度省钱趋势**：柱状图展示每月省钱金额变化
- **各类别砍价战绩**：环形饼图展示各品类省钱占比
- **砍价成功率统计**：柱状图展示成功/部分成功/失败比例
- 高分屏 DPR 适配 + 淡入动画 + 窗口自适应

![数据洞察界面](截图位置3)

### 功能四：📋 省钱记录管理
- 自动记录每次砍价的商品、原价、AI 预测价、实际成交价、省下金额
- 每条记录显示 AI 准确度标签（神预测/精准/尚可/偏差较大）
- 顶部导航栏统计：累计省钱、砍价次数、AI 平均准确度
- **算法自优化**：基于历史反馈数据，自动微调折扣率（同类别+渠道≥3条数据时启动）

![省钱记录界面](截图位置4)

### 功能五：🎨 战绩海报与社交分享
- **砍价战绩海报**：Canvas 自动生成精美战绩海报（2× DPR 高清输出，商品名/原价/成交价/省下金额/成功率/准确度）
- **下载分享**：海报可下载为 PNG 图片（1200×1600 高分辨率，高分屏不糊）
- **一键分享**：生成分享文案并复制到剪贴板
- **帮朋友砍价**：输入朋友的商品信息，帮朋友分析砍价

![战绩海报界面](截图位置5)

## 2. Demo 创作思路

**灵感来源：**
每次网购看到"先涨价再打折"的套路就很头疼，线下买菜又不好意思开口砍价。尤其是闲鱼买二手商品时，完全不知道该怎么出价才合理。想有个"高情商砍价搭子"帮我出招，不用自己费脑子。

**想解决的问题：**
- 不知道商品真实底价，砍少了亏、砍多了尴尬
- 面对面砍价经验不足，怕被拒绝没面子
- 网购想讨价还价却没有合适的渠道和话术
- 砍价浪费时间，影响购物体验

**为什么做这个方向：**
砍价是日常生活中高频出现但大多数人缺乏技巧的场景。市面上的比价工具只告诉你"哪里便宜"，却不教你"怎么砍下来"。AI 砍价助手填补了这个空白——不仅告诉你底价在哪，还手把手教你怎么说、怎么应对，把"砍价"这件事从"靠经验"变成"靠数据+话术+大模型"。

**迭代升级思路：**
初版只有基础的砍价分析和简单对话，在优化阶段加入了 6 大核心升级：
1. 对话引擎从正则匹配升级为多轮状态机，实现真正的智能对话
2. 接入硅基流动 LLM 接口，对话从规则模板升级为自然语言生成
3. 扩充到 12 个国产大模型可选（DeepSeek / Qwen / GLM 三大品牌）
4. 增加砍价反馈闭环，让 AI 能"学习"并优化预测准确度
5. 增加数据可视化，让省钱成果一目了然
6. 增加社交分享，让好工具能传播给更多人

### 4.3 开发心得

1. **AI 辅助开发效率极高**：从需求规划到完整 Demo 落地，TRAE 帮我大幅缩短了开发周期。尤其是砍价算法引擎和话术模板库的设计，AI 给出了很多我没想到的策略维度。

2. **对话式开发体验好**：不需要从零手写每一行代码，把需求描述清楚，AI 就能生成结构完整、逻辑清晰的代码，我再在此基础上微调优化。

3. **迭代式优化效果显著**：初版功能比较基础，通过 10 轮迭代升级（对话状态机 → LLM 接入 → 模型扩充 → 反馈闭环 → 数据可视化 → 海报高清 → UI 美化 → 脱敏交付），产品完整度和竞争力大幅提升。

4. **踩坑与解决**：
   - 初版砍价算法过于简单（固定折扣率），后来加入了"渠道系数"维度和"算法自优化"机制
   - 对话模拟器最初靠正则匹配，场景覆盖有限，升级为 LLM + 状态机混合架构
   - 遇到 JavaScript 暂时性死区（TDZ）崩溃：`const` 变量在声明前被调用，通过把全局配置移到初始化函数之前解决
   - 价格提取正则误匹配（"iPhone 13" 提取出 13 而非 2500），改用 4 层优先级匹配（价格关键词 > 数字+单位 > 多数字取最大 > 兜底）
   - 成交状态误判（用户出价被识别为成交），改为只在用户明确说"成交"时才进入成功状态
   - 海报在 Retina 屏模糊，通过 2× DPR 缩放解决
   - 数据可视化最初想用 Chart.js CDN，但考虑到离线可用性，改为纯 Canvas 实现
