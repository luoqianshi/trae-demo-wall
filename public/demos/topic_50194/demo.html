是什么 / 面向谁 / 核心功能

💻

产品形态

02 · Demo 创作思路

灵感 / 痛点 / 取舍

💡

灵感来源

受 macOS Launchpad、Windows PowerToys Run 与游戏环形菜单（如 Dota 2 物品栏）启发 —— 「按住 → 滑动 → 松开」的三段式肌肉记忆操作，比「点击图标」更快。

😖

真实痛点

① 桌面图标越来越多，找应用靠眼睛扫；② Alt+Tab 只显示标题，切窗口像开盲盒；③ 开始菜单搜索延迟高、动辄跳转 Bing。键盘用户的「启动焦虑」真实存在。

🧭

方向取舍

选择「角度盲选」而非「列表搜索」：8 个扇区对应 8 个应用，位置固定形成肌肉记忆，实现「不看屏幕也能启动」。原生 Rust 后端保证热键响应 < 16ms。

04 · TRAE 实践过程

用 TRAE 完成开发的完整流程

1

项目脚手架与架构设计

在 TRAE 中描述需求，生成 Tauri 2 + React + TS 项目结构。通过对话明确「多面板窗口 + 系统托盘 + 全局热键」的分层架构：commands / services / models / utils 四层 Rust 模块。

2

Win32 全局热键与鼠标角度追踪

通过 TRAE 编写 WH_KEYBOARD_LL 低级键盘钩子，捕获全局热键；用 GetCursorPos 获取鼠标位置，前端 useMouseAngle hook 计算角度与扇区索引，实现盲选交互。

3

多窗口 WebView2 数据目录隔离

遇到 WebView2「resource in use」错误，TRAE 协助定位到多窗口共享数据目录问题，通过为每个窗口分配独立 data_directory 解决。

4

IME 快捷键冲突排查与迁移

中文 Windows 下 Ctrl+Space 被 IME 拦截，热键无响应。TRAE 通过添加诊断日志定位到「钩子未收到 keydown」，自动迁移到 Alt+Space 并在 StorageService 加入版本迁移逻辑。

5

Apple Design 风格 UI 重构

提供设计参考图，TRAE 生成浅色主题 CSS 变量、玻璃态面板组件、SVG 环形高亮，统一应用到 4 个面板与设置页。

6

窗口显示时序优化

setup 钩子同步创建 5 个 WebView2 窗口导致启动卡顿，TRAE 建议改用 run_on_main_thread 后台线程逐个调度，事件循环立即启动。

05 · 开发心得

经验总结

🔧

Win32 与 WebView2 的边界

热键、窗口管理走 Win32 原生（Rust），UI 渲染走 WebView2（React）。两者通过 Tauri IPC 桥接，性能与开发效率兼得。

🐛

诊断日志驱动调试

IME 拦截、窗口时序等问题靠「日志先行」定位 —— 先加 info 日志确认事件链路，再动手修复，比盲猜快 10 倍。

🎨

设计系统的统一性

4 个面板 + 设置页共享一套 CSS 变量与组件库（GlassPanel / AppIcon），保证视觉一致性的同时降低维护成本。
