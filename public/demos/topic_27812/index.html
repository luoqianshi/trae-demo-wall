第一步 · 让 TRAE 知道你在做什么

贴 URL 抽取上下文

TRAE 不知道你是谁、不知道你在做什么。最快的方式是「把活动 URL 贴进对话」，TRAE 会自动抽取活动规则、评审维度、赛道要求作为上下文。

01

复制活动 URL

打开 TRAE AI 创造力大赛，选「学习工作赛道」分类页，复制浏览器地址栏。

https://forum.trae.cn/c/38-category/38
02

贴进 TRAE 对话

在 TRAE IDE 对话框写：

我要参加这个 TRAE 活动：<URL>

TRAE 会自动 fetch 页面内容。

03

确认上下文生效

TRAE 会输出「我理解的活动规则」总结，检查是否包含：

  • 4 评审维度（创新/完成度/体验/文档）
  • 赛道定位与时间窗口
  • 参赛作品提交要求
进阶用法 · 技能 + URL 组合

/brainstorming + 活动 URL 一起发：TRAE 启动 brainstorming 时已经掌握活动规则 + 评审 4 维度，澄清问题会直接对齐这些维度。

GanttLens 案例：第一天 brainstorming 时贴了 GanttLens 论坛 URL，TRAE 立刻按"评审 4 维度"反向印证 spec。

贴 URL 跟加载技能是两件事。贴 URL 解决「TRAE 知道你要做什么」，加载技能解决「TRAE 知道怎么做」。下面两步分别讲。

第二步 · 让 TRAE 知道怎么做

superpowers 4 件套

superpowers 是 TRAE IDE 的核心技能套件，覆盖从创意到交付全流程。下方 4 个技能是必装，每个都配 GanttLens 真实使用案例。

SKILL · 01

brainstorming

触发：从 0 到 1 想清楚要做什么

模糊想法 → 澄清问题 → 完整 spec。skill 会一次问 1 个问题，避免信息过载。

→ spec 文档 (docs/superpowers/specs/...md)
GanttLens 案例

Day 1 用 brainstorming 收需求：项目形态 / 目标用户 / 7 天计划 / 脱敏基线，最终产出 8 节 spec。

SKILL · 02

writing-plans

触发：spec 写完后怎么执行

完整 spec → 拆任务 → N 天可执行计划。每步带「写代码 → 测试 → commit」三段。

→ plan 文档 (docs/superpowers/plans/...md)
GanttLens 案例

Day 1 末尾调 writing-plans：把 spec 拆成 7 天可勾选任务，每步含 typecheck + Playwright 验证。

SKILL · 03

test-driven-development

触发：开始写代码时

先写失败测试 → 实现最小代码 → 重构。保证每个函数都有对应测试。

→ 测试代码 + 实施代码
GanttLens 案例

shiftMilestone 联动算法用 TDD：先写 3 个测试 case（顺延/逆推/边界），再实现，零调试。

SKILL · 04

verification-before-completion

触发：宣布"完成"前

交付前自查：跑测试 / typecheck / 浏览器实测 / 截图。证据先于断言。

→ 自查 checklist + 截图证据
GanttLens 案例

每天结束前必跑：npm run typecheck (0 错) + Playwright 截图 (无控制台错)，不通过不算完成。

第三步 · 让产品长得专业

frontend-design 3 要素

再好的功能，用户第一眼丑了也不会用。frontend-design 技能负责"美"的部分。掌握 3 个要素就够用：色彩 / 字体 / 间距。

字体

1 标题 + 1 正文 + 1 等宽

Bricolage + Inter + JetBrains Mono

03

间距

8px 基准网格

所有间距 = 8 / 16 / 24 / 32 / 48

5 个反模式避免

× 圆角泛滥 (border-radius: 999px 滥用)
× 字体混搭 (3 个以上字体家族)
× 间距乱 (7px / 13px / 19px)
× 颜色超标 (主色 + 强调色 + 5 个语义色)
× 阴影糊脸 (多层 blur shadow 堆叠)
