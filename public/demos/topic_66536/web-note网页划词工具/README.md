# 网页划词批注工具

一个基于Chrome Extension的浏览器插件，支持在网页上划词批注，数据存储到Gitee仓库。

## 功能特性

- ✏️ **划词批注**：在任意网页上选择文本进行批注
- 🎨 **自定义颜色**：6种预设颜色可选，默认粉色
- 📱 **侧边栏界面**：右侧侧边栏操作，界面简洁
- 💾 **Gitee存储**：数据同步到Gitee仓库，无服务器依赖
- 🔄 **自动同步**：批注数据自动保存到Gitee
- 🔍 **批注管理**：查看、编辑、删除页面上的所有批注

## 安装步骤

### 1. 生成图标文件

**方法一：使用HTML生成器（推荐）**
1. 在浏览器中打开 `icon-generator.html` 文件
2. 点击对应的下载按钮下载图标：
   - 点击"下载 16x16"按钮，保存为 `icon16.png`
   - 点击"下载 48x48"按钮，保存为 `icon48.png`
   - 点击"下载 128x128"按钮，保存为 `icon128.png`
3. 将下载的文件放到项目根目录

**方法二：使用在线工具**
1. 访问 https://realfavicongenerator.net/ 或 https://favicon.io/
2. 上传 `icon.svg` 文件或创建新图标
3. 生成并下载以下尺寸的PNG图标：
   - `icon16.png` (16x16)
   - `icon48.png` (48x48)
   - `icon128.png` (128x128)
4. 将生成的PNG文件放到项目根目录

### 2. 加载插件到浏览器

#### Chrome浏览器：
1. 打开 `chrome://extensions/`
2. 开启"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择项目文件夹 `d:\goproject\src\web-note`

#### Edge浏览器：
1. 打开 `edge://extensions/`
2. 开启"开发人员模式"
3. 点击"加载解压缩的扩展"
4. 选择项目文件夹 `d:\goproject\src\web-note`

### 3. 配置Gitee

插件使用 Gitee 仓库作为数据存储后端，需要完成以下配置：

#### 3.1 在 Gitee 上新建仓库

1. 登录 [Gitee](https://gitee.com)
2. 点击右上角 **"+"** → **"新建仓库"**
3. 填写仓库信息：
   - **仓库名称**：随意取名，例如 `web-note-data`
   - **是否开源**：建议选择 **"私有"**（批注数据属于个人隐私）
   - **初始化仓库**：勾选 **"使用 Readme 文件初始化这个仓库"**（确保仓库有 master 分支）
4. 点击 **"创建"** 完成
5. 在仓库中**新建文件** `web-note-annotations.json`：
   - 进入刚创建的仓库页面
   - 点击 **"新建文件"**
   - 文件名输入：`web-note-annotations.json`
   - 文件内容输入：`{}`
   - 点击 **"提交"** 保存

> ⚠️ **重要**：必须勾选"使用 Readme 文件初始化"，否则仓库没有 master 分支，插件无法写入数据。

#### 3.2 生成 Personal Access Token

1. 访问 [Gitee 私人令牌设置页](https://gitee.com/personal_access_tokens)
2. 点击 **"生成新令牌"**
3. 填写令牌描述，例如 `web-note插件`
4. 权限勾选：**只需勾选 `projects`**（仓库相关权限即可）
5. 点击 **"提交"**，复制生成的 Token

> ⚠️ Token 只会显示一次，请妥善保存。如果丢失需要重新生成。

#### 3.3 在插件中填写配置

1. 插件安装后会自动打开配置页面（也可以右键插件图标 → 选项）
2. 填写以下信息：
   - **Personal Access Token**：粘贴上一步生成的 Token
   - **用户名/组织名**：您的 Gitee 用户名（即 `gitee.com/你的用户名` 中的用户名）
   - **仓库名称**：第一步创建的仓库名，例如 `web-note-data`
3. 点击 **"测试连接"** 验证配置是否正确
4. 点击 **"保存配置"** 完成设置

## 使用方法

### 划词批注流程：

1. **划词选择**：在网页上选择要批注的文本
2. **显示图标**：鼠标附近会显示插件图标（✏️）
3. **打开侧边栏**：点击图标打开右侧侧边栏
4. **选择颜色**：点击颜色方块选择划词颜色
5. **输入批注**：在文本框中输入批注内容
6. **提交批注**：按Enter提交，Shift+Enter换行
7. **自动收起**：提交后侧边栏自动关闭，划词颜色保留

### 查看和管理批注：

1. **悬浮显示**：鼠标悬浮在已划词内容上，显示插件图标
2. **点击图标**：打开侧边栏，显示对应批注内容
3. **查看所有批注**：侧边栏底部显示页面所有批注列表
4. **编辑批注**：点击列表中的批注可进行编辑
5. **删除批注**：点击"删除批注"按钮删除当前批注

### 快捷键：

- `Enter`：提交批注
- `Shift+Enter`：批注内容中换行

## 数据存储

- 批注数据以JSON格式存储在Gitee仓库的 `web-note-annotations.json` 文件中
- 每个URL对应一组批注数据
- 数据结构：
```json
{
  "https://example.com": {
    "1234567890": {
      "id": "1234567890",
      "text": "选中的文本内容",
      "content": "批注内容",
      "color": "#ff69b4",
      "url": "https://example.com",
      "createdAt": "2024-01-01T00:00:00.000Z",
      "updatedAt": "2024-01-01T00:00:00.000Z"
    }
  }
}
```

## 技术栈

- **Chrome Extension Manifest V3**
- **原生JavaScript**（无框架依赖）
- **Gitee REST API**
- **HTML/CSS**

## 兼容性

- ✅ Chrome 88+
- ✅ Edge 88+
- ❌ Firefox（Side Panel API支持有限）

## 注意事项

1. **Gitee Token安全**：请妥善保管您的Personal Access Token
2. **网络连接**：需要网络连接才能同步数据到Gitee
3. **页面动态内容**：页面动态加载的内容可能需要刷新才能看到批注
4. **隐私保护**：批注数据存储在您自己的Gitee仓库中

## 故障排除

### 插件无法加载
- 检查图标文件是否完整
- 确认manifest.json格式正确
- 查看浏览器控制台错误信息

### 无法保存批注
- 检查Gitee配置是否正确
- 确认Token有repo权限
- 检查网络连接

### 批注不显示
- 刷新页面重新加载批注
- 检查Gitee仓库中是否有数据
- 查看浏览器控制台错误信息
