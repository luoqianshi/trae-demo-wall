# ValueLedger MVP Demo 开发方案

## 一、技术栈约束（严格遵守）

**全部使用 Python 标准库，不安装任何第三方包**：
- 服务端：`http.server`（HTTPServer + BaseHTTPRequestHandler）
- 数据库：`sqlite3`（Python 内置，零配置）
- HTTP 客户端：`urllib.request`（调用 DeepSeek API）
- 客户端界面：**命令行 CLI**（MVP 阶段不做 GUI，最简单直接）
- 网络：TCP/HTTP，支持局域网访问
- 其他：`json`、`hashlib`、`uuid`、`datetime`、`mac` 地址获取用标准库实现

在ValueLedger里，Session ID就是用户登录后返回的**Token（令牌）**，可以在两个地方找到：

### 1. 客户端本地（当前登录用户的Session）
登录成功后Token会保存在你电脑用户目录下的配置文件里：
```
C:\Users\蘑菇\.valueledger_client.json
```
打开这个JSON文件，里面`token`字段的值就是当前Session ID。

### 2. 服务端数据库（所有登录Session）
所有活跃的Session都保存在服务端SQLite数据库中：
```
e:\kaifeidemo\doubao2.1\valueledger\data\valueledger.db
```
可以在终端运行下面命令查看所有Session记录：
```powershell
cd e:\kaifeidemo\doubao2.1
python -c "import sqlite3; conn = sqlite3.connect('valueledger/data/valueledger.db'); conn.row_factory = sqlite3.Row; cur = conn.cursor(); cur.execute('SELECT s.id, s.token, u.username, s.created_at FROM sessions s JOIN users u ON s.user_id = u.id'); [print(dict(r)) for r in cur.fetchall()]"
```

每条记录里的`token`字段就是Session ID，`id`是数据库内部自增主键，真正用来鉴权的是`token`字符串。

### 查找方法：
1. 看TRAE界面**左侧边栏**，有一个时钟/历史图标（聊天历史列表），点击就能看到你所有和TRAE的对话会话
