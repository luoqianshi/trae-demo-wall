WEB SCOUT
让网页资源获取，像浏览一样简单
01产品简介 · 它是什么？

一款轻量级、跨平台的桌面应用。在普通浏览器的基础上，额外提供一套"抓取模式"，让用户可以可视化地获取网页中的图片、音视频、文档和文本内容。

它不复杂：默认就是浏览器，浏览网页毫无区别。

它很聪明：一键切换抓取模式后，自动识别页面中的图片(JPG/PNG/GIF)、音频(MP3)、视频(MP4)、文档(PDF)和可见文本，鼠标悬停即显示资源类型，点击即可查看详情或批量下载。

它面向未来：自动记录每次抓取操作的时间、URL、资源数量与耗时，形成结构化的任务流数据，为接入 AI 辅助抓取做好准备。

跨平台 (Windows / Linux) 轻量级应用 零代码操作 AI 就绪
02核心功能 · 它能做什么？
① 双模式无缝切换
默认"浏览模式"就是一个简洁的网页浏览器；点击顶部开关进入"抓取模式"，激活资源提取能力。
② 资源自动识别
扫描页面中的图片、视频、音频、PDF 文档和可见文本，自动识别格式并分类展示。
③ 可视化点击抓取
在抓取模式下，鼠标悬停即高亮资源，点击即可查看资源详情、一键下载或批量提取。
④ 批量下载与导出
支持同类资源批量下载到本地；资源清单可导出为 JSON / CSV / Markdown 多种格式。
⑤ 任务流自动记录
以日期为单位，记录每次抓取的时间戳、目标 URL、资源数量与耗时，形成可分析的任务时间线。
⑥ 插件式扩展接口
预留 Plugin API，后续可接入 AI 辅助抓取、OCR 图片文字识别、智能内容分类等功能。
03使用场景 · 什么时候用它？

浏览模式 · 日常使用

  • 日常阅读网页、浏览新闻
  • 访问图片网站、浏览视频
  • 查找文档资料、阅读在线 PDF
  • 无需任何抓取操作时

抓取模式 · 工作采集

  • 需要批量保存页面中的图片时
  • 需要下载视频/音频资料时
  • 需要提取并导出文章文本内容时
  • 需要收集 PDF 文档并整理时
  • 需要记录抓取操作，供 AI 分析调度时

07技术方案 · 它如何实现？
应用形态桌面客户端应用，独立安装运行
运行平台Windows 10+ / Linux (Ubuntu 等主流发行版)
核心框架Electron 28 · 内嵌 Chromium 浏览器内核
前端技术React 18 + TypeScript · 响应式界面
资源扫描基于 Content Script 注入，识别 img / video / audio / a[href$=.pdf] / 可见文本节点
下载模块Electron 原生下载 API · 支持并发、暂停、进度显示
数据导出JSON / CSV / Markdown / TXT 四种格式
任务流存储本地结构化存储，按日期组织，支持导出
扩展机制预留 Plugin API，支持第三方插件与 AI 接入
