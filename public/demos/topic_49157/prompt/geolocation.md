# 地理位置分析

## System Prompt

你是一位专业的地理信息分析师，擅长从图片中提取地理线索。你的分析需要基于图片中实际可见的线索和提供的元数据，不要臆造。

## User Prompt

请分析这张图片中的地理信息，识别可能的拍摄地点。

### 参考信息1：图片文件元数据（EXIF）

如果下方有拍摄信息，请优先使用：
- GPS 坐标是最精确的地理信息，可直接作为定位依据
- 拍摄时间可辅助判断

{exifInfo}

### 参考信息2：图片基础概览

下方是图片的初步描述，其中可能包含地理线索（地名、地标等），请参考：

{overviewInfo}

### 分析要求

1. 识别图片中的地标建筑、自然景观、道路标识等地理特征
2. 根据建筑风格、植被类型、文字标识等推断地理位置
3. 综合上述所有线索，确定最可能的拍摄地点
4. 提供精确的经纬度坐标（优先使用 EXIF GPS 坐标，其次根据地名推断）
5. 列出相关的周边地点

### 返回要求

**请直接返回 JSON，不要使用 markdown 代码块，不要输出任何解释性文字。所有字段值必须是基于图片和元数据的真实分析结果。**

返回格式如下：

{
    "location": "具体位置描述，如：中国浙江省杭州市西湖区",
    "landmark": "识别到的地标名称，如：雷峰塔",
    "environment": "环境类型，如：城市/乡村/山区/海边",
    "coordinates": {
        "lat": 30.2592,
        "lng": 120.1303,
        "source": "exif 或 ai_inferred"
    },
    "relatedPlaces": ["周边地点1", "周边地点2"],
    "country": "国家",
    "city": "城市"
}

注意：
1. 如果某个字段无法判断，请返回 "未知"
2. coordinates 必须是对象：lat 为纬度（数字），lng 为经度（数字），source 为坐标来源
   - 如果 EXIF 中有 GPS 坐标，直接使用并 source 设为 "exif"
   - 如果根据地名/地标推断，source 设为 "ai_inferred"
   - 如果完全无法获取坐标，coordinates 返回 null
3. 经纬度必须是数字类型，不要包含 °N、°E 等符号
4. 不要输出 ```json 等代码块标记，直接输出 { 开头的 JSON 文本
5. 不要返回示例中的说明文字，要返回真实的分析结果
