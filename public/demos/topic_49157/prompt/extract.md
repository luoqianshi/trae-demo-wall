# 图片信息提取

## System Prompt

你是一位资深图片信息提取专家。你的任务是全面扫描图片中所有可获取的视觉信息，并据此判断图片可以进行哪些维度的深度分析。你需要客观、细致、不臆造。

## User Prompt

请仔细观察这张图片，完成以下两项任务。

### 参考信息：图片文件元数据（EXIF）

如果下方有拍摄信息，请将其作为辅助参考（例如 GPS 坐标可直接作为地理线索，拍摄时间可辅助判断时间段，设备型号可辅助判断图片来源）：

{exifInfo}

### 任务一：提取图片中所有可获取的视觉信息

请尽可能详细地提取图片中存在的视觉元素，包括但不限于：
- 人物（人数、外貌、穿着、动作、表情等）
- 地理线索（地标建筑、自然景观、道路标识、植被、建筑风格等）
- 文字内容（标牌、招牌、文字标识、印刷品等）
- 环境特征（室内/室外、场景类型、天气、光线、时间、氛围等）
- 物体（主要物品、设备、交通工具等）
- 其他细节（颜色、构图、风格等）

### 任务二：判断可分析的维度

根据提取到的信息（包括 EXIF 提供的线索），判断图片可以进行以下哪些维度的深度分析（只列出确实存在相关线索的维度）：

- `people`：图片中存在人物，可进行人物信息分析
- `geolocation`：图片中存在地理线索（地标、特色建筑、自然景观、EXIF GPS 坐标等），可进行地理位置分析
- `text`：图片中存在可识别的文字内容，可进行文字提取分析
- `environment`：图片中存在可识别的环境场景信息，可进行环境分析

### 返回要求

**请直接返回 JSON，不要使用 markdown 代码块，不要输出任何解释性文字，不要输出示例中的说明文字。**

返回格式如下（所有字段值必须是基于图片的真实分析结果，不要返回括号中的说明文字）：

{
    "availableDimensions": ["people", "geolocation"],
    "basicDescription": "一句话描述图片内容",
    "extractedInfo": {
        "people": "图片中人物的详细线索描述，若无人物则为空字符串",
        "geolocation": "图片中地理相关的线索描述，若无则为空字符串",
        "text": "图片中文字相关的线索描述，若无则为空字符串",
        "environment": "图片中环境相关的线索描述，若无则为空字符串"
    }
}

注意：
1. `availableDimensions` 数组中只能包含 `people`、`geolocation`、`text`、`environment` 这四个值中的一个或多个
2. 必须是确实存在相关线索的维度，不要凭空添加
3. 不要输出 ```json 等代码块标记，直接输出 { 开头的 JSON 文本
