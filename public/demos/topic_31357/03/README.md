# PRD智能生成助手 - Edge浏览器插件

> 通过监听网页交互行为，AI智能解析页面功能，自动生成标准化产品PRD文档

## 简介

PRD智能生成助手是一款运行于Edge浏览器的轻量插件，通过监听、捕捉用户当前网页的交互行为和页面DOM信息，结合AI大模型智能解析当前Web页面的核心功能、业务逻辑、操作流程，自动生成标准化产品PRD文档，支持Markdown格式导出，同时提供历史文件的本地查询、预览和下载功能。

## 核心功能

### 六大功能模块

| 模块 | 说明 |
|------|------|
| 页面行为监听 | 自动采集点击、输入、下拉、弹窗、跳转等交互行为，实时监听DOM变化，支持14种元素类型采集 |
| Web功能智能识别 | 基于采集数据调用AI大模型，自动归纳页面功能、操作流程、业务规则 |
| 操作手册生成 | AI生成面向终端用户的操作手册，包含功能导航、操作步骤、常见问题 |
| PRD自动生成 | 按标准模板生成8大模块的完整PRD文档 |
| 交互记录导出 | 将采集的交互行为导出为结构化Markdown报告 |
| 历史文件管理 | IndexedDB本地持久化存储，支持预览、下载、删除 |

### 生成的PRD文档结构

每份PRD文档包含以下8个标准模块：

1. **功能概述** — 当前页面/模块的业务用途、使用场景、核心价值
2. **页面信息** — 页面URL、名称、类型（列表页/详情页/编辑页等）
3. **核心功能列表** — 逐条罗列所有识别到的功能点
4. **操作流程说明** — 用户操作步骤、功能触发逻辑、前置条件、后置结果
5. **页面元素说明** — 按钮、表单、弹窗、筛选项的功能解释
6. **业务规则** — 表单校验、状态限制、权限逻辑、特殊场景规则
7. **异常场景** — 报错提示、空数据场景、操作失败场景说明
8. **功能优先级** — 默认普通优先级

## 安装方法

### 前置要求

- Edge浏览器（Chromium内核，版本88+）
- AI大模型API Key（支持OpenAI兼容接口）

### 安装步骤

1. 下载或克隆本项目到本地

```bash
git clone https://github.com/HwangJL/lin-Automation-PRD.git
```

2. 打开Edge浏览器，地址栏输入 `edge://extensions/`

3. 开启右上角 **"开发人员模式"**

4. 点击 **"加载解压缩的扩展"**，选择项目根目录

5. 插件安装完成，浏览器工具栏出现插件图标

## 操作手册

### 第一步：配置API Key

1. 点击浏览器工具栏的插件图标，打开弹窗
2. 切换到 **"API设置"** 标签页
3. 填写以下配置信息：

| 配置项 | 说明 | 是否必填 |
|--------|------|----------|
| API Key | 大模型的访问密钥 | 必填 |
| API接口地址 | 自定义接口地址，默认为OpenAI地址 | 选填 |
| 模型名称 | 如 `gpt-4o-mini`，默认 `gpt-4o-mini` | 选填 |

4. 点击 **"保存配置"** 保存
5. 点击 **"测试连接"** 验证API Key是否有效，状态提示"连接成功"即配置完成

> 支持所有OpenAI兼容接口（如DeepSeek、通义千问、智谱等），只需修改API接口地址和模型名称即可。

### 第二步：启动监听

1. 打开需要分析的Web页面
2. 点击插件图标，在 **"功能操作"** 标签页
3. 点击 **"启动监听"** 按钮，状态变为"监听中"
4. 正常操作页面（点击按钮、填写表单、打开弹窗、切换页面等）
5. 插件自动采集交互行为，实时显示交互记录数和页面元素数

### 第三步：导出交互记录（可选）

1. 操作完成后，点击 **"导出交互记录"** 按钮
2. 自动下载Markdown格式的交互记录文件
3. 文件包含：页面信息、页面元素表格、交互行为记录表格
4. 同时自动保存到历史记录

### 第四步：生成PRD文档

1. 操作完成后，点击 **"生成PRD文档"** 按钮
2. 插件调用AI接口分析页面数据，生成完整PRD文档
3. 生成完成后，弹窗内直接预览文档内容
4. 点击 **"导出MD文件"** 下载Markdown文件到本地
5. 文档自动保存到历史记录

### 第五步：管理历史记录

1. 切换到 **"历史记录"** 标签页
2. 所有已生成的PRD文档和交互记录按时间倒序排列
3. 每条记录显示类型标签（蓝色"PRD文档" / 黄色"交互记录"）
4. 点击记录可在线预览完整内容
5. 支持以下操作：
   - **重新下载** — 再次导出MD文件
   - **删除记录** — 删除单条历史记录
   - **清空全部** — 一键清空所有历史记录

## 安全说明

- 所有数据**纯本地存储**（IndexedDB + chrome.storage.local），不上传云端
- 仅读取当前页面公开DOM信息，不获取用户隐私账号、密码等敏感数据
- API Key加密存储在浏览器本地，仅本地可见
- 无后台接口请求、无数据泄露风险
- 关闭浏览器、重启电脑后，历史记录和配置永久保留

## 适用范围

- Edge浏览器内核网页
- 普通H5网页
- 后台管理系统网页
- 前端交互类Web页面

> 不适用于：加密防DOM读取页面、完全静态无交互纯展示页面

## 常见问题

**Q：启动监听后提示"页面未加载插件，请刷新页面后重试"？**
A：请刷新当前网页后重试，content script需要在页面加载后才能注入。

**Q：生成PRD时提示"后台服务响应超时"？**
A：插件已优化为popup直连模式，如仍出现超时，请检查网络连接和API接口地址是否正确。

**Q：测试连接失败？**
A：请检查：1) API Key是否正确；2) API接口地址是否正确；3) 网络是否能访问对应接口；4) 模型名称是否匹配。

**Q：未配置API Key时可以使用哪些功能？**
A：未配置API Key时，仅可查看、预览、下载历史记录，无法启动监听和生成PRD文档。

**Q：支持哪些浏览器？**
A：目前仅支持Edge浏览器（Chromium内核），理论上也可在Chrome中使用，但未做专门适配。
