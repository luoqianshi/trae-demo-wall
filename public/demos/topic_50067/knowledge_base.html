EmoMind 解决通用大模型对中文短文本情绪识别不准的问题。系统包含Web前端、FastAPI后端、LoRA微调引擎、131,693条中文知识库和DeepSeek API兜底策略，支持私有化部署，适用于内容平台舆情监控、心理咨询辅助评估、教育机构学生状态监测、企业客服情绪感知等场景。

🎯 目标用户

内容平台（舆情监控）、心理咨询师（辅助情绪评估）、教育机构（学生心理监测）、企业客服（客户情绪感知）、个人用户（情绪日记/自我认知）

⚡ 核心功能

情绪分类机器人（6维分类）· LoRA高效微调（0.29%参数）· 13万条三风格知识库 · 混合预测策略（模型+关键词+API）· 实时训练监控 · 自学习闭环

🔧 技术栈

HTML/CSS/JS前端 · FastAPI后端 · Transformers + PEFT/LoRA · SQLite知识库 · bert-base-chinese基础模型 · DeepSeek API兜底

📈 价值与意义

效率提升：相比人工标注提升数百倍，实时分析替代批量处理。社会价值：辅助心理健康早期预警，帮助平台建设更健康社区氛围，降低客服情绪误判率。

🔴 当前痛点

1

通用大模型情绪识别不准

尤其对中文短文本，中文特有的表达方式和文化语境导致频繁误判

2

缺乏中文语境优化的轻量级方案

现有方案体积大不适合边缘部署，且未针对中文语境专门优化

3

私有化部署成本高

多数方案依赖云端API，企业私有化面临高昂算力和数据安全风险

4

无法自定义情绪分类维度

标准方案仅提供正/负/中三分类，无法根据业务灵活定义情绪维度

🛡️ 三级兜底策略

第一级
BERT + LoRA 模型推理
优先使用本地微调模型，毫秒级响应，处理绝大多数输入
第二级
关键词智能匹配
模型置信度低时，通过关键词规则引擎快速匹配
第三级
DeepSeek API 兜底
前两级均无法判断时，调用DeepSeek大模型并自动回写知识库
