万能调试日志器

零依赖 | 一行启动 | 线程安全 | 复制即用

让 Python 调试告别 print() 散落四处的痛苦——
为学习与工作场景打造的开箱即用型全量日志方案

from debug_logger import start

开发者调试的「痛点」vs「解法」

每个 Python 学习者、嵌入式工程师、算法研究员都会遇到这些场景

❌ 常见痛点

  • print() 散落代码各处，上线前逐个删除
  • 终端输出一闪而过，出错后找不到历史
  • 多线程程序输出交织，完全无法阅读
  • 程序崩溃后不知道最后执行到哪一行
  • 手动加时间戳、文件名，重复造轮子
  • 引入 logging 模块配置繁琐、学习成本高

✅ 万能调试日志器

  • 一行 start() 自动捕获所有终端输出
  • 毫秒级时间戳 + 文件名:行号:函数名 全记录
  • 线程安全，多线程输出有序可追溯
  • 未捕获异常自动记录完整堆栈
  • 每行日志自动附加调用位置，零侵入
  • 纯标准库，0 依赖，复制文件即可用

3 种使用方式，总有一款适合你

从最简侵入到全功能定制，灵活适配各类场景

方式一：最小侵入（推荐）— 放在程序开头即可 
from debug_logger import start
start()

# 你原有的代码，完全不动
print("程序启动")
print("计算结果:", 42)
# 所有输出自动写入日志文件！
方式二：自定义参数 
from debug_logger import start
start(
    log_dir="./my_logs",
    max_size_mb=20,
    show_caller=True,
    enable_runtime_stats=True
)
方式三：上下文管理器 + 手动日志 
from debug_logger import DebugLogger, log

with DebugLogger(log_dir="./mylogs"):
    log.info("这是一条信息")
    log.warning("这是一条警告")
    log.error("这是一条错误")
    # 退出 with 块自动停止日志
