## Session 1：需求拆解与产品方案
请你作为产品经理，基于下面创意生成初赛 Demo PRD：

产品名：知行板 LearnFlow
赛道：学习工作
目标用户：备考、自学、项目推进、职场汇报人群
核心痛点：目标太大、资料太散、缺少每日行动计划、缺少复盘调整
Demo 目标：做一个可交互单页网页，用户输入目标、截止日期、投入时间、当前基础和卡点后，生成阶段计划、今日任务、风险提醒和复盘建议。

请输出：1）用户场景；2）核心功能；3）页面结构；4）交互流程；5）初赛评审亮点。

## Session 2：生成可运行 HTML Demo
请根据上一个 PRD，生成一个可直接运行的单页 HTML Demo。要求：
1. 只有一个 index.html 文件，CSS 和 JS 内联；
2. 中文界面，适合 TRAE AI 创造力大赛学习工作赛道；
3. 包含目标拆解器、任务看板、复盘助手和投稿摘要 4 个模块；
4. 点击按钮后必须有真实交互结果，不能只是静态页面；
5. UI 要清晰、美观、移动端可用；
6. 不依赖外部资源，方便 zip 上传社区。

---

## Session 3：按评审标准优化
请按初赛评审维度优化这个 Demo：创新性 30%、实用性 30%、完成度 20%、美观度/设计体验 20%。
请重点检查：
1. 是否有真实学习/工作场景；
2. 核心链路是否跑通；
3. 是否能让评审 1 分钟内理解价值；
4. 页面是否有清晰视觉层级；
5. 交互是否有反馈、默认示例和可复盘闭环。
请给出修改后的代码或明确的修改清单。
