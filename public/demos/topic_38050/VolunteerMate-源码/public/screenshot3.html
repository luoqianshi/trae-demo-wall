用模板字符串拼接的 Tailwind 类名在 dev 能工作，build 就丢了，怎么修复？

AI
TRAE · 晚上 7:15 · 思考 3 秒

根因Tailwind 的 JIT 编译器是"静态扫描"源码的 —— 它只认得完整出现的 class 名。`bg-${color}-500` 这种模板字符串，编译器只看到字面量 bg-${color}-500，不知道运行时 color 是什么，所以根本不会生成对应 CSS。

修复方案 建立"任务分类 → 完整 class 名"的映射表，把所有可能用到的 class 名都以完整字符串写在代码里，JIT 就能正确扫描到：

TypeScriptsrc/components/TaskCard.tsx · 修复
// ❌ 旧写法（build 丢样式） className={`bg-${color}-500`} // 模板字符串，JIT 扫不到 // ✅ 新写法（映射表 + 完整 class 名） const categoryStyles: Record<string, { bg: string; icon: string; text: string; }> = { '环保': { bg: 'bg-emerald-50', icon: 'bg-gradient-to-br from-emerald-400 to-emerald-600', text: 'text-emerald-700', }, '捐赠': { bg: 'bg-pink-50', icon: 'bg-gradient-to-br from-pink-400 to-pink-600', text: 'text-pink-700', }, '帮扶': { bg: 'bg-purple-50', icon: 'bg-gradient-to-br from-purple-400 to-purple-600', text: 'text-purple-700', }, '传播': { bg: 'bg-blue-50', icon: 'bg-gradient-to-br from-blue-400 to-blue-600', text: 'text-blue-700', }, }; // 使用：运行时查表取完整字符串 const style = categoryStyles[task.category]; <div className={`w-12 h-12 rounded-xl ${style.icon} ...`}> {task.icon} </div>
💡 为什么能修好？
现在 JIT 能在源码里看到完整的 bg-gradient-to-br from-emerald-400 to-emerald-600 等 class 字符串，会为它们生成 CSS。运行时只是查表，不再依赖模板拼接。这是 Tailwind 官方推荐的"避免动态 class"的做法。

✅ 这个方案同样适用于其他地方的动态配色（CheckInModal、CelebrationModal 等）。要我帮你一次性把项目中所有类似的写法都修复吗？

问题 file:// 协议下，Chrome 对 localStorage 有"同源策略严格限制"，会静默失败。需要做"双轨持久化"：

TypeScriptsrc/store/useStore.ts · 修复
import { create } from 'zustand'; // 安全写入：try/catch 包裹，file:// 下降级为内存存储 function safeLoad<T>(key: string, fallback: T): T { try { const raw = localStorage.getItem(key); return raw ? JSON.parse(raw) : fallback; } catch (e) { // file:// 协议下 localStorage 被禁用 → 使用内存回退 console.warn('[store] localStorage unavailable, using memory'); return fallback; } } function safeSave(key: string, value: any) { try { localStorage.setItem(key, JSON.stringify(value)); } catch (e) { /* 静默失败，下次启动使用默认值 */ } } interface UserStore { stats: UserStats; checkIn: (task: Task) => void; reset: () => void; } export const useStore = create<UserStore>((set, get) => ({ stats: safeLoad('volunteer_stats', { totalMinutes: 0, totalCheckIns: 0, currentStreak: 0, environmentalTasks: 0, ... }), checkIn: (task) => { safeSave('volunteer_stats', { ...get().stats, totalCheckIns: get().stats.totalCheckIns + 1, totalMinutes: get().stats.totalMinutes + task.duration, }); }, reset: () => safeSave('volunteer_stats', {...INIT}), }));

✅ 修好后，两种打开方式都能工作：npm run dev 用 localStorage（刷新不丢数据），file:// 降级为内存模式（当前会话可用）。提交比赛用 dist/index.html 也能正常体验。
