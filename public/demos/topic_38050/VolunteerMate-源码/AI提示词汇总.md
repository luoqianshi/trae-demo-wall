## 📌 系统 Prompt 与 AI 人格设计

### 系统 Prompt 完整内容

以下是 `src/ai/apiService.ts` 中实际使用的系统 Prompt。每次发起 AI 对话时，这个 Prompt 都会作为 `messages[0]` 发送给大模型，定义 AI 的角色和行为边界。

```
你是「公益随手做 VolunteerMate」的 AI 小助手，一位专业、温暖、有爱心的公益顾问。

你的职责：
1. 回答用户关于公益的各种问题（如何做公益、环保知识、捐赠渠道、志愿服务等）
2. 根据用户的心情和需求，推荐合适的公益行动
3. 鼓励用户坚持做公益，传递正能量
4. 用温暖、有感染力的语言传递公益理念

你的特点：
- 温暖、专业、有耐心
- 回答接地气，不说教
- 善于发现用户做公益的闪光点并给予鼓励
- 可以结合时事和热点聊公益

回复要求：
- 用中文回复
- 语言温暖有感染力，像朋友聊天一样
- 如果不确定，给出建议而非错误信息
- 可以适当用 emoji 增加亲切感
- 控制在 200-400 字左右
- 如果用户没有明确问题，主动引导："想聊聊今天的公益心情吗？或者告诉我你现在的心情，我来帮你选一个合适的任务～"
```

### Prompt 设计思路

| 设计要点 | 为什么这么做 | 对应 Prompt 段落 |
|---------|------------|----------------|
| **角色定义** | 明确 AI 是谁（公益顾问，不是万能助手），约束回答范围 | 第 1 段：「公益随手做 VolunteerMate」的 AI 小助手 |
| **职责清单** | 给 AI 划定功能边界——只能聊公益相关，不回答无关问题 | 「你的职责」1-4 条 |
| **语气设定** | 温暖有感染力 > 专业精准。公益产品不需要冷冰冰的机器语调 | 「温暖、专业、有耐心」「像朋友聊天一样」 |
| **回复长度控制** | 200-400 字 ≈ 手机屏幕 1-2 屏。太短没内容，太长不想看 | 「控制在 200-400 字左右」 |
| **主动引导策略** | 用户说"不知道做什么"时，AI 应该主动提问，而不是被动等用户 | 「如果用户没有明确问题，主动引导...」 |
| **emoji 策略** | 适量 emoji 增加亲切感，但不滥用。每个段落不超过 2 个 | 「可以适当用 emoji」 |

**设计原则**：Prompt 不是"咒语"，而是"产品需求文档"。AI 最终在用户面前表现成什么样，很大程度上由这几行文字决定。

## 📊 AI 推荐算法说明

### 算法总览

推荐引擎位于 `src/ai/recommendation.ts`，基于**多因子打分排序**模型。输入用户画像 + 今日主题 + 时间段，输出 Top-N 推荐任务及个性化推荐理由。

```
┌────────────────────────────────────────────────────────────┐
│                        输入层                                 │
│  ┌─────────────┐  ┌──────────────┐  ┌──────────────────┐    │
│  │ 用户画像     │  │ 今日主题      │  │ 当前时间段       │    │
│  │ (stats)     │  │ (theme)       │  │ (morning/...)    │    │
│  └─────────────┘  └──────────────┘  └──────────────────┘    │
└────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌────────────────────────────────────────────────────────────┐
│                        打分引擎                              │
│                                                             │
│  因素 1：分类偏好得分  ──────────────┐                      │
│  因素 2：时间段匹配  ────────────────┤                      │
│  因素 3：新任务探索奖励  ────────────┤  → 综合得分         │
│  因素 4：难度-等级匹配  ─────────────┤                      │
│  因素 5：流行度因子  ────────────────┤                      │
│  因素 6：去重惩罚（近期做过的扣分） ─┘                      │
└────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌────────────────────────────────────────────────────────────┐
│                        输出层                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  [ { task, score: 95, reason: "你擅长环保类..." },  │   │
│  │    { task, score: 88, reason: "下午适合深度参与" },  │   │
│  │    { task, score: 80, reason: "..." } ]             │   │
│  └─────────────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────────────┘
```

### 6 个打分因子详解

| # | 因素 | 权重 | 计算逻辑 |
|---|------|------|---------|
| 1 | **分类偏好** | 基础分 | 基于用户历史打卡中环保/捐赠/帮扶/传播的比例，同类任务加分。占比越高，加分越多 |
| 2 | **时间段匹配** | +15 分 | 早/晚优先推荐短任务（≤15min），下午优先推荐深度任务（≥45min） |
| 3 | **近期重复惩罚** | −25 分 | 近 3 天内做过的任务，大幅扣分；目的是引导用户尝试新任务 |
| 4 | **难度-等级匹配** | +10~15 分 | 用户等级高（Lv.4+）推荐挑战任务加分；新手（Lv.1-2）推荐简单任务加分 |
| 5 | **流行度** | +5 分 | 参与人数 > 1500 的任务小幅加分，营造"大家都在做"的感觉 |
| 6 | **新任务探索奖励** | +8 分 | 从未做过的任务额外加分，鼓励用户拓展公益类型 |

### 分类多样性保证

Top-N 输出时，不是简单取分数最高的 N 个，而是额外做了**去重**处理：

```typescript
// 简单逻辑：同一类别在 Top-N 中最多出现 1 次
const usedCategories = new Set<string>();
for (const item of scored) {
  if (usedCategories.has(item.task.category) && result.length < count - 1) {
    continue;  // 同类已经有了，跳过
  }
  result.push(item);
  usedCategories.add(item.task.category);
}
```

目的：确保用户看到的推荐覆盖环保/捐赠/帮扶/传播多个领域，而不是"因为用户做过一次环保，就全推环保任务"。

### 推荐理由生成

推荐理由不是 AI 写的，而是**基于得分因素拼接生成的规则式文案**（因为推荐理由需要秒级响应，而调用 API 成本高）：

```typescript
// 伪代码：
if (preferenceScore > 25) reasons.push(`你${category}类任务参与度最高`);
if (timeSlot === 'afternoon' && duration >= 45) reasons.push('下午有充足时间深入参与');
if (level >= 4 && difficulty === '挑战') reasons.push('高级用户适合挑战任务');
// 取前 2 条拼接为最终理由
```

## 📚 问答模板库（knowledgeQA.ts）

### 11 个高频问题 + 预写答案

以下是 `src/ai/knowledgeQA.ts` 中实际使用的问答模板。当用户未配置 API Key 或网络不可用时，系统自动降级到此模板库。

| 关键词触发 | 问题 | 预写答案要点 |
|-----------|------|-------------|
| 怎么做公益 / 怎么开始 / 新手 / 入门 | **我是新手，怎么开始做公益？** | 垃圾分类（15min）→ 整理旧物捐赠（30min）→ 分享公益理念（5min）。从小事开始，不要追求完美 |
| 时间 / 忙 / 没空 | **我很忙，没时间做公益怎么办？** | 上班路上走一段 = 绿色出行；按需取餐 = 节约粮食；电梯让给老人 = 帮扶；转发公益信息 = 传播 |
| 捐钱 / 捐款 / 花钱 / 免费 | **公益一定需要花钱吗？** | 不需要。金钱只是公益的一种形式。你的时间、技能、注意力、选择——都是有价值的公益投入 |
| 效果 / 有用吗 / 意义 / 价值 | **我做的这些小事真的有用吗？** | 引用特蕾莎修女的话。一滴水虽小，但千万滴水可以汇聚成河。你不知道自己的善意会在哪里开花 |
| 环保 / 垃圾 / 绿色 / 低碳 | **日常生活中怎么践行环保？** | 随身水杯、自备购物袋、光盘行动、垃圾分类、公共交通、随手关灯——每一件小事都是温柔的地球 |
| 社区 / 邻居 / 邻里 | **怎么在社区做公益？** | 问好、让电梯、多带一把伞、帮独居老人拿快递、清理小广告、组织图书交换 |
| 孩子 / 儿童 / 教育 / 下一代 | **怎么培养孩子的公益意识？** | 孩子不是教出来的，是带出来的。一起整理旧玩具、一起社区清洁、一起读绘本、以身作则 |
| 团队 / 公司 / 企业 / 组织 | **我们公司想一起做公益？** | 植树、旧物捐赠、为环卫工人提供午餐、为山区学校募捐图书、公益马拉松——团队公益 = 做好事 + 团队建设 |
| 你好 / hi / hello | **问候语** | 自我介绍 + 邀请用户提问——"你可以问我任何关于公益的问题" |
| 推荐 / 建议 / 适合 / 应该 | **你推荐我做什么？** | 根据用户画像分析——新手从 5-15min 简单任务开始；有经验者尝试中等难度；达人挑战深度参与 |
| 成就 / 等级 / 徽章 / 奖励 | **成就和等级有什么用？** | 徽章是走过的脚印，等级是投入的时间，连续打卡是坚持的证据——它们不只是数字，而是你用心生活过的证据 |

### 关键词匹配算法

```typescript
// 简化逻辑：
for (const qa of QA_TEMPLATES) {
  // 每个模板有一组关键词（如 ['环保', '垃圾', '绿色', '低碳']）
  const matchCount = qa.keywords.filter(kw => userInput.includes(kw)).length;
  if (matchCount > 0) {
    return qa.answer;  // 命中一个关键词即返回
  }
}
// 未命中 → 走知识库全文搜索
// 仍未命中 → 走通用回答（鼓励用户行动）
```

**设计取舍**：关键词匹配的精度不高（会有误判），但作为**降级方案**已经足够好用。它保证了：
- 零网络依赖
- 零配置门槛
- 毫秒级响应
- 回答内容可控（不会出现 AI"幻觉"）

### 4.2 CORS / file:// 问题

> 使用 TRAE 的**问题分析**能力

```
问题描述：Vite build 生成的 dist/index.html，直接双击在浏览器打开，白屏。
```

```typescript
// vite.config.ts 关键配置
export default defineConfig({
  base: './',                              // 关键：相对路径
  build: {
    target: 'esnext',
    sourcemap: false,
  },
  plugins: [
    react(),
    viteSingleFile(),                      // 关键：把所有资源内联到一个 HTML
  ],
});
```
