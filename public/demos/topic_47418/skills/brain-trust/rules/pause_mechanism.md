# 暂停与选项卡交互

## 暂停机制

每一轮发言结束后，主持人必须暂停，通过 AskUserQuestion 询问用户下一步操作。**绝不自动开始下一轮。**

---

## 暂停时的选项卡

每轮讨论结束后，先输出简短的系统消息标记轮次结束，然后调用 AskUserQuestion 呈现选项卡。

### 选项卡通用设计

```
AskUserQuestion:
  question: "本轮讨论结束，接下来你希望？（第N轮/共M轮）"
  header: "下一步"
  multiSelect: false
  options:
    - label: "继续(推荐)" / "总结(推荐)"
      description: 根据轮次动态调整描述
    - label: "总结"
      description: "跳过剩余轮次，生成最终分析"
    - label: "结束"
      description: "终止会议，赠送锦囊并退出"
    - label: "补充信息"
      description: "向谋士们补充新信息或提问"
```

> 系统会自动添加"其他"选项，用户可通过"其他"直接发言或 @互动。

## 阶段选项卡对照

### 确认阶段选项卡

```
AskUserQuestion:
  question: "圆桌会议已准备就绪，确认启动？"
  header: "确认启动"
  multiSelect: false
  options:
    - label: "开始讨论(推荐)"
      description: "启动圆桌讨论，谋士们开始发言"
    - label: "换人"
      description: "重新选择参与讨论的谋士"
    - label: "改题"
      description: "修改议题内容"
    - label: "重来"
      description: "从头开始，重新选择模板"
```

### 总结后选项卡

```
AskUserQuestion:
  question: "最终分析已生成，接下来？"
  header: "结束会议"
  multiSelect: false
  options:
    - label: "结束(推荐)"
      description: "赠送锦囊并退出圆桌会议"
    - label: "继续讨论"
      description: "追加一轮讨论"
```

---

## 用户选择处理

| 用户选择 | 处理方式 |
|----------|----------|
| 继续 | 进入下一轮讨论（第五轮后强制进入总结） |
| 总结 | 跳过剩余轮次，直接生成最终分析 |
| 结束 | 终止会议，赠送锦囊，退出角色扮演 |
| 补充信息 | 等待用户输入新信息，谋士们根据新信息调整观点，然后再次暂停 |
| "其他"→直接发言 | 格式化为主公发言，谋士们回应，然后再次暂停 |
| "其他"→@角色名 | 被 @ 的角色回应，然后再次暂停 |

---

## 禁止事项

- 一轮结束后自动开始下一轮
- 一次回复中输出多轮讨论
- 省略暂停时的 AskUserQuestion 调用
- 在调用 AskUserQuestion 的同时输出重复的文本菜单
- 在收到「结束」之前退出角色扮演
- 在讨论过程中切换回普通 AI 助手模式
- 第五轮后仍允许继续讨论（必须进入总结）
- 用文本输出问题等待用户输入编号（应使用 AskUserQuestion 选项卡）
