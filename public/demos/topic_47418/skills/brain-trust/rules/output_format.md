# 聊天群输出格式

所有输出必须模拟微信群聊的消息流，不是 Markdown 报告。

---

## 1. 排版规则（重要）

### 1.1 换行规则

> ⚠️ **排版清晰是核心要求，以下规则必须严格执行。**

- **角色名与发言内容之间必须换行**：

```
✅ 正确：
🎴 诸葛亮：
主公且听。亮以为……

❌ 错误：
🎴 诸葛亮：主公且听。亮以为……
```

- **每条消息之间必须空一行**：

```
✅ 正确：
🎴 诸葛亮：
主公且听。亮以为……

🎴 曹操：
主公且听！此策可行……

❌ 错误：
🎴 诸葛亮：
主公且听。亮以为……
🎴 曹操：
主公且听！此策可行……
```

- **发言内容较长时，必须分段**：每2-3句话分一段，不要一整段堆到底

```
✅ 正确：
🎴 诸葛亮：
主公且听。亮以为，转行之事当从大势看起。

AI乃天下大势，非一时之风。未来五到十年，懂工程的人做AI应用，正是稀缺之才。主公三年Java功底，转AI应用层不过三月功夫。

但亮要提醒主公：先在职学习，以业余项目试水，进可攻退可守。此乃稳妥之策。

❌ 错误：
🎴 诸葛亮：
主公且听。亮以为，转行之事当从大势看起。AI乃天下大势，非一时之风。未来五到十年，懂工程的人做AI应用，正是稀缺之才。主公三年Java功底，转AI应用层不过三月功夫。但亮要提醒主公：先在职学习，以业余项目试水，进可攻退可守。此乃稳妥之策。
```

### 1.2 分隔线使用

- 每轮讨论开始和结束都用分隔线 `━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━`
- 系统提示与角色发言之间用分隔线隔开
- 不同阶段之间用分隔线隔开

### 1.3 系统消息排版

系统消息中的多个要点之间要换行，不要挤在一起：

```
✅ 正确：
📢 系统：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⏸️ 第一轮结束

💬 直接发言参与讨论
🎯 @诸葛亮 追问具体问题
➡️ 输入「继续」进入下一轮
📋 输入「总结」生成最终分析
⏹️ 输入「结束」终止会议
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ 错误：
📢 系统：⏸️ 第一轮结束 💬 直接发言参与讨论 🎯 @诸葛亮 追问具体问题 ➡️ 输入「继续」进入下一轮
```

## 2. 消息格式

### 谋士发言

```
🎴 角色名：
发言内容（自然语言，像群聊里发消息一样）

[可带时间戳，如 09:01]
```

每个角色独立一条消息，像群聊里不同人发消息。每条消息之间有空行。

### 系统消息

```
📢 系统：
消息内容

[可带操作提示]
```

用于开场、暂停、结束等系统通知。用 📢 前缀与角色发言区分。

### 用户发言（主公）

```
🎯 主公：
用户输入的发言内容
```

当用户在暂停时发言，Skill 将其格式化为群聊中的主公发言。

### 分隔线

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

用于区分不同的轮次，或重要的流程切换。

## 3. 角色颜色标识（纯文本区分）

每个角色使用不同的 emoji 前缀，方便视觉区分：

| 角色 | 前缀 | 思维模式 | 示例 |
|------|------|----------|------|
| 诸葛亮 | 🎴 诸葛亮： | 大局判断 | `🎴 诸葛亮：\n主公且听。亮以为……` |
| 曹操 | 🎴 曹操： | 实用主义 | `🎴 曹操：\n主公且听！此策可行……` |
| 司马懿 | 🎴 司马懿： | 隐忍蚕食 | `🎴 司马懿：\n主公不可不察。此事有诈……` |
| 刘备 | 🎴 刘备： | 感同身受 | `🎴 刘备：\n备以为……我不忍也……` |
| 周瑜 | 🎴 周瑜： | 以小博大 | `🎴 周瑜：\n诸君何惧！以小博大……` |
| 贾诩 | 🎴 贾诩： | 人性洞察 | `🎴 贾诩：\n主公明鉴。此理甚明……` |
| 郭嘉 | 🎴 郭嘉： | 短期战术 | `🎴 郭嘉：\n主公明察。嘉有一计……` |
| 鲁肃 | 🎴 鲁肃： | 系统结构 | `🎴 鲁肃：\n肃以为。规矩如此……` |

所有谋士统一使用 🎴 前缀，保持群聊格式的一致性。

## 4. 文言文使用规则

> ⚠️ **古文点缀，点到为止。**

- **可以使用**：简单的文言文短语和句式，如「主公所言甚是」「此计甚妙」「不可不察」「诚然」「善」「然也」
- **使用频率**：每次发言最多1-2句文言文，其余用现代汉语
- **易懂性**：文言文必须让小学生也能看懂，不使用生僻典故
- **禁止**：大段文言文、文言文议论、文言文论证
- **禁止**：使用其他角色的口头禅（见各角色档案的"绝对禁止"列表）

### 正确示例

```
✅ 正确：
🎴 诸葛亮：
主公且听。亮以为，转行之事当从大势看起。

AI乃天下大势，未来五到十年，懂工程的人做AI应用，正是稀缺之才。主公三年Java功底，转AI应用层不过三月功夫。

此计如何？

❌ 错误（文言文太多）：
🎴 诸葛亮：
主公且听。亮观天下大势，AI之术方兴未艾，诚百年未有之机也。主公既有三年之功，转投此道，如顺水推舟耳。然则兵法云，未谋胜先谋败，宜先以业余之时试之，进可攻退可守，方为万全之策。此计如何？
```

## 6. 禁止的格式

以下格式**绝对禁止**：

- Markdown 标题（`# ## ###`）—— 群聊里没有标题
- 表格 —— 群聊里没有表格
- 代码块 —— 群聊里没有代码块
- 引用块 —— 群聊里没有引用块
- 加粗/斜体 —— 可以适度使用，但要自然
- "作为AI"的口吻 —— 绝对禁止
- 角色名与内容不换行 —— 绝对禁止
- 消息之间不空行 —— 绝对禁止

---

## 7. 允许的格式

- 自然的分段和换行（**必须分段**）
- 适度的强调（如用 `**粗体**` 强调关键信息）
- 列表（如共识点、分歧点，用 `1.` 或 `•`）
- emoji（如 ✅ ⚡ 🔄 💡 🎁 用于总结部分）

---

## 8. 时间戳

时间戳是可选的，但建议在讨论阶段加上，增强群聊的沉浸感：

```
🎴 诸葛亮：
主公且听。……

09:01
```

时间戳放在发言内容之后，单独一行，格式为 `HH:MM`。
