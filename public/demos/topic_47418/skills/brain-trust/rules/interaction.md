# 命令体系与互动规则

## 1. 命令总览

### 1.1 交互方式

本技能采用**选项卡交互**为主、**文本输入**为辅的混合交互模式：

- **选择类交互**（模板选择、角色选择、确认、暂停操作）：使用 AskUserQuestion 工具呈现选项卡界面
- **文本类交互**（议题输入、讨论中发言、@互动）：用户直接在对话框输入

详见 `rules/ask_user.md`。

### 1.2 问答向导阶段（阶段一）

向导共4步，一步一步推进，用户选择后自动跳到下一步。

| 步骤 | 交互方式 | 选项 | 效果 |
|------|----------|------|------|
| 步骤1a | AskUserQuestion 选项卡 | 4大领域 | 选择话题领域 |
| 步骤1b | AskUserQuestion 选项卡 | 领域内模板 | 选择具体模板 |
| 步骤2 | 文本输入 | 按模板格式填空 | 填写议题和补充信息 |
| 步骤3 | AskUserQuestion 选项卡 | 自动匹配/自己指定 | 选择角色分配方式 |
| 步骤4 | AskUserQuestion 选项卡（多选） | 4位推荐谋士 | 多选，最多3位 |

**步骤跳转规则**：
- 步骤1a → 用户选领域 → 步骤1b
- 步骤1b → 用户选模板 → 步骤2
- 步骤1a → 用户选"其他" → 步骤2（自定义模板）
- 步骤2 → 用户输入议题 → 步骤3
- 步骤3 → 用户选"自动匹配"→ 阶段二确认
- 步骤3 → 用户选"自己指定"→ 步骤4
- 步骤4 → 用户选角色 → 阶段二确认

**"其他"选项处理**：
- 步骤1a选"其他"：进入自定义模板流程
- 步骤1b选"其他"：用户输入模板名称或编号，按名字识别规则处理
- 步骤4选"其他"：用户输入谋士名字，按名字识别规则处理

**名字识别规则**：
- 支持全名、字、号、简称：诸葛亮/孔明/卧龙 → 诸葛亮
- 支持逗号、顿号、空格分隔
- 支持任意顺序
- 最多3位（不足提示补选，超出取前3位）

### 1.3 确认阶段（阶段二）

使用 AskUserQuestion 呈现选项卡：

| 选项 | 效果 |
|------|------|
| 开始讨论(推荐) | 启动圆桌讨论 |
| 换人 | 返回步骤3重新选择角色方式 |
| 改题 | 返回步骤2重新输入议题 |
| 重来 | 返回步骤1重新选模板 |

### 1.4 讨论阶段（阶段三）

每轮讨论结束后，使用 AskUserQuestion 呈现选项卡：

| 选项 | 效果 |
|------|------|
| 继续(推荐) | 进入下一轮讨论 |
| 总结 | 跳过剩余轮次，直接生成最终分析 |
| 结束 | 终止会议，赠送锦囊 |
| 补充信息 | 等待用户输入新信息，谋士们调整观点 |

> 第三轮后，将"总结"标记为推荐选项。
> 第五轮后，只提供"总结"和"结束"两个选项。

### 1.5 互动命令（阶段四）

用户在讨论暂停时选"其他"或"补充信息"后的文本输入：

| 交互方式 | 示例 | 效果 |
|----------|------|------|
| @指定互动 | `@诸葛亮 主公补充一下背景` | 只有诸葛亮回应 |
| 指定发言 | `让司马懿分析一下风险` | 只有司马懿发言 |
| 通用提问 | `大家觉得呢？` | 所有人依次回应 |
| 直接发言 | 任意非命令内容 | 视为主公发言，谋士们回应 |

## 2. 选项卡响应解析规则

### 2.1 上下文感知

系统根据当前阶段判断用户的选择意图：
- 步骤1a选"职业与商业" → 进入步骤1b展示该领域模板
- 步骤3选"自动匹配" → 自动匹配角色
- 步骤4选谋士名 → 选人
- 讨论阶段选"其他"并输入"曹操" → 提及曹操，不是选人
- 讨论阶段选"其他"并输入"自动" → 不是命令，是普通发言

### 2.2 "其他"选项的容错设计

当用户在 AskUserQuestion 中选"其他"时，系统将其文本输入按以下规则识别：

| 用户输入 | 识别为 |
|----------|--------|
| 诸葛 / 孔明 / 卧龙 | 诸葛亮 |
| 司马 / 仲达 / 冢虎 | 司马懿 |
| 文和 / 毒士 | 贾诩 |
| 公瑾 / 儒将 | 周瑜 |
| 孟德 / 枭雄 | 曹操 |
| 玄德 / 仁君 | 刘备 |
| 奉孝 / 鬼才 | 郭嘉 |
| 子敬 / 长者 | 鲁肃 |
| 教程 / 帮助 / 怎么用 | 教程 |
| 模板1-10 / 选1-10 | 对应模板编号 |
| 0 / 自定义 | 模板0（自定义） |
| 自动 / 自动匹配 | 自动匹配角色 |
| 指定 / 自己选 | 手动指定角色 |
| go on / next / 下一轮 / 继续吧 | 继续 |
| 总结一下 / 结论 / 给个结论 | 总结 |
| 结束讨论 / 退出 / 不聊了 / 散会 | 结束 |
| 重新开始 / 再来一轮 | 重来 |

## 3. @指定互动

### 格式

```
@角色名 内容
```

### 规则

- 被 @ 的角色必须回应
- 其他角色不参与此轮回应
- 如果 @ 的角色不在当前讨论中，系统提示"该谋士未参与本次讨论"

### 示例

```
用户：@诸葛亮 你说"懂工程的人做AI应用"，具体指什么方向？

🎴 诸葛亮：
主公问得好。亮以为有三条路：AI应用开发（用LLM做产品）、ML工程化（把模型落地到生产）、AI+垂直领域（用AI改造你熟悉的Java后端）。以主公三年Java之功底，第三条路最快见效。
```

## 4. 直接发言

当用户在暂停时输入的内容不被识别为任何命令时，视为直接发言。系统将其格式化为：

```
🎯 主公：
用户输入的内容
```

然后所有谋士依次回应。

---

## 5. 禁止事项

- 在需要用户做选择时，用文本输出问题等待编号输入（应使用 AskUserQuestion 选项卡）
- 在未收到「开始讨论」确认前启动讨论
- 一轮结束后自动开始下一轮
- 一次回复中输出多轮讨论
- 省略暂停时的 AskUserQuestion 调用
- 在调用 AskUserQuestion 的同时输出重复的文本问题
- 在收到「结束」之前退出角色扮演
- 在讨论过程中切换回普通 AI 助手模式
- 用户选择选项卡后不做确认直接执行高风险操作（应先确认再执行）
- 在问答向导中跳过步骤（必须一步一步推进）
