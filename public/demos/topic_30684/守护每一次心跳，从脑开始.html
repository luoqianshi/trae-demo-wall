01 / 方案概述

为什么我们需要它

每一条生命都值得被温柔守护，尤其是那些从脑卒中中走出来的家人。

想象一下——家中老人刚刚经历了一次脑卒中，出院后回到熟悉的家中。医生说恢复期需要持续监测大脑供氧情况，但总不能天天跑医院。您白天要上班，晚上回到家，心里总有一根弦绷着：他今天脑部供氧还好吗？有没有悄无声息的缺血风险？

这就是 NeuroVitals 诞生的初衷。我们想做的，不是一台冷冰冰的医疗设备，而是一个像家人一样默默守护的伙伴——一条轻便的监测头带，戴上它，就能在家中持续感知大脑的氧合状态，把关键的脑氧数据及时传递给关心的人。

技术如何落地

本方案采用 NIRS（近红外光谱）技术，通过多波长近红外光穿透皮肤和颅骨，非侵入式测量脑组织中的氧合血红蛋白与脱氧血红蛋白浓度变化，从而计算出脑氧饱和度 rSO₂ 与 灌注指数。硬件端基于 ESP32-C3 主控搭配 NIRS 近红外传感器阵列（前额+太阳穴双位置），通过标准 BLE 协议与浏览器无线通信，Web 端以可视化方式呈现监测数据，并内置三级预警响应机制。系统同时支持真实硬件连接与纯模拟演示两种模式，确保在任何场景下均可完整展示。

温暖的设计理念

监测程序采用 Web Bluetooth API 而非传统 App，家人只需在浏览器中打开页面即可查看数据。不需要安装软件，不需要复杂设置——就像打开一个网页一样简单。同时，任何兼容 BLE 的设备都可以接入，数据可以共享给异地工作的子女，让关心跨越距离。

02 / 硬件方案

轻巧贴合的 NIRS 近红外监测头带

用一束近红外光，安静地感知大脑的每一次呼吸。

主控：ESP32-C3 SuperMini

芯片ESP32-C3 (RISC-V 32-bit, 160MHz)
无线Wi-Fi 4 + BLE 5.0
Flash / SRAM4MB / 400KB
尺寸22.5 x 18mm（拇指大小）
接口I2C x 1, GPIO x 11, USB-C
功耗深度睡眠模式 ＜ 5uA

NIRS 传感器阵列

NIRS 近红外模块760/810/850nm 多波长 x 3（前额+两侧太阳穴）

压力传感FSR 402 薄膜压力传感器

硬件接线方案（仅 4 线 I2C 总线）

// ESP32-C3 SuperMini <--> 传感器阵列 (共享 I2C)
GPIO6 (SDA) <---> NIRS(L) SDA, NIRS(R) SDA, NIRS(F) SDA, GY-906 SDA
GPIO7 (SCL) <---> NIRS(L) SCL, NIRS(R) SCL, NIRS(F) SCL, GY-906 SCL
3.3V <---> 所有传感器 VIN
GND <---> 所有传感器 GND
GPIO0 <---> FSR 402 压力传感器（ADC 读取）

// 各传感器 I2C 地址不同，可共享同一总线
// NIRS(L): 0x57, NIRS(R): AD0 拉高, NIRS(F): 0x58
// 150mAh 锂电池可持续运行约 4-6 小时

供电方案

3.7V 150mAh 聚合物锂电池，通过 TP4056 充电模块进行 USB-C 充电。 系统全速功耗约 30-50mA，续航 4-6 小时，满足日常监测需求。支持深度睡眠模式延长续航。

前额+太阳穴双位置佩戴方案

三枚 NIRS 近红外传感器模块分别固定于弹性头带前额位置（正中）和两侧太阳穴区域。 前额传感器覆盖前额叶皮层区域，两侧传感器覆盖颞叶区域，实现前额+太阳穴双位置脑氧监测。 3D 打印外壳收纳 ESP32 和电池，整体重量＜25g，佩戴无感，适合长时间连续监测。

03 / 固件架构

ESP32 固件设计

基于 Arduino 框架，使用标准 BLE 协议，实现 NIRS 多传感器数据融合与实时传输。

图 2: 固件架构 — 标准 BLE Brain Oxygen Service + 自定义 NIRS 多传感器扩展服务

开发框架

使用 Arduino IDE 或 PlatformIO 开发，依赖库：NIRS_Sensor_Driver（近红外传感器驱动）、 Adafruit_MLX90614（温度传感器）、ArduinoBLE（BLE 协议栈）。 开发环境和依赖库均为开源方案，便于社区协作与迭代。

数据处理流程

1. 三通道 NIRS 同步采样 (100Hz) → 2. 滑动窗口均值滤波去噪 → 3. 修正 Beer-Lambert 定律计算 rSO₂ → 4. 多波长差分算法提取灌注指数 → 5. 压力传感器 ADC 读取接触状态 → 6. 温度传感器 I2C 读取 → 7. 数据打包为 BLE 特征值 → 8. Notify 推送至浏览器端。

关键优势

使用标准 BLE 协议，任何支持 BLE 设备的 App（如 nRF Connect、LightBlue）均可直接连接验证数据真实性。同时扩展自定义服务传输 NIRS 多传感器原始数据，兼顾兼容性与功能性。设备名称 "NeuroVitals" 让家人一眼就能认出。

预警事件时间线，支持 CSV 格式导出原始数据，便于家人分享给医生进行后续分析。

05 / 预警响应

三级预警响应预案

当大脑发出求救信号，每一秒都至关重要。系统自动分级响应，守护黄金时间。

每天早晨，王阿姨戴上 NeuroVitals 监测头带，数据安静地传输到手机上。大多数时候，屏幕上显示的是绿色的"正常"——这是一天中最让人安心的时刻。但我们知道，对于脑卒中康复者来说，情况可能在几分钟内发生变化。这就是为什么我们设计了三级预警：像一位不知疲倦的守护者，时刻准备着。

I 级 · 正常守护

  • 持续常规监测，系统每 30 分钟自动生成脑氧简报
  • rSO₂ 与灌注指数均在正常范围内，波形稳定
  • 建议保持规律作息，适度活动，心情愉悦
  • 正常监测频率：1 次/秒

II 级 · 注意提醒

  • 系统自动推送通知至家人手机和主治医师终端
  • 建议佩戴者静卧休息，避免头部剧烈运动
  • 监测频率自动提升至高频采集模式
  • 若 30 分钟内未恢复至正常，自动升级至 III 级
  • 触发条件：rSO₂ 下降 > 10% 或 灌注指数 < 70% 或 左右梯度 > 15%

III 级 · 紧急响应

  • 立即触发多通道报警（APP推送 + 短信 + 家属端）
  • 自动拨打 120 并共享患者定位与实时脑氧数据
  • 头带终端持续记录异常时段完整波形，锁定关键证据
  • 建议保持平卧，头部抬高 30度，等待急救
  • 触发条件：rSO₂ 下降 > 20% 或 异常风险评分 ≥ 45%

监测参数说明

脑氧饱和度 rSO₂（0-100%）：反映脑组织氧合状态，正常 > 60%，下降 > 10% 需关注，下降 > 20% 为高危。
灌注指数（0-100%）：反映脑组织血流供给充足度，正常 > 85%，< 70% 表示灌注不足。
左右侧梯度（0-50%）：反映左右脑氧合差异，正常 < 10%，> 15% 表示单侧异常。
异常风险（0-100%）：综合评分，≥ 45% 触发高危预警。

06 / 双模式设计

真实硬件 + 模拟演示 双模式切换

确保在任何情况下都能完整展示系统功能，模拟模式内置可控异常触发。

真实模式 (REAL)

通过 Web Bluetooth API 连接 ESP32 头带终端。
NIRS 传感器阵列实时采集脑氧/压力/温度信号，
BLE 传输 rSO₂ + 灌注指数 + 原始波形，
3D 脑部影像实时渲染并执行预警逻辑。

模拟模式 (DEMO)

内置生理信号模拟器，
生成逼真的多参数脑氧监测数据。
可一键触发"异常事件"演示完整预警流程，
无需任何硬件即可展示全部功能。

模拟模式的价值

1. 无硬件环境下仍可完整演示系统功能，降低演示门槛；
2. 可精确控制异常事件的触发时机，配合讲述节奏展示关键功能；
3. 无需担心现场蓝牙干扰、设备电量等意外因素，确保演示稳定可靠。

08 / 体验优化

让每一天的佩戴都自然舒适

从可穿戴性、日常使用、数据安全及长期佩戴体验等维度，提出具体优化方向。

穿戴舒适性优化

1

医用级亲肤材料

头带内衬采用医用级硅胶或记忆海绵，接触皮肤区域通过 ISO 10993 生物相容性测试，避免长时间佩戴引起过敏或压痕。传感器接触面采用透光医用凝胶垫片，减少传感器与皮肤之间的空气间隙，提升 NIRS 信号质量。

2

无感佩戴设计

将电池与主控模块从头部移至颈后或衣领夹式设计，通过柔性 FPC 排线连接传感器，将头部负载降至 5g 以下。头带宽度控制在 15-20mm，采用透气网眼布料，减少闷热感。

3

自适应贴合结构

引入记忆金属骨架或弹性编织结构，自动适配不同头围（54-62cm），无需手动调节。传感器模块采用弹簧浮动结构，确保与皮肤保持恒定接触压力（约 10-15mmHg），避免过紧或过松。

日常使用场景适配

4

多场景模式切换

设置"睡眠监测"、"日常活动"、"静息检测"三种模式。睡眠模式下降低采样率至 10Hz 以延长续航；日常活动模式下启用运动伪影消除算法，确保数据可靠性。

5

防水防尘设计

外壳采用 IPX5 防水等级设计，支持日常洗漱、小雨天气佩戴。传感器窗口采用光学级防水透光膜，不影响 NIRS 信号穿透。充电接口采用磁吸触点，取消外露 USB 端口。

6

静默提醒交互

内置微型振动马达，异常预警时通过振动模式（短振/长振/连续振）区分预警等级，避免声音报警引起社交尴尬。日常可设置定时佩戴提醒，培养规律监测习惯。

数据安全与隐私保护

7

端到端加密传输

BLE 通信采用 AES-128 加密，配对时使用随机 PIN 码验证，防止中间人攻击。所有健康数据在传输层加密，即使蓝牙信号被截获也无法解密原始数据。

8

本地优先存储

敏感脑氧数据默认存储在设备端加密 Flash 中，仅当用户主动授权时才上传至云端。浏览器端使用 IndexedDB 加密存储历史数据，支持数据一键清除。

9

合规认证规划

产品设计预留医疗器械认证路径：CFDA II 类医疗器械注册、ISO 13485 质量管理体系、IEC 62304 医疗软件生命周期。数据存储符合《个人信息保护法》及 HIPAA 标准。

长期佩戴与续航优化

10

智能功耗管理

采用动态采样率策略：正常时 10Hz 低频采样，异常检测到波动时自动升至 100Hz，预警触发后全速 200Hz 记录。结合 ESP32 深度睡眠模式，续航可从 4-6 小时提升至 24-36 小时。

11

无线充电方案

下一代设计采用 Qi 无线充电线圈集成在头带后部，用户只需将头带放置在充电底座上即可充电，无需插拔线缆。充电底座同时作为数据同步中继站，通过 Wi-Fi 上传加密数据至医院系统。

12

皮肤友好与长期佩戴

传感器接触面采用可更换的医用级水凝胶贴片（单次使用 24-48 小时），避免长期重复使用同一位置造成皮肤刺激。头带本体支持 75% 酒精擦拭消毒，适合多用户共用场景。

优化优先级建议

第一优先级（演示阶段）：自适应贴合结构、硅胶内衬舒适度、IPX5 基础防水；
第二优先级（小批量试用）：智能功耗管理、静默振动提醒、加密传输；
第三优先级（量产阶段）：无线充电、多场景模式、CFDA 认证申请。
