💡 创意介绍

想解决什么问题？

当前产业工人（如养护工、桥梁工、隧道工等）面临安全生产考核与职业技能培训的双重压力，但市面缺乏一款针对该群体的专业学习工具。传统的纸质题库不易携带、无法自动判分、操作反馈滞后，学习效率低下。

为什么会想到做这个？

团队成员深入一线工厂调研后发现：许多工友在工间休息、通勤途中有强烈的碎片化学习需求，但苦于没有趁手的工具。市面上通用学习App要么内容不匹配，要么功能过于复杂。我们决定用Android原生 + 离线优先的技术方案，让学习真正触手可及。

产品形态

一款基于 Kotlin + Jetpack Compose + MVVM 架构的 Android 原生应用，支持完全离线使用，集题库管理、多模式练习、智能搜题、TTS语音朗读于一体。

🎯 目标用户及痛点

面向哪些用户？

核心用户群体为一线产业工人：包括养护工、桥梁工、隧道工、施工员、安全员等需要进行安全生产知识学习和职业技能考核的从业人员。同时也可覆盖企业内部培训部门和职业培训机构。

在什么场景下使用？

工间休息刷题巩固、通勤路上语音听题、上岗前安全知识自测、考前集中模拟冲刺——真正实现随时随地、想学就学。

当前痛点分析

📄

纸质题库不便

携带不便、无法自动判分，做完题不知道对错，学习反馈滞后

📶

网络环境受限

工地或车间信号差，在线学习App难以使用，急需离线方案

🔊

阅读困难

部分工人年龄偏大、视力下降，纯文字学习体验差，需要语音辅助

📊

错题管理混乱

手抄错题效率低，无法精准定位薄弱环节，重复犯错难以避免

⏱️

学习中断

工作随时被打断，做题进度无法保存，每次都要重新开始

🔍

搜题困难

遇到陌生题目只能翻书或问人，没有快速查找答案的方式

🛠️ 开发历程

v1.0项目立项

完成基础架构搭建：MVVM分层 + Room数据库 + 基础练习功能。使用传统Activity + ViewBinding构建UI。

v1.2.5功能完善

加入TTS语音朗读、拍照搜题、学习统计等核心功能；集成友盟统计分析；引入离线eSpeak-NG语音引擎。

v2.0.0架构升级

全面迁移至Jetpack Compose声明式UI；升级数据库到加密版本；新增多套题库支持；重构数据层为Repository模式。

v2.3.02026-06-18

重大更新：实现断点续做功能；新增PracticeProgress持久化进度表；优化交卷评分界面；修复答案判定逻辑；数据库升级至v6。

v2.3.12026-06-18

创建统一答案判定工具类AnswerJudge，消除多模块重复代码，实现判定逻辑全面标准化，支持多输入格式。

✨ 项目亮点

  • 🔧
    自研离线OCR引擎 基于像素分析的完整OCR实现（灰度化→二值化→轮廓检测→字符分类），不依赖任何第三方OCR库，100%离线运行
  • 🌐
    8种TTS引擎工厂模式适配 通过工厂模式统一管理华为HMS、小米AIVS、OPPO AI、vivo、魅族、荣耀、eSpeak-NG、讯飞等8种语音引擎
  • 📦
    离线优先设计 所有核心功能（练习、搜题、语音朗读）均支持完全离线使用，适应工地/车间等网络受限环境
  • 🛡️
    三重数据安全防护 SQLCipher加密数据库 + AES敏感数据加密 + 网络时间验证防篡改，全方位保护用户数据
  • 📋
    精密的题库预设机制 内置50套随机练习预设 + 20套模拟考试预设题组，保证每次练习题目不同但可追溯
