使用场景

TempVault 专为安全临时凭证分享设计

🏢

外包交接

开发完成后需要将生产环境的服务器密码、数据库凭据、API 密钥等传递给外包团队。使用 TempVault 生成一次性链接，设定有效期，交接完成后凭证自动销毁，无需担心密码长期暴露在聊天记录中。

🎓

线上教学

教师在实验课上需要为学生提供云服务器或实验平台的登录凭据。通过 TempVault 分享，可设置课程时长为有效期，课程结束后自动过期，避免学生课后仍能访问系统，也无需手动修改密码。

📶

访客 Wi-Fi

为来访客户提供公司 Wi-Fi 密码时，不用再大声念出复杂密码。生成一条限时链接发送给客户，设定当天有效且仅可查看一次，密码安全又礼貌，完美兼顾安全与便利。

安全设计

🔐

端到端加密

使用 Web Crypto API 的 AES-256-GCM 算法在浏览器端加密，密钥不经过服务端，实现真正的端到端加密。

🛡️

零知识架构

服务端仅存储密文，无法还原明文内容。AES 密钥通过 URL hash 传递，不发送到服务端。

⏱️

自动销毁

支持时间过期、次数耗尽、阅后即焚三种销毁机制，确保临时凭证不会被永久留存。

📝

完整审计

每次访问都自动记录时间、IP 地址和 User-Agent，提供完整的访问审计日志，安全事件可追溯。
