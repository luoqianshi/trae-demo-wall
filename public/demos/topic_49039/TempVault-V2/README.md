# TempVault V2 – 临时凭证安全分享

> TRAE AI 创造力大赛参赛作品 | 学习工作赛道 | 网络安全领域

## 项目简介

TempVault 是一个**真正可跨设备分享**的临时凭证安全分享 Web 应用。面向远程办公团队、线上讲师、自由职业者等群体，解决账号密码、Wi-Fi 密码、测试账号等敏感信息通过微信、邮件明文传播并长期留存的问题。

核心理念：把"发送密码"这件事变成一个**受控、可追溯、自动失效**的安全行为。

## 产品痛点

在日常协作中，我们经常需要把后台账号、Wi-Fi 密码、测试账号等敏感信息发给他人：

- **明文传播**：大多数人用微信、邮件直接发密码，没有任何保护
- **长期留存**：聊天记录和邮件存档永久保存，密码满天飞
- **无法追溯**：不知道对方何时查看、是否保存、是否转发
- **操作繁琐**：加密压缩包步骤多、体验差，私密笔记缺少凭证管理

TempVault 让临时密码不再永久留在聊天记录里。

## 核心功能

### 1. 创建临时凭证包裹
- 输入凭证标题、账号、密码、补充说明
- 设置有效时间（10分钟 / 1小时 / 24小时 / 7天）
- 设置最大查看次数（1次 / 3次 / 5次 / 10次）
- 开启「阅后即焚」模式
- 开启「禁止复制」模式

### 2. 浏览器端 AES-256 加密
- 使用 Web Crypto API 在浏览器端加密凭证内容
- 服务端仅保存密文，无法读取凭证明文
- 解密密钥仅在分享链接的 URL hash 中

### 3. 一次性安全链接
- 生成包含 token 和解密密钥的完整链接
- 链接格式：`/view/TOKEN#key=BASE64_KEY`
- URL hash 不会发送到服务端，实现零知识架构

### 4. 阅后即焚
- 查看后立即销毁凭证
- 再次打开显示"凭证已销毁"
- 适合高敏感信息的单次分享

### 5. 有效期和查看次数限制
- 超过有效时间不可查看
- 超过查看次数不可查看
- 管理页可手动销毁和模拟过期

### 6. 管理页状态追踪
- 5 种状态标签：有效 / 已查看 / 已过期 / 已销毁
- 凭证列表展示所有已创建的凭证
- 支持手动销毁和模拟过期按钮

### 7. 访问日志和生命周期时间线
- 每次查看自动记录时间和浏览器信息
- 生命周期时间线：创建 → 分享链接生成 → 被访问 → 销毁/过期
- 详情弹窗展示完整审计信息

## 安全设计说明

### 架构图

```
创建者浏览器                          服务端                          接收者浏览器
┌──────────────┐                    ┌──────────┐                    ┌──────────────┐
│ 1. 输入凭证  │                    │          │                    │              │
│ 2. AES加密   │──密文+IV──────────→│ 存储密文 │──密文+IV──────────→│ 4. 从hash取key│
│ 3. 生成AES密钥│                    │ 存储元数据│                    │ 5. AES解密    │
│ 4. 密钥→URL  │                    │ 记录日志 │←──查看记录────────│ 6. 显示凭证  │
│    hash      │                    │ 管理状态 │                    │              │
└──────────────┘                    └──────────┘                    └──────────────┘
                         链接格式：/view/TOKEN#key=BASE64_KEY
                         服务端永远无法获取 key（hash 不发送到服务端）
```

### 安全要点

1. **浏览器端加密**：使用 Web Crypto API 的 AES-256-GCM 算法，凭证内容在离开浏览器前就已加密
2. **服务端仅存密文**：服务端只接收和存储密文（Ciphertext）和初始化向量（IV），无法解密
3. **解密密钥位于 URL hash 中**：AES 密钥通过 URL hash 传递，HTTP 请求不会携带 hash
4. **服务端无法读取凭证原文**：整个架构中，服务端始终无法获取解密密钥，因此无法读取明文
5. **Demo 环境请勿输入真实密码**：本 Demo 为演示用途，数据保存在服务器 JSON 文件中

## 核心体验流程

### 创建凭证
1. 打开首页，点击"立即创建安全凭证"
2. 填写凭证标题、账号、密码
3. 设置有效时间和查看次数
4. 可选开启阅后即焚和禁止复制
5. 点击"生成安全分享链接"

### 复制完整链接
1. 创建成功后，页面显示完整分享链接
2. 链接格式：`http://域名/view/TOKEN#key=BASE64_KEY`
3. 点击"复制链接"或"在新窗口中模拟接收者打开"

### 接收者查看
1. 对方打开链接，看到安全确认页
2. 点击"查看凭证内容"
3. 前端从 URL hash 中提取密钥，解密凭证
4. 显示账号和密码（可显示/隐藏）

### 失效机制
- **阅后即焚**：第一次查看后立即销毁，再次打开显示"凭证已销毁"
- **次数耗尽**：达到最大查看次数后显示"访问次数已用完"
- **到期失效**：超过有效时间后显示"凭证已过期"

### 管理页审计
1. 查看所有已创建凭证的状态
2. 点击"详情"查看生命周期时间线和访问日志
3. 使用"模拟过期"快速测试过期状态
4. 使用"手动销毁"立即使凭证失效

## 本地运行

### 前置要求
- Node.js >= 14

### 步骤

```bash
cd TempVault-V2
npm install
npm start
```

打开浏览器访问 http://localhost:3000

## 部署说明

### Render / Railway（推荐）

1. 将代码推送到 GitHub
2. 在 Render/Railway 创建新 Web 服务
3. 设置启动命令：`node server.js`
4. 设置端口环境变量：`PORT`（Render/Railway 会自动设置）
5. 部署完成后访问自动生成的域名

### Vercel（需适配）

Vercel 默认仅支持静态站点。如需后端 API，需要使用 Vercel Serverless Functions 或单独部署后端。

### 简易部署

任何支持 Node.js 的云平台均可直接部署，无需数据库配置。

## 验收测试步骤

### 测试 1：阅后即焚
1. 创建凭证，maxViews=1，开启阅后即焚
2. 复制完整链接
3. 用新窗口或无痕窗口打开
4. 点击查看凭证
5. 再次刷新 → 应显示"凭证已销毁"
6. 管理页 → 状态应为"已销毁"，访问日志应有记录

### 测试 2：多次访问
1. 创建凭证，maxViews=3，不开启阅后即焚
2. 打开链接查看一次
3. 管理页 → 状态应为"已查看"，剩余 2/3
4. 再查看两次后 → 应显示"访问次数已用完"

### 测试 3：过期状态
1. 创建凭证 → 管理页点击"模拟过期"
2. 打开链接 → 应显示"凭证已过期"

### 测试 4：跨设备分享
1. 创建凭证 → 复制完整链接
2. 在不同浏览器/设备中打开链接
3. 应能正常查看凭证（密钥在 URL hash 中）

### 测试 5：详情弹窗
1. 打开任意凭证详情
2. 应正确显示状态、剩余次数、生命周期时间线、访问日志
3. 不允许出现 undefined、NaN 或空白内容

## 已知限制

1. **Demo 环境不采集真实 IP**：访问日志中 IP 显示为"浏览器演示访问"，User-Agent 仅提取浏览器名称。这是演示环境的设计限制，非生产功能。
2. **禁止复制不能真正防止截图或人工记录**：浏览器端的"禁止复制"模式仅隐藏复制按钮，无法阻止截图或手动记录密码。
3. **创建者浏览器 localStorage 保存 token-key 映射**：管理页生成完整链接需要从创建者浏览器的 localStorage 中读取解密密钥。这是零知识架构下的演示限制——密钥不会存储到服务端。
4. **数据保存在服务器 JSON 文件中**：适合 Demo 演示，不适合生产环境高并发场景。
