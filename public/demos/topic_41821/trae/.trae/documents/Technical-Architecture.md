# 技术架构文档

## 1. 架构设计

```
┌─────────────────────────────────────────┐
│              前端 (HTML/CSS/JS)          │
├─────────────────────────────────────────┤
│  UI层: 场景选择 | 输入区 | 输出区 | 历史  │
├─────────────────────────────────────────┤
│  业务层: 提示词转换引擎 | 模板系统        │
├─────────────────────────────────────────┤
│  数据层: LocalStorage 历史记录            │
└─────────────────────────────────────────┘
```

## 2. 技术选型

- **前端**: 纯HTML5 + CSS3 + Vanilla JavaScript
- **样式**: 内联CSS，使用CSS变量实现主题
- **数据持久化**: LocalStorage
- **字体**: Google Fonts (Orbitron, Inter)
- **无外部依赖**: 单文件可独立运行

## 3. 路由定义

单页应用，无需路由。

## 4. 功能模块

### 4.1 场景选择模块
支持的AI场景：
- 🎨 绘画 (Midjourney, Stable Diffusion, DALL-E)
- ✍️ 写作 (ChatGPT, Claude, 文心一言)
- 💻 编程 (GitHub Copilot, Cursor)
- 💬 对话 (各类AI对话助手)
- 📊 分析 (数据分析AI)
- 🎬 视频脚本 (AI视频生成)

### 4.2 提示词转换引擎
转换逻辑：
1. 提取用户输入的核心意图
2. 根据场景添加专业术语和格式
3. 添加质量修饰词和参数建议
4. 输出结构化的提示词

### 4.3 历史记录
- 使用LocalStorage存储
- 最多保存20条记录
- 支持快速复用历史提示词

## 5. 数据模型

### 5.1 历史记录结构
```javascript
{
  id: timestamp,
  input: "用户输入的原始话语",
  scene: "场景类型",
  output: "生成的提示词",
  createdAt: "创建时间"
}
```
