## 三、启动方式（推荐 → 备选）

### ✅ 方式 1：本地 HTTP 服务器（强烈推荐）

> AI 接口直连需要稳定的网络环境，HTTP 服务器方式可避免 `file://` 协议导致的跨域限制，**功能最完整**。

#### 1A. 使用 Node.js（如已安装）

在 `dist-single` 目录下执行：

```powershell
cd "d:\hzx_code\trae ai 创造力大赛\qingxin-shudong\dist-single"
node -e "require('http').createServer((q,s)=>{require('fs').readFile(q.url==='/'?'index.html':q.url.slice(1),(e,d)=>{s.writeHead(e?404:200,{'Content-Type':'text/html;charset=utf-8'});s.end(d||'404')})}).listen(8000,()=>console.log('Server: http://localhost:8000/'))"
```

看到 `Server: http://localhost:8000/` 后，浏览器访问该地址即可。

#### 1B. 使用 Python（如已安装）

```powershell
cd "d:\hzx_code\trae ai 创造力大赛\qingxin-shudong\dist-single"
python -m http.server 8000
```

浏览器访问：`http://localhost:8000/`

#### 1C. 使用 VS Code Live Server 插件

在 VS Code 中右键 `index.html` → `Open with Live Server`。

---

### ⚠️ 方式 2：直接双击打开（应急方案）

直接双击 `index.html`，默认浏览器会以 `file://` 协议打开。

**已知限制**：
- 部分浏览器（如 Chrome）会拦截 `file://` 下的跨域 fetch 请求，导致 **AI 接口调用失败**
- 此时应用会自动降级为**预设回复**（不会白屏，但 AI 失去真实智能）
- IndexedDB 在 `file://` 下行为不稳定，历史记录可能无法持久化

**如必须使用此方式并启用 AI**，请按下文「浏览器权限设置」调整。

## 四、浏览器权限设置

### 1. 跨域请求（AI 功能必需）

若使用 `file://` 打开，AI 调用可能被浏览器拦截。解决方法：

#### Chrome
1. 关闭所有 Chrome 窗口
2. 创建快捷方式，目标末尾追加：
   ```
   --allow-file-access-from-files --disable-web-security --user-data-dir="C:\chrome-temp"
   ```
3. 用此快捷方式启动 Chrome，再打开 `index.html`

#### Firefox
地址栏输入 `about:config`，搜索 `security.fileuri.strict_origin_policy`，设置为 `false`。

#### Safari
启用「开发」菜单 → 勾选「停用跨域限制」。

> 推荐使用方式 1（HTTP 服务器），可避免以上所有配置。

### 2. IndexedDB（数据持久化）

应用使用 IndexedDB 保存对话记录与每日卡片。默认开启，无需额外配置。
- 若使用「无痕模式」或清理浏览器数据，历史记录会丢失
- 不要在浏览器设置中禁用「网站数据存储」

### 3. 剪贴板权限

「复制到剪贴板」功能需要 `clipboard` 权限，浏览器会弹出授权提示，请点击「允许」。

### 4. 字体加载

应用使用 Google Fonts（Noto Sans SC / Noto Serif SC）。若网络无法访问 Google 服务，会自动回退到系统默认中文字体，不影响功能。

## 七、常见问题排查

### Q1：打开后白屏
**可能原因**：
- 浏览器版本过低 → 升级到 Chrome 90+/Firefox 90+/Safari 15+
- 文件损坏 → 重新构建（见第九节）

**排查方法**：按 F12 打开开发者工具，查看 Console 报错信息。

### Q2：AI 对话无响应或显示预设回复
**可能原因**：
- 使用 `file://` 协议，跨域被拦截 → 改用 HTTP 服务器（方式 1）
- 网络无法访问 `apihub.agnes-ai.com` → 检查网络代理/防火墙
- API Key 已过期或被限流 → 重新构建时替换 `.env.single` 中的 Key

### Q3：图片无法生成或显示
**可能原因**：
- 图像接口返回 URL 在 `file://` 下加载失败 → 使用 HTTP 服务器
- 浏览器拦截了图片跨域请求 → 见第四节「跨域请求」配置

### Q4：登录后数据丢失
**可能原因**：
- 使用了无痕模式 → 改用正常模式
- 清理了浏览器数据 → 数据无法恢复，重新使用即可
- 切换了浏览器/设备 → 每个浏览器数据独立

### Q5：字体显示为方块或默认字体
**原因**：Google Fonts 加载失败（网络问题）。
**解决**：不影响功能，可忽略；或科学上网后重新加载。

### Q6：手机端访问
**方法**：
1. 确保电脑与手机在同一 Wi-Fi
2. 启动 HTTP 服务器时监听 `0.0.0.0`：
   ```powershell
   node -e "require('http').createServer((q,s)=>{require('fs').readFile(q.url==='/'?'index.html':q.url.slice(1),(e,d)=>{s.writeHead(e?404:200,{'Content-Type':'text/html;charset=utf-8'});s.end(d||'404')})}).listen(8000,'0.0.0.0',()=>console.log('Server: http://0.0.0.0:8000/'))"
   ```
3. 查看电脑内网 IP（`ipconfig`），手机浏览器访问 `http://<电脑IP>:8000/`
