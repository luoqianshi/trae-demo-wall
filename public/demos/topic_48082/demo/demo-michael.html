01 · 痛点

散户为什么总是亏

A 股散户贡献 80% 交易量，却拿走不足 30% 的利润。问题不在选股，在执行。

原始散户

  • 凭感觉买卖，没规则
  • 追涨杀跌，单日 -7% 还死扛
  • 亏损时补仓摊低成本
  • 突破后涨幅 15% 才敢追
  • 同时持有 10+ 只，看不过来
  • 止损触发"再等等"
  • 听消息面炒作，不查基本面
→

Michael 系统

  • 规则驱动，参数全部可调
  • 单日 -7% 立即清仓，无情绪
  • 只在盈利仓位加仓
  • 突破后涨幅 > 9.5% 不追
  • 同时最多 5 只，单股上限 30%
  • 止损触发当日收盘前执行
  • ROE/PE/股息率强制门槛
02 · 系统设计

四大模块 + 两大工作流

每个模块可独立触发，模块间通过数据流衔接。所有买卖建议必须人工确认，系统绝不自动执行。

MODULE · I

选股

429 只自选股 → 四维评分（涨幅·量比·冲破·换手）→ Top 8 → 基本面筛选 → Top 5 → 关键点监控

scripts: screener.py / watchlist.py
MODULE · II

买入

入场五条件验证（基本面/突破/大盘/无利空/周线MACD）→ 深度分析 → 三阶段建仓 3:3:4

scripts: analyzer.py
MODULE · III

持有

持仓检查 → 三重止损（单日-7%/累计4日-10%/MA55）→ 高位回撤 5% 锁利 / 12% 清仓

scripts: position_manager.py / stoploss.py
MODULE · IV

卖出

退出触发（8 类）→ 决策流程 → 禁忌清单（8 条红线）→ 冷静期 2 日 → 写入交易日志

scripts: stoploss.py
选股 → 买入 → 持有 → 卖出 ⚡ 每日巡检 15:30 · 每周复盘 · 策略回测
03 · AI 赋能

TRAE 的三种武器

不是"AI 帮你写代码"，而是"AI 是你的交易秘书"。三种武器协同，覆盖从对话到执行的全链路。

SKILL

对话即工作流

用 SKILL.md 把"选股→买入→持有→卖出"封装成可被自然语言触发的模块化工作流。用户说"巡检"，AI 自动走完 5 个 Step。

// 触发示例 "帮我巡检一下" → 同步自选股 → 全量条件检查 → 输出结构化报告 → 等待人工确认
MCP

实时行情接入

通过 TDX MCP 工具，AI 直接调用 tdx_quotes / tdx_kline / tdx_indicator_select 拿到 A 股实时行情、K线、财务指标，无需写爬虫。

// MCP 工具调用 tdx_kline(code="600519", setcode="1", period="4", // 日线 wantNum="100")
AGENT

定时巡检自动化

每日 15:30 收盘后自动触发巡检任务，全量检查持仓与自选股信号，输出报告等待人工确认。绝不自动交易。

// 自动化任务 cron: "30 15 * * MON-FRI" trigger: 每日收盘 action: 全条件巡检 output: 报告 + 待确认建议
