LLVM SelectionDAG 入门系列

8. DAGCombiner：SelectionDAG 里的 InstCombine

DAGCombiner 是 SelectionDAG 里负责优化的 pass，有点像 IR 层的 InstCombine。

DAGCombiner 的核心职责是：

核心职责

在合法化之后持续清理冗余、优化模式、执行目标相关的 DAG 级别优化。

8.1 为什么要"合法化之后"还做 Combine？

因为合法化过程有时候会引入冗余。例如：

  • 类型提升可能引入 mask；
  • Expand 操作可能引入可以合并的相邻操作；
  • Promote 操作可能暴露新的优化机会。

8.2 DAGCombiner 的核心循环

DAGCombiner 是一个迭代优化循环：

Repeat:
  For each node in DAG (bottom-up):
    Try to combine this node with its operands
    If combined:
      Replace old node with new result
      Update users
      Repeat
Until no changes

8.3 常见 Combine 案例

DAGCombiner 会执行大量优化，常见例子包括：

  • 常量折叠：add(c1, c2) → constant(c1 + c2)
  • 代数简化：mul(x, 2) → shl(x, 1)
  • 公共子表达式消除：相同子树只保留一份
  • 模式匹配优化：mul(add(x, y), z) → mad(x, z, y*z)（需要目标支持 MAD）

8.4 目标后端自定义 Combine

目标后端可以实现额外的 Combine 规则：

SDValue TargetLowering::Combine(SDNode *N, ...);

SelectionDAG 在执行通用 Combine 之后，会调用这个 hook 执行目标相关的 Combine。
