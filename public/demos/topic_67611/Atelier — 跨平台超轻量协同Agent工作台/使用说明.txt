【一、文件清单】
─────────────────

  • 使用说明.txt      本文件
  • 启动.bat          Windows 一键启动脚本（启动主程序）
  • 启动.sh           Linux / macOS 一键启动脚本（启动主程序）
  • tempus_daemon.py  Tempus 守护进程（独立运行，监听全局键鼠）

      └─ 公文写作技能包.zip



【二、快速开始】
─────────────────
1. 双击「智能Agent.html」或运行启动脚本
   - Windows：双击「启动.bat」
   - Linux/macOS：终端运行 bash 启动.sh，或右键赋予执行权限后双击

2. 在浏览器中打开后，首次使用需要配置 DeepSeek API：
   点击右上角齿轮图标，填入：
     · API Key    （在 https://platform.deepseek.com 申请）
     · Base URL   （默认 https://api.deepseek.com，可换代理）
     · 模型       （推荐 deepseek-chat）
   点击「测试连接」确认无误后保存。

3. 配置完成后，在底部输入框下达任务即可。


【三、六大核心能力】
─────────────────

■ 1. 多模态内容生成
   一条指令即可生成：文档（.doc）、表格（.xlsx/.csv）、
   图表（.png）、演示文稿（.pptx/.html）。
   系统会根据任务自动选择格式，也可在指令中明确要求：
     示例：「分析Q3销售数据，生成柱状图+数据表+5页汇报PPT」

■ 2. 自主任务规划与交付
   Agent 自主完成需求拆解 → 步骤规划 → 执行 → 整合交付。
   全程无需人工干预，最终直接给出可用结果。
   规划/执行/整合各阶段在界面中以可视化步骤展示。

■ 3. 多 Agent 并行处理
   当任务含多个独立子任务时，自动启动多个 Agent 同时处理。
   界面右侧显示并行进度卡，实时展示每个子 Agent 的状态。
   示例：「并行完成：①季度汇报 ②KPI数据表 ③5页PPT」

■ 4. Skills 扩展（压缩包上传）
   点击左侧「技能」Tab → 底部「上传技能压缩包」按钮。
   支持两种 ZIP 格式：
     a) 单技能包：包含 SKILL.md（YAML 前置元数据 + 正文 prompt）
     b) 技能集包：包含 manifest.json + skillsets/*.md（工作流）
                   + skills/*.zip（子技能）
   示例：可上传「示例技能包/公文写作技能包.zip」试用。

■ 5. 超级兼容
   纯前端 HTML，无需安装任何运行环境。
   兼容：Windows 7+、Linux（Ubuntu/CentOS/Arch 等）、macOS
   浏览器要求：Chrome 90+、Edge 90+、Firefox 88+、Safari 14+
   （需支持 ES2017 + localStorage）

■ 6. 方便查找
   全部文件集中在「智能Agent工作台」一个文件夹内
   所有文件使用中文命名，便于定位
   主程序为单文件 HTML，可直接 U 盘携带、网盘分享

■ 7. 护眼淡黄主题（v2.1 新增）

■ 8. 提问处文件上传（v2.1 新增）
   底部输入框左侧新增两个工具按钮：
     · 📎 附件按钮：点击或拖拽文件到输入框即可上传
     · 🖥️ AI 操控电脑按钮：见下条说明
   支持的文件类型：
     · 文本类：.txt .md .json .yaml .log .docx（≤2MB 自动读取文本）
     · 数据类：.csv .xlsx .xls
     · 代码类：.js .ts .py .java .go .html .css .sql 等
     · 图片类：自动转 dataURL 附入消息

   上传后会以「卡片」形式列在输入框上方，可单独 × 移除。
   发送时附件文本会拼到消息内供 AI 阅读。

■ 9. AI 操控电脑（双模式 + 撤回缓冲）（v2.1 新增）
   点击输入框右侧 🖥️ 按钮 → 打开「AI 操控电脑」配置面板。

   ◆ 双模式
     · 模式 1 · 确认模式：每个动作执行前弹行内审批按钮
       （确认 / 跳过 / 取消），最安全。
     · 模式 2 · 自主模式：动作按设定间隔自动连续执行，
       无需人工干预，效率高。

   ◆ 撤回缓冲（两模式均有）
     所有成功执行的动作都会进入「撤回缓冲」（默认 30 步），
     可在动作面板上：
       · 单步「撤回」：回退该步操作（如恢复输入框原值、反向滚动、blur 焦点等）
       · 「全部撤回」：一键回退到本批动作开始前的状态
     在配置面板点「清空缓冲」可重置所有已执行动作。

   ◆ 权限白名单
     可勾选允许 AI 执行的 6 种动作类型：
       open_url / type_text / scroll / click_element / copy_clipboard / focus_input
     未勾选的类型会被自动跳过并标记「已禁用」。

   ◆ 安全说明
     受浏览器沙箱限制，AI 实际可执行的动作仅限本应用页面与浏览器能力
     （打开网页、填写表单、滚动、复制剪贴板、聚焦输入框等）。
     真正的跨应用 OS 级鼠标键盘控制需要配套本地助手程序。
     本实现重点保障「双模式审批流程 + 撤回缓冲」机制，
     确保任何模式下出问题都能一键回退。

   ◆ 触发方式
     AI 在用户明确要求「操控电脑」「自动操作」「帮我打开」「帮我填入」
     等场景下会输出 computer_use 动作面板。用户点「开始执行」启动流程。

■ 10. 强化意图识别 · 自动出 Office 文档（v2.2 新增）
   当用户请求涉及正式文体时，AI 禁止只输出纯 Markdown 文本，
   必须以对应的 Office 块产出完整正式成果：
     · 文档类（用 docx:html 块）
       报告/总结/发言稿/致辞/讲话稿/公文/通知/通报/请示/批复/决定/意见
       /纪要/函/命令/方案/制度/规定/办法/条例/章程/规范/汇报材料/邀请函
       /感谢信/倡议书/建议书/申请书/承诺书/新闻稿/通讯稿/宣传稿/通稿
       /调研提纲/说明书/操作手册/技术文档/教案/合同/协议/简历/求职信
       /评论员文章/社论/编者按 等
     · 演示类（用 slides:markdown 块）
       演讲稿/致辞/开幕词/闭幕词/汇报材料/演示/汇报 PPT 等
     · 数据类（用 table:csv 块）
       清单/数据统计/数据对比/对比表/统计表/预算表/数据明细/数据汇总 等
   即便用户未明说「用 Word/用 PPT」，只要文体属于上述类别，
   就默认且必须输出对应 Office 块，禁止推诿。
   触发时右上角徽章会显示「Word 文档/幻灯片/表格」标识，
   并有 toast 提示「意图识别 → XXX」。

■ 11. 强化学习复用 · GitHub / 学习强国（v2.2 新增）
   生成任何内容前，AI 优先参考成熟来源，再在它的基础上完成：

   ◆ 写代码场景（识别关键词：写代码/写程序/写脚本/实现功能/写一个XX/
     用XX语言/做组件/做页面/爬虫 等）
     → 先参考 GitHub 上同类成熟开源项目
       （requests / flask / ant-design / vue-element-admin / AdminLTE /
        scrapy / cheerio / puppeteer / electron-quick-start 等）
     → 以它们的目录结构、代码组织、API 设计、最佳实践为模板复刻
     → 说明复用了哪个项目/库的什么模式
     → 然后在它的基础上完成用户的具体需求
     → 禁止凭空从零写代码

   ◆ 写文章场景（识别关键词：写文章/写稿/写评论/写时评/写通讯稿/
     心得体会/理论学习/读书笔记/评论员文章 等）
     → 先参考学习强国（xuexi.cn）上的同类文章
       （评论员文章、理论学习、心得体会、通讯稿、专题报道、思想汇报、
        读书笔记等）
     → 以它们的篇章结构、立意、行文风格、政治站位、用词为模板复刻
     → 说明参考了哪类文章的什么特点
     → 然后在此基础上完成用户的具体需求
     → 禁止凭空从零写文章

   ◆ 复用来源标注
     AI 在最终输出开头用一行 [REUSE] 前缀注明本次复用的来源，
     例如：[REUSE] GitHub·flask 项目的 app 工厂模式 + 蓝图组织
     [REUSE] 标注会被渲染为带「REUSE」徽标的青色提示框，醒目可见。
     触发时右上角徽章会显示「复刻 GitHub」/「复刻 学习强国」标识。

■ 12. 持续改进能力 · Self-Improvement（v3.0 新增）
   AI 每次回复后会自动自我反思，输出一行 [IMPROVE] 标注的改进点。
   这些改进点会被自动提取入库，在后续对话中作为经验注入系统提示，
   帮助 AI 在后续回复中主动避免已发现的问题，持续提升回答质量。

   ◆ 工作流程
     1. AI 回复末尾添加 [IMPROVE] 改进反思（如：「应先确认需求再动手」）
     2. 系统自动提取并入库（去重，最多保留 100 条）
     3. 下次对话时，历史改进经验注入 system prompt
     4. AI 在回答中主动应用这些经验

   ◆ 查看与管理
     左侧侧栏新增「改进」Tab，可查看所有改进记录（含时间/类别）。
     点击底部「清空改进」按钮可重置全部记录。
     每条记录可单独 × 删除。

   ◆ 改进示例
     [IMPROVE] 用户要求报告但未明确字数，应主动询问目标篇幅
     [IMPROVE] 代码示例应附带运行环境说明
     [IMPROVE] 长文分块时应更注意段落衔接

■ 13. Office 套件直接输出文件（v3.0 新增）
   当 AI 输出 docx:html / slides:markdown / table:csv 块时，
   不再在聊天中显示完整内容，而是直接生成可下载的 Office 文件：

   ◆ 输出形式
     · Word 文档 → 自动生成 .doc 文件
     · PowerPoint → 自动生成 .pptx 文件
     · Excel 表格 → 自动生成 .xlsx 文件
     生成过程显示「正在生成文件…」状态，完成后变为「✓ 已就绪」。

   ◆ 文件卡片
     每个生成的文件以紧凑卡片形式显示：
       · 左侧：文件类型图标（蓝=Word, 红=PPT, 绿=Excel）
       · 中间：文件名 + 文件类型 + 文件大小
       · 右侧：「下载」按钮 + 「预览」按钮
     点击「下载」直接下载文件。
     点击「预览」在新标签页打开文件预览。

   ◆ 资产管理
     所有生成的文件自动进入左侧「资产」Tab，
     可随时重新下载或删除。最多保留 30 个文件。

   ◆ 库依赖
     · SheetJS (xlsx) — Excel 文件生成
     · PptxGenJS — PowerPoint 文件生成
     · Word 文档使用 HTML+MIME 方式生成（兼容 Word/WPS）
     首次使用需联网加载 CDN 库，之后浏览器缓存。

■ 15. 融合 7 大技能包（v3.1 新增）
   本工作台融合了用户上传的 skills-pack 中的 7 个专业技能，
   已适配纯前端运行环境（原技能依赖 Python/bash，已转换为 prompt 规则）。
   在左侧「技能」Tab 中可看到 10 个内置技能（3 个原有 + 7 个融合）：

   ◆ Word 文档（融合自 docx 技能）
     - 用 docx:html 块输出，系统自动生成 .doc 文件
     - 注入严肃公文样式（仿宋 14pt、黑体标题、@page A4 边距）
     - 基于事实创作，禁止编造数据/日期/公司名
     - 严格遵循字数/字体/演讲时长等约束

   ◆ PowerPoint（融合自 pptx 技能）
     - 用 slides:markdown 块输出，系统自动生成 .pptx 文件
     - 5 种设计风格：general/teach/persuade/showcase/inform
     - 每页标题 ≤12 字 + 3-5 个要点（每点 ≤30 字）
     - 支持演讲稿（> 引用块附加）

   ◆ Excel 表格（融合自 xlsx 技能）
     - 用 table:csv 块输出，系统自动生成 .xlsx 文件
     - 零公式错误（避免 #REF!/#DIV/0!/#VALUE!）
     - 视觉美观、多 Sheet、数据源标注、图表配套
     - 修改现有文件保留原数据

   ◆ PDF 文档（融合自 pdf 技能）
     - 用 pdf:html 块输出，系统生成 HTML 并提供「打印为 PDF」按钮
     - 内置 A4 排版（思源宋体/仿宋、12pt、行距 1.6、@page 边距）
     - 用户点「打印为 PDF」→ 浏览器打印对话框 → 另存为 PDF
     - 上传 .pdf 附件时自动激活此技能

   ◆ 学术论文（融合自 paper-writer 技能）
     - 支持 5 种文体：学位论文/课程论文/开题报告/文献综述/英文论文
     - 中文 GB/T 7714 引用、英文 APA/IEEE/Chicago
     - 每段 ≥250 字（英文 ≥180 words），一次成稿
     - 禁止编造引用/数据/实验结果

   ◆ Web 构建（融合自 web-builder 技能）
     - 计划驱动：先输出项目计划（技术栈/页面/组件）
     - 4 种原型：static/spa/fullstack/separated
     - 实现顺序铁律：项目结构→样式→布局→路由→逐页→数据
     - TypeScript strict、单组件 ≤200 行、Tailwind 优先

   ◆ 持续改进（融合自 self-improvement 技能）
     - 每次回复末尾输出 [IMPROVE] [类型:优先级] 改进反思
     - 4 种类型：error/correction/insight/feature
     - 4 级优先级：critical/high/medium/low
     - 去重机制：相似条目不重复记录
     - 晋升规则：同类问题 ≥3 次自动晋升为永久经验

   ◆ 自动激活机制
     当用户请求涉及论文/Web 项目/PDF 等专业领域时，
     系统自动激活对应技能（无需手动开启）：
       - 写论文/开题报告/文献综述 → 自动激活「学术论文」
       - 做网站/构建 Web 应用 → 自动激活「Web 构建」
      - 上传/生成 PDF → 自动激活「PDF 文档」
    激活时有 toast 提示「已激活技能：XXX（原因）」

■ 16. 撤回功能（v3.2 新增）

■ 18. Tempus 模式 · 桌面小球 + 全局键鼠记录（v3.2 新增）
   ─────────────────────────────────────────────────────
   浏览器沙箱禁止网页监听其它软件的键鼠操作，
   因此 Tempus 采用「混合方案」：
     · Python 守护进程（tempus_daemon.py）独立运行在系统层
     · 用 pynput 库监听全局键盘鼠标，写入本地 JSONL
     · HTML 仅作为「展示/整理前端」，通过 HTTP 轮询拉取事件

   ◆ 启动方式
     1) 安装依赖（首次）：
        Windows: 双击 启动Tempus.bat（自动检查并安装 pynput）
        Linux:   ./启动Tempus.sh（自动检查并安装 pynput + xlib）
     2) 守护进程会驻留终端窗口，监听全局键鼠
     3) 在主程序点击右上角 Tempus 切换按钮（⊕ 图标）
     4) 主界面关闭，只剩桌面右下角琥珀色脉冲小球
     5) 点击小球重新打开主界面 → 自动调用 AI 整理日志为 1 条记忆

   ◆ 守护进程能力
     · 监听全局键盘（包括其它软件中的输入）
     · 监听全局鼠标点击和滚动
     · 词级累积（连续字符自动合并为 word 事件，更紧凑）
     · 自动脱敏：API key（sk-***）、密码关键词、长数字串自动替换
     · 本地持久化：~/.atelier_tempus/log.jsonl
     · HTTP 端点：http://127.0.0.1:8791
       - GET  /status         查询运行状态
       - GET  /events?since=N 拉取某时间戳之后的事件
       - POST /clear          清空日志
       - POST /shutdown       关闭守护进程
     · Ctrl+C 优雅退出

   ◆ Tempus 面板（点击小球打开）
     · 三项统计：已记录操作数 / 已整理为记忆数 / 运行时长
     · 守护进程连接状态指示器（绿点=已连接 / 红点=未运行）
     · 最近 30 条操作日志（type / 时间 / 内容预览）
     · 四个按钮：立即整理为记忆 / 清空日志 / 退出 Tempus / 关闭守护进程

   ◆ 整理机制（API 安全）
     · 用户重新打开界面（点击小球）时自动触发整理
     · 或手动点「立即整理为记忆」按钮
     · 调用用户当前在 API 配置中选择的模型（不切换模型）
     · 把最近 200 条日志归纳为 1 条 ≤80 字的记忆
     · 整理后日志清空（前端 + 守护进程双清）
     · 保证用户的 API 配置不被绕过

   ◆ 跨平台支持
     · Windows：开箱即用
     · Linux：需要 X11 会话（Wayland 下键盘监听可能受限）
     · macOS：未做专门适配（pynput 需要「辅助功能」权限）


【四、数据存储】
─────────────────
所有数据均保存在浏览器本地 localStorage 中：
  • API 配置（密钥仅本地存储，不会上传）
  • 会话历史（多会话切换、重命名、删除）
  • 技能库（含 ZIP 上传的技能）
  • 记忆库（Agent 自动沉淀的关键事实）
  • 改进记录（AI 自我反思的持续改进经验）
  • 资产清单（生成的图表/表格/PPT/文档/Office 文件）
  • Computer Use 配置（模式/缓冲深度/权限白名单）
  • Tempus 操作日志（从守护进程拉取的键鼠事件）

  Tempus 守护进程的键鼠日志独立存储在文件系统：
  • Windows: %USERPROFILE%\.atelier_tempus\log.jsonl
  • Linux:   ~/.atelier_tempus/log.jsonl

注意：清空浏览器缓存会删除 localStorage 数据，但不影响守护进程日志。
不同浏览器之间数据不互通。
请定期通过「数据管理」面板清空不需要的数据。

【七、常见问题】
─────────────────
Q: 图表/PPT 导出按钮无反应？
A: 这些功能依赖联网加载 Chart.js / PptxGenJS / SheetJS 库。
   首次使用需联网，浏览器会缓存。完全离线场景下，
   仍可使用对话、规划、长文生成、技能等核心功能。

Q: 报错「API 401 未授权」？
A: API Key 错误或已过期。请到 DeepSeek 平台重新生成。

Q: 报错「API 429 限流」？
A: 请求频率过高或额度用完。等待几分钟后重试，
   或在配置中降低 max_tokens。

Q: 上传 ZIP 后技能没出现？
A: 请确认 ZIP 内含 SKILL.md 或 manifest.json。
   可用解压软件先打开 ZIP 检查结构。

Q: 文件上传后 AI 读取不到内容？
A: 文本类文件 ≤2MB 才会读取全文；超过则只附文件名与大小。
   单文件硬上限 5MB，超出会跳过上传。

Q: AI 操控电脑的动作面板没出现？
A: AI 仅在用户明确要求「操控电脑」「自动操作」「帮我打开网址」
   「帮我填入」等场景下才输出 computer_use 块。
   也可手动让 AI 输出测试，例如：
     「请用 computer_use 块输出 3 个测试动作」
   然后在面板上点「开始执行」。

Q: 模式 2 自主模式下出问题如何回退？
A: 在动作面板每一步右侧点「撤回」单步回退，
   或点底部「全部撤回」一键回到本批开始前状态。
   也可在配置面板点「清空缓冲」重置全部。

Q: Computer Use 配置丢失？
A: 配置保存在 localStorage 键 atelier.cuConfig.v2 中。
   清空浏览器缓存会一并清除，请谨慎清理。

Q: 如何迁移数据到另一台电脑？
A: 浏览器控制台执行：
   copy(JSON.stringify({
     settings: localStorage.getItem("atelier.settings.v2"),
     sessions: localStorage.getItem("atelier.sessions.v2"),
     skills: localStorage.getItem("atelier.skills.v2"),
     memory: localStorage.getItem("atelier.memory.v2"),
     improvements: localStorage.getItem("atelier.improvements.v2"),
     assets: localStorage.getItem("atelier.assets.v2"),
     cuConfig: localStorage.getItem("atelier.cuConfig.v2")
   }))
   将复制的内容在新电脑浏览器控制台中反向写入即可。
