# FC 关卡仿真方案

> 目标：在 H5 网页中实现高度仿真的 FC（红白机）风格关底场景，替代当前用纯 CSS 绘制的粗糙场景。
>
> 应用场景：玩家行进到关卡终点后，切换到该关卡的关底场景，完成打卡任务后通关。

---

> ⚠️ **2026-06-23 重要更新**
>
> **技术栈已变更**：本方案最初选型 Phaser.js 3，实际开发中改为**纯 Canvas 2D**。
>
> | 项 | 原方案 | 实际采用 |
> |----|--------|----------|
> | 引擎 | Phaser 3.70 | 纯 Canvas 2D |
> | 体积 | 1.1MB（需内联） | 零依赖 |
> | 瓦片渲染 | Tilemap API | 手写 drawImage 循环 |
> | 精灵动画 | Animation Manager | 手写帧切换 |
> | 物理碰撞 | Arcade Physics | 手写 AABB |
> | `file://` 兼容 | 需 base64 内联 | 原生兼容 |
>
> **变更原因**：① 单文件交付体积 ② `file://` 协议兼容性 ③ 评委双击即开
>
> **当前进度**：第一关（咖啡店）完整流程已实现于 `preview/stage-1-flow.html`，包含行进场景+转场动画+关底场景+通关动画。详见 [项目进度记录.md](../../项目进度记录.md)。
>
> 下方原方案内容保留作为设计参考（场景布局、瓦片构成、交互逻辑等仍然适用），但技术实现部分（Phaser API、Tilemap JSON、资源加载策略）已废弃。

## 一、技术选型（已废弃，保留参考）

### 1.1 引擎：Phaser.js 3

| 项 | 说明 |
|----|------|
| 版本 | Phaser 3.70+（CDN：`https://cdn.jsdelivr.net/npm/phaser@3.70.0/dist/phaser.min.js`） |
| 体积 | ~1.1MB（minified），gzip 后 ~350KB |
| 加载方式 | 开发期 CDN，最终交付 base64 内联到单文件 HTML |
| 渲染器 | WebGL 优先，Canvas 2D 回退（移动端兼容性好） |
| 选型理由 | ① 内置瓦片地图（Tilemap）渲染 ② 内置精灵动画、物理引擎、相机系统 ③ 社区成熟，文档完善 ④ 无依赖，单文件即可运行 |

**为什么不选纯 Canvas / PixiJS / Cocos Creator**：
- 纯 Canvas：需要手写瓦片渲染、碰撞、动画系统，工作量大
- PixiJS：仅渲染层，缺少游戏逻辑（输入、场景管理、物理）
- Cocos Creator：需 IDE 工程，不符合"单文件 H5"形态

### 1.2 素材来源

#### 已下载并验证可用的素材

| 素材 | 用途 | 来源 | 许可 | 状态 |
|------|------|------|------|------|
| 16x16 Indoor RPG Tileset (112x192, 84瓦片) | 室内场景：地面、墙、家具、装饰 | OpenGameArt / Redshrike | CC-BY 3.0 | ✅ 已下载可用 |

#### 暂未获取的素材

| 素材 | 用途 | 来源 | 状态 |
|------|------|------|------|
| Kenney Pixel Platformer | 瓦片+角色 | kenney.nl | ❌ 官网需浏览器交互下载，GitHub raw 被墙，ghproxy 代理失败 |
| Kenney Roguelike Pack | 室内家具 | kenney.nl | ❌ 同上 |
| AI 生成图片 | 背景图 | TRAE 图片生成 API | ❌ API 返回"image is generating"占位图，缓存问题 |

#### 角色精灵策略

由于无法下载角色素材图，**用 Phaser Graphics API 代码生成像素小人**：
- 32x32 帧，2帧待机 + 4帧行走 + 2帧端杯
- 多色层次（皮肤/衣服/裤子都有明暗）
- 带阴影、头发、衣领细节
- 调色板可控，FC 风格统一

#### 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| 用 OpenGameArt 真实 tileset + 代码生成角色 | ✅ 立即可用、零依赖、风格统一 | 角色精灵精细度有限 |
| 下载 Kenney 完整素材包 | 素材更丰富 | ❌ 网络限制无法获取 |
| 用 AI 生成背景图 | 视觉震撼 | ❌ API 不可用 |

**当前采用**：方案一（OpenGameArt + 代码生成角色）

#### 协议说明
- OpenGameArt 16x16 Indoor RPG Tileset：CC-BY 3.0，使用需署名作者 Redshrike
- 代码生成的角色精灵：自研，无协议限制
- 最终交付时需在页面底部添加 "Assets by Redshrike (CC-BY 3.0)"

## 五、与现有应用的集成方案

### 5.1 当前架构

```
index.html (主应用)
  ├── boot 启动屏
  ├── home 主页
  ├── dialog 剧情对话
  ├── map 地图+行进场景（Canvas绘制）
  ├── stage-detail 关卡详情（CSS绘制，待替换）
  ├── clear 通关动画
  └── share 分享页
```

### 5.2 集成方案

**方案：Phaser 嵌入 div 容器**

```javascript
// 在 stage-detail 页面创建 Phaser 实例
const phaserContainer = document.getElementById('phaser-stage');

const game = new Phaser.Game({
  parent: phaserContainer,
  width: 480,
  height: 270,
  zoom: 3,  // 放大到 1440×810
  pixelArt: true,  // 保持像素清晰
  scene: [CoffeeShopScene]  // 根据当前关卡选择场景
});
```

**场景选择逻辑**：
```javascript
const SCENE_MAP = {
  coffee: CoffeeShopScene,
  book: BookStoreScene,
  mall: MallScene,
  park: ParkScene
};

const currentStage = STAGES[stageIndex];
const SceneClass = SCENE_MAP[currentStage.sceneType];
```

### 5.3 状态机集成

```
map (行进场景)
  └─ arriveAtStage() → 切换到 stage-detail
       └─ stage-detail (Phaser 关底场景)
            └─ completeCheckin() → 切换到 clear
                 └─ clear (STAGE CLEAR 动画)
                      └─ nextStage() → 回到 map（下一关）
```

### 5.4 资源加载策略

**开发期**：从 `assets/` 目录加载
```javascript
this.load.image('coffee_tileset', 'assets/tilesets/coffee_shop.png');
this.load.tilemapTiledJSON('coffee_map', 'assets/maps/coffee_shop.json');
this.load.spritesheet('player', 'assets/sprites/player.png', { frameWidth: 32, frameHeight: 32 });
```

**最终交付**：base64 内联
```javascript
this.load.image('coffee_tileset', 'data:image/png;base64,iVBORw0KGgo...');
// JSON 直接内联为 JS 对象
const coffeeMapData = { width: 30, height: 17, ... };
this.cache.tilemap.add('coffee_map', { data: coffeeMapData });
```

## 七、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| Kenney 素材风格不统一 | 视觉割裂 | 统一调色板，用 Aseprite 二次调整 |
| Phaser 体积过大（1.1MB） | 加载慢 | 最终交付用 base64 内联，开发期 CDN |
| Tiled 学习成本 | 开发慢 | 先用代码生成简单地图，复杂场景再用 Tiled |
| 移动端性能 | 卡顿 | 限制帧率 30fps，减少粒子效果 |
| 素材版权问题 | 法律风险 | 仅用 CC0 素材，保留素材来源记录 |

## 八、验收标准

### 8.1 视觉验收

- [ ] 场景渲染清晰，瓦片无错位
- [ ] 像素风一致，无抗锯齿模糊
- [ ] 角色精灵动画流畅（8fps）
- [ ] 场景色调符合设计（咖啡店暖棕、书店深绿等）

### 8.2 交互验收

- [ ] 玩家可左右移动（键盘 A/D 或 方向键）
- [ ] 到达任务点显示提示
- [ ] 按空格触发任务动画
- [ ] 任务完成后播放 STAGE CLEAR 动画
- [ ] 点击继续可切换下一关

### 8.3 性能验收

- [ ] 首屏加载 < 3 秒（CDN）
- [ ] 运行帧率稳定 60fps（桌面）/ 30fps（移动端）
- [ ] 内存占用 < 100MB
