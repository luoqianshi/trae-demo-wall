## 目录职责

| 目录 | 用途 | 是否提交 |
|------|------|----------|
| `docs/` | 文档、设计资料、原始地图数据 | 是 |
| `preview/` | 效果预览页面（当前主成果 stages-preview + js/） | 是 |
| `项目进度记录.md` | 进度跟踪，新会话快速恢复 | 是 |
| `assets/` | 图片、音频、地图数据等静态素材 | 是 |
| `libs/` | 第三方库本地化副本（含 phaser.min.js，已弃用） | 是 |

## 技术栈说明

> ⚠️ **重要变更**：原方案选用 Phaser.js（见 FC关卡仿真方案.md），实际开发中改为**纯 Canvas 2D**。
> 原因：单文件交付体积考虑 + `file://` 协议兼容性。所有场景用手写瓦片渲染 + 精灵动画。
>
> **架构**：多个普通 `<script>` 标签加载的 JS 模块（**非 ES module**，因 `file://` 拦截），通过全局对象 `window.WM` 通信；路线数据内联（**不用 fetch**）。
