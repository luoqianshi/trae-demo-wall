# Weekend Missions · 周末密令 — UI/UX 视觉设计规范

> **文档版本：** v1.2  
> **最后更新：** 2026-06-22  
> **产品形态：** 离线 H5 单文件应用

---

## 1. 设计理念

整体设计遵循"掌上红白机"的核心理念——用户打开 H5 就像打开一台 NES 游戏机。所有游戏化界面严格遵循像素风美学，但分享卡片采用 3 套不同风格的模板（像素复古 / 现代简约 / 文艺清新），形成"游戏内像素、游戏外可定制"的反差体验。

**视觉优先原则：3秒内必须让评委感受到"这是一个有趣的NES风格游戏"。** 视觉的精致度直接决定创意可信度和评委第一印象分。

设计原则：一切交互都应该有"打游戏"的仪式感——按钮像 A/B 键，进度像血条，消费像金币，通关有 STAGE CLEAR。

---

## 1.1 视觉冲击力设计要点

### 开机仪式（Boot Sequence）
用户双击打开 HTML 的第一秒就开始视觉叙事，模拟真实 NES 开机的戏剧感：
- 黑屏 0.5秒（制造悬念）
- 屏幕"滚动"出现像素内容（模拟CRT启动）
- 标题 LOGO 闪烁登场
- PRESS START 闪烁邀请用户行动

### 情绪反差（Emotional Contrast）
- **游戏内**：纯 NES 像素风，CRT 扫描线，8-bit 动效，给用户"打游戏"的沉浸感
- **游戏外**：分享卡片 3 套模板覆盖不同审美，让用户愿意发朋友圈
- 这个反差本身是设计亮点，不是妥协

### 细节生命（Detail Alive）
静态像素元素要有"动起来的欲望"：
- 当前关卡的像素小人在节点上左右"巡逻"
- HP 降低时，像素角色从"精神"→"流汗"→"半跪"的状态图标变化
- 预算超支时整个地图屏幕轻微"红闪"
- 通关时金币不是冷冰冰的数字变化，而是"飞散"的粒子效果

### 情绪高潮（STAGE CLEAR）
通关动画是整个 Demo 的情绪顶点，必须做得比任何其他部分都华丽：
- 屏幕爆裂式出现"STAGE CLEAR!"
- 像素碎片飞散
- 数据像游戏得分表一样逐条点亮
- 评级字母 + 勋章出现
- 整个过程 5-8 秒，有开始有结束，像一场微型演出

| 场景 | 字体 | 大小 |
|------|------|------|

- 所有像素英文必须大写（PRESS START, STAGE CLEAR）

## 4. 像素风设计规则

### 4.1 像素网格

- 基础像素单元：4px × 4px
- 所有元素尺寸必须是 4 的倍数
- 边框宽度统一为 4px（NES 风格粗边框）

### 4.2 边框样式

**凸起边框（Raised）—— 按钮、卡片：**
```
border-color: #fff #555 #555 #fff;
border-width: 4px;
border-style: solid;
```

**凹陷边框（Inset）—— 输入框、内容区：**
```
border-color: #555 #fff #fff #555;
border-width: 4px;
border-style: solid;
```

### 4.3 图标规则

- 所有图标必须是像素风格，使用 16×16 或 32×32 像素网格
- 颜色限制在主色板范围内
- 不使用渐变、阴影、模糊
- 参考 NES 游戏图标的简洁程度

### 4.4 动画规则

- **禁止**平滑过渡（transition / ease）
- 使用**逐帧动画**（step-end / step-start）
- 闪烁文字：`animation: blink 1.2s step-end infinite`
- 扫描线效果：`repeating-linear-gradient(0deg, transparent 2px, rgba(0,0,0,0.1) 2px 4px)`
- 页面转场：直接切换，不用淡入淡出（模拟卡带加载）

### 4.5 离线 H5 适配规则

- 不使用需要联网的字体（除首次缓存）
- 不使用需要 WebGL 的 3D 效果
- 动画使用 CSS 而非 JS（性能优化）
- 适配 `file://` 协议（不能用 fetch 加载资源）

## 5. 核心组件库

### 5.1 像素按钮

**A 键按钮（主操作）：**
```css
background: #E74C3C;
color: #fff;
border: 4px solid;
border-color: #ff8a80 #8b0000 #8b0000 #ff8a80;
font-family: 'Press Start 2P';
font-size: 12px;
padding: 12px 24px;
/* 按下态：反转边框色 */
```

**B 键按钮（次要操作）：**
```css
background: #3B5998;
color: #fff;
border: 4px solid;
border-color: #6d8cc4 #1a2f5a #1a2f5a #6d8cc4;
```

**START 键（特殊行动）：**
- 长方形按钮，宽度 100%
- 背景 NES Yellow，黑色文字
- 带闪烁动画 "▶ PRESS START"

### 5.2 血条 / 金币条

**体力血条（HP Bar = 当日精力值）：**
```
[████████████░░░░] 75%
```
- 外框：NES 凹陷边框
- 满血（100%-61%）：NES Green `#27AE60`
- 中血（60%-31%）：NES Yellow `#F1C40F`
- 低血（30%-1%）：NES Red `#E74C3C`
- 每通过一关扣除对应精力值

**HP 状态图标（配合血条变化）：**
像素小人图标随 HP 状态变化，增加情感反馈：
```
HP 100%-61%: [🧍 像素小人不同时期不同状态]  ← 精神，站立
HP 60%-31%:  [😓 像素小人轻微低头 + 小汗滴] ← 疲惫
HP 30%-1%:   [😰 像素小人弯腰 + 大汗滴]     ← 危险
HP 0%:        [💀 像素小人在地上]            ← Game Over（本局不阻止通关）
```
状态切换时，图标做一次 `scale 1→1.2→1` 的弹跳动画。

**金币预算条：**
```
💰 [████████░░░░░░] ¥320/500
```
- 外框同上
- 填充色：NES Yellow `#F1C40F`
- 预算内：正常显示
- 超出预算时：金币条整体 **红闪 3 次**（每次 0.2s），配合屏幕边缘轻微红闪

**金币变化动画：**
每通过一关获得估算金币时，数字不是直接跳变，而是：
1. 当前数字短暂放大 + 高亮
2. 飞散出 5-8 个金色像素圆点
3. 新数字淡入
4. 效果：`+¥XX coins` 文字短暂飘出消失

### 5.3 对话框（NES Dialog Box）

```css
background: #000;
border: 4px solid;
border-color: #fff #555 #555 #fff;
padding: 16px;
/* 文字逐字显示，模拟打字机效果 */
/* 底部三角箭头闪烁提示"点击继续" */
```

- 文字逐字出现（每字 50ms 延迟）
- 底部有闪烁的 ▼ 三角提示继续
- AI 回复时顶部显示 "AI-GUIDE" 像素头像

### 5.4 关卡节点（Stage Node）

**6 种状态：**

| 状态 | 样式 | 说明 |
|------|------|------|
| 锁定 | 灰色虚线边框，内部 "?" | 未到达的后续关卡 |
| 当前 | Cyan 边框 + 脉冲动画 | 正在进行的关卡 |
| 已通关 | Green 填充，内部 "✓" | 已打卡的关卡 |
| BOSS | Red 边框，内部 "★"，尺寸更大 | 最终关卡（晚餐/压轴） |
| 隐藏关卡 | Purple 边框，内部 "?" | 彩蛋触发的隐藏关卡 |
| 彩蛋已发现 | 带 ⭐ 标记 | 隐藏关卡通关后 |

**关卡之间用虚线连接：**
```
[✓] - - - - [✓] - - - - [3] - - - - [4] - - - - [★]
```

**当前关卡像素小人动画（细节生命）：**
当前关卡节点上有一个 **16x16 像素小人在巡逻**：
```
动画循环（2秒）：
第0-0.5s: 小人从节点左边缘走到节点中心
第0.5-1s: 小人从节点中心走到节点右边缘
第1-1.5s: 小人从节点右边缘走回节点中心
第1.5-2s: 小人从节点中心走到节点左边缘（循环）
```
像素小人颜色：NES Cyan `#00D2D3`，带简单的 2-3 帧走路循环。

**节点尺寸规范：**
- 普通节点：48×48 px（含内边距）
- BOSS 节点：64×64 px（更大，有压迫感）
- 隐藏关卡节点：48×48 px，Purple 边框闪烁

**已通关节点通关标记动画：**
关卡通关时，节点上短暂出现一个绿色 "✓" 弹出动画（从节点中心弹出缩小回原点）：
```
弹出 scale 1→1.5→0，200ms
```

评委在桌面端体验时，如果页面只是"放大的手机界面"，会失去游戏感。需要在桌面端模拟"NES 主机屏幕"的氛围。

```

│  ║                                                   ║  │

│  ║  │                                             │ ║  │

│  ║  └─────────────────────────────────────────────┘ ║  │

        ↑                                              ↑
   像素装饰边框（左）                          像素装饰边框（右）
```

**视觉规范：**
- 两侧像素装饰边框：深色背景 + 像素图案（重复的 16×16 小砖块）
- "NES 主机屏幕"边框：凸起像素边框效果
- 屏幕顶部有像素"天线"装饰
- 底部提示："Insert Coin to Continue"（桌面端固定显示）
- 整体模拟 4:3 或 16:9 的老式 CRT 屏幕比例

### 9.3 桌面端交互规范

| 场景 | 处理方式 |
|------|----------|
| 鼠标悬停像素按钮 | 按钮边框色反转（模拟 NES 手柄按下） |

| 滚动条 | 隐藏，使用内部滚动（如需要） |
| 右键菜单 | 禁用，防止干扰游戏感 |
