# Weekend Missions · 周末密令 — 交互流程文档

> **文档版本：** v1.2  
> **最后更新：** 2026-06-23  
> **产品形态：** 离线 H5 单文件应用

---

> ⚠️ **2026-06-23 更新：第一关实际流程已实现**
>
> 第一关完整流程已在 `preview/stage-1-flow.html` 实现，与下方原设计有以下差异：
>
> | 环节 | 原设计 | 实际实现 |
> |------|--------|----------|
> | 关卡进入 | 点击关卡节点→详情页→签到 | 点击「▶ 出发」→行进场景自动播放 |
> | 行进表现 | 地图页静态 | 上半地图+下半像素横版滚动，小人走向终点旗 |
> | 到达关底 | 详情页签到按钮 | 转场动画（黑条合拢+任务文字+展开） |
> | 关底交互 | 签到按钮 | 自动移动到柜台→弹窗确认点单（Space/点击） |
> | 通关动画 | 简单小动画 | 闪白→黑边合拢→STAGE CLEAR→任务回顾 |
>
> **后续关卡**建议沿用 stage-1-flow 的流程结构。详见 [项目进度记录.md](../项目进度记录.md)。

## 1. 页面架构总览

```
H5 单页应用
├── 首页（Title Screen）
│   ├── 主题卡列表（深圳/杭州/上海）
```

## 2. 核心用户流程

### 2.1 主线流程：从打开到通关

```
用户双击打开 index.html
    │
    ▼
首页（Title Screen）
    │ 显示 3 张主题卡 + 演示模式开关
    │ （如有存档）显示 "继续上局" 按钮
    │
    ├─ [继续上局] → 恢复存档 → 跳转到密令地图页
    │
    └─ [选择主题卡]（点击任一主题）
            │
            ▼
    主题确认页
            │ AI 指挥官逐字发言（5-8 秒）
            │ "特工，本次任务：深圳·情侣甜蜜大作战"
            │ "作战时间：今日 10:00 - 19:00"
            │ "作战预算：¥200"
            │
            │ 加载动画（2-3 秒）
            │ "正在扫描区域情报..."
            │ "规划潜入路线..."
            │ "密令生成完毕！"
            │
            │ （可点击"跳过"直接进入地图）
            │
            ▼
    密令地图页
            │ 5 个关卡节点横向排列
            │ 第 1 关为"当前"（Cyan 脉冲）
            │ HP 100%，金币 0/¥200
            │
            │ [演示模式开启] 可点击任意关卡直接进入
            │
            │ 点击"当前"关卡
            ▼
    关卡详情页
            │ STAGE 1 标题
            │ 地点名 + 地址 + 活动描述
            │ AI 发现笔记（打字机效果逐字显示）
            │
            │ 3 个按钮：
            │   [🔄 换一个] → 重新生成当前关卡
            │   [✅ 已到达·签到] → 完成本关
            │   [返回地图] → 回到地图页
            │
            ├─ [签到] → 通关小动画 → 返回地图（解锁下一关）
            │
            └─ [换一个] → "AI 重新规划" 过渡动画 → 显示新关卡
            │
            ▼
    关卡通关判定
            │
            ├─ 是第 2 关通关？
            │   │ 是 → 触发彩蛋弹窗
            │   │      ├─ [接受] → 隐藏关卡插入地图
            │   │      └─ [跳过] → 继续下一关
            │   │
            │   └─ 否 → 继续
            │
            ├─ 体力值 -20%
            ├─ 金币 +预估花费
            ├─ HP 条 / 金币条动画更新
            │
            ├─ 还有下一关？
            │   │ 是 → 下一关变"当前"，回到地图
            │   │
            │   └─ 否（最后一关）
            │           │
            │           ▼
    STAGE CLEAR 动画（5-8 秒）
            │ "STAGE CLEAR!" 弹入
            │ "ALL STAGES CLEARED!" + 烟花
            │ 冒险标题 + CONGRATULATIONS
            │
            │ （可点击"跳过"进入总结）
            │
            ▼
    冒险总结页
            │ 显示：冒险名 / 通关数 / 步数 / 耗时 / 花费 / 评级
            │
            ├─ [生成分享卡片] → 分享卡片页
            └─ [再来一局] → 首页
```

### 2.2 主题卡选择流程

```
首页加载
    │
    ▼
渲染 3 张主题卡（深圳/杭州/上海）
    │ 每张卡片显示：城市标签、主题名、缩略插画、人数/预算/风格
    │
    │ 用户点击
    ▼
卡片翻转/高亮动画（200ms）
    │
    ▼
navigate('dialog', { theme: selectedTheme })
    │
    ▼
进入 AI 对话页
```

### 2.3 AI 对话流程

```
主题确认页打开
    │
    ▼
[加载主题信息]
    │ 主题标题、参数、缩略图
    │
    ▼
[AI 指挥官发言]（打字机效果，每字 50ms）
    │ "特工，密令已下达。"
    │ "本次任务：深圳·情侣甜蜜大作战"
    │ "作战时间：今日 10:00 - 19:00"
    │ "作战预算：¥200"
    │ "作战人数：2 人"
    │ "任务：穿越南山区，打卡 5 个甜蜜坐标。"
    │
    │ （每句之间停顿 800ms）
    │
    ▼
[加载动画阶段]（2-3 秒）
    │ 文字逐行出现：
    │ "▶ 正在扫描区域情报..."
    │ "▶ 规划潜入路线..."
    │ "▶ 密令生成完毕！"
    │
    │ 同时显示像素小人走路动画
    │
    ▼
[1.5秒延迟] 后自动跳转到密令地图页
    │
    │ （可点击"跳过动画"直接进入）
    ▼
密令地图页
```

### 2.4 关卡推进流程

```
用户点击"当前"关卡节点
    │
    ▼
关卡详情页加载
    │
    │ 显示关卡基本信息（静态）
    │
    ▼
[AI 发现笔记] 打字机效果逐字显示
    │ "推开玻璃门，海风混着书页的香气扑面而来..."
    │ "这里不是网红打卡点，更像是一个被驯服的、可以对话的邻居。"
    │
    │ 50ms / 字
    │
    ▼
[按钮区域] 3 个按钮显示
    │
    ├─ [🔄 换一个]
    │   │ 点击 → 1 秒过渡动画
    │   │ 节点闪烁 + "AI 重新规划路线..."
    │   │ → 新关卡淡入
    │   │ → 重新显示发现笔记
    │   │
    │   │ 限制：同一关卡最多换 3 次
    │   │
    │   └─ 新关卡的 replaceCount + 1
    │
    ├─ [✅ 已到达·签到]
    │   │ 点击 → 弹出确认"确认已到达【XX】？"
    │   │
    │   ├─ [取消] → 留在详情页
    │   └─ [确认] → 触发通关逻辑
    │           │
    │           ▼
    │       通关小动画（1.5 秒）
    │       │ 像素小人跳跃 + STAGE X CLEAR! 文字
    │       │ 金币飞溅 + "+XX coins"
    │       │
    │       ├─ 是第 2 关？
    │       │   └─ 是 → 触发彩蛋
    │       │
    │       ├─ HP -20%（如果有）
    │       ├─ 预算 +花费
    │       ├─ 状态：当前关 → 已通关
    │       ├─ 下一关：锁定 → 当前
    │       │
    │       ├─ 还有下一关？
    │       │   │ 是 → 自动跳回地图页
    │       │   └─ 否 → 触发 STAGE CLEAR
    │       │
    │       └─ 状态保存到 localStorage
    │
    └─ [返回地图]
        └─ 回到密令地图页
```

### 2.5 彩蛋触发流程

```
第 2 关通关完成
    │
    ▼
通关小动画结束
    │
    ▼
[彩蛋触发]（约 0.5 秒延迟）
    │
    │ 屏幕闪烁 0.3 秒
    │ 警报音（Web Audio API）
    │
    ▼
[彩蛋弹窗] 弹出
    │
    │ 顶部："! 发现隐藏密令 !"
    │ 中部：隐藏关卡信息
    │   - 地点：老陈红烧肉（隐藏关）
    │   - 任务：向老板点一份招牌红烧肉
    │   - 预计花费：¥80
    │
    │ 按钮：
    │   [接受挑战]   [跳过]
    │
    ├─ [接受挑战]
    │   │ 地图更新：插入 "?" 隐藏关卡节点
    │   │ 隐藏关卡变为"当前"状态
    │   │ 继续冒险
    │   │
    │   └─ 隐藏关卡通关后，地图显示 ⭐ 标记
    │
    └─ [跳过]
        │ 不影响主线通关判定
        │ 继续下一关
        │
        └─ 隐藏关卡永远消失
```

```
        │ "✓ 可一键通关"
        │ "✓ 可切换主题"

        │ 关卡详情页：增加"跳到此关"按钮

        │
        ├─ [一键通关]
        │   │ 全部关卡 + 隐藏关卡标记为已通关
        │   │ 直接跳转到 STAGE CLEAR
        │
        ├─ [跳到此关]
```

### 2.8 异常与边界场景处理

| 场景 | 处理方式 |
|------|----------|
| 浏览器禁用 localStorage | 降级为内存存档，提示"无法保存进度，刷新会丢失" |
| 字体加载失败 | 降级为 monospace 字体，UI 仍有 NES 风格（粗体+大写） |
| 浏览器不支持 Canvas | 分享卡片页提示"请使用现代浏览器" |
| 用户在加载中刷新页面 | 加载页重置，回到首页 |
| 演示模式与存档冲突 | 演示模式进度不写入存档 |
| AI 打字效果中点击跳过 | 立即结束打字，进入下一步 |
| 关卡"换一个"用完 3 次 | 按钮置灰，提示"AI 已尽力，换个心情试试？" |
| 同一冒险多次刷新 | 检测到 localStorage 存档，首页显示"继续上局" |
| 用户直接关闭页面 | localStorage 已保存，下次打开可恢复 |
