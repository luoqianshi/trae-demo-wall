# 【生活娱乐赛道】周末密令 Weekend Missions —— 把你的周末城市探索变成一场 FC 红白机闯关冒险

> 一款离线可运行的 H5 单文件应用，消灭周末决策瘫痪，让 AI 指挥官给你下达城市冒险任务。

## 一、Demo 简介

**产品形态：** 离线 H5 单页应用（单 HTML 文件，双击即可运行，无需服务器、无需注册）

**面向谁：** 20-35 岁都市年轻人，尤其是每逢周末陷入"不知道去哪"决策瘫痪的情侣、朋友聚会群体，以及想出门但懒得做攻略的独居青年。

**主要功能：**

| 功能 | 说明 |
|------|------|
| **AI 指挥官下达密令** | 选择主题后，AI 以"特工指挥官"身份逐字打字下发任务简报，加载动画模拟 AI 生成过程 |
| **FC 像素风密令地图** | 经典 NES 风格关卡地图，含 HP 条、金币条、5 个关卡节点，小人自动巡逻 |
| **行进与打卡** | 点击出发后，像素小人在真实地图底图上行走，到达后触发关底交互（如咖啡店点单弹窗） |
| **通关动画与评级** | 全部关卡通关后触发 STAGE CLEAR 动画，像素碎片飞散，数据逐条点亮，获得 S/A/B/C 评级 |
| **3 套分享卡片** | 通关后生成分享卡片，支持像素复古、现代简约、文艺清新三种风格，一键下载 PNG |
| **演示模式** | 可一键跳关、一键通关、切换主题，方便评委快速体验完整流程 |

## 二、Demo 创作思路

### 灵感来源

每周末早上，我和女朋友都会陷入同样的困境：打开大众点评刷了 20 分钟，最后她说"随便"，我们又去了同一家商场。四个人的兄弟群从周四讨论到周六，还是没决定去哪。这种"周末决策瘫痪"不只是我们的问题——小红书、大众点评解决的是"信息检索"，但没有人解决"决策娱乐化"。

如果把"制定周末计划"本身变成一场游戏呢？让 AI 充当"任务指挥官"，把城市探索包装成 FC 闯关冒险，用户不需要做攻略——他们只需要接受任务、出发、通关。

### 想解决的问题

- **决策瘫痪**：30 秒内获得一份完整的周末计划，而非刷 30 分钟信息流
- **众口难调**：AI 生成方案后，情侣/朋友只需要"跟着任务走"，减少协商摩擦
- **缺乏仪式感**：线下打卡被包装成"关卡签到"，通关动画和分享卡片让周末变得有纪念意义

### 为什么做这个方向

现有产品空白：
- 小红书/大众点评 = 信息流搜索
- 去哪儿/携程 = 旅行规划工具
- 各类打卡 App = 被动签到工具

没有产品将 **AI 生成 + 游戏化叙事 + 线下探索** 三者融合。而 TRAE 的 AI 能力恰好可以承担"指挥官"角色，让技术有了情感出口。

1. 下载附件中的 `stages-preview.html` 和 `js/` 目录
2. 双击 `stages-preview.html` 即可在浏览器中直接运行（无需服务器，无外部依赖）

**在线预览：**

> 待部署后补充链接（如已部署请替换此处）

**体验路径推荐（3 分钟完整流程）：**

1. 双击打开 HTML → 观看 FC 开机仪式 → 选择【深圳·情侣甜蜜大作战】
2. 阅读 AI 指挥官打字下发的任务简报 → 等待"密令生成"加载动画
3. 进入 FC 像素风地图 → 点击第 1 关「▶ 出发」
4. 观看行进动画（像素小人沿真实地图路线走向咖啡店）
5. 到达后进入关底交互：在像素咖啡店场景中移动角色、触发点单弹窗
6. 触发 STAGE CLEAR 通关动画 → 查看冒险总结与评级
7. 选择分享卡片模板 → 下载 PNG

**演示模式快捷键：** 页面右下角开关可一键跳关 / 切换主题 / 一键通关

## 四、TRAE 实践过程

### 开发完整流程

本项目从 0 到可运行 Demo，完全由 TRAE 驱动开发。以下是关键开发阶段：

#### Stage 1：创意构思与产品定义（TRAE Work）

在 TRAE Work 中，我用自然语言与 AI 反复迭代产品概念：
- 最初想法只是一个"周末计划生成器"
- 经过多轮对话，AI 建议将"决策"包装成"FC 游戏任务"，并引入"AI 指挥官"角色
- 最终确定"掌上红白机"的核心视觉方向，以及 7 页应用架构（boot → home → dialog → map → stage → clear → share）

**【截图占位 5：TRAE Work 中创意构思对话截图】**

#### Stage 2：技术架构与开发实现（TRAE IDE）

在 TRAE IDE 中，我要求 AI 将设计文档转化为可运行代码。关键技术决策包括：

| 决策 | TRAE 建议 | 实际效果 |
|------|----------|----------|
| 渲染方案 | 原计划用 Phaser.js，TRAE 建议改为纯 Canvas 2D | 单文件体积更小，`file://` 协议兼容性更好 |
| 模块化 | TRAE 提醒 `file://` 下 ES module 会被浏览器拦截 | 改用普通 `<script>` 标签 + `window.WM` 全局对象通信 |
| 地图数据 | 将 GPS 路线 JSON 内联到 JS 中（而非 fetch） | 完全离线可用，无需网络 |
| 视觉风格 | 手写瓦片纹理 + CRT 扫描线效果 | 第一秒就让评委感受到 FC 游戏氛围 |

核心代码全部由 TRAE 逐模块生成，包括：
- `core.js`：全局状态机 + 主渲染循环 + 调色板 + 输入分发
- `textures.js`：程序化生成像素纹理（草地、道路、建筑）
- `map.js`：真实地图底图 cover 填充 + GPS 路线投影
- `flow.js`：7 状态机（intro → walk → transition → ending → clear → finale）
- `scenes/endings/`：4 个关底场景（咖啡店/书店/商场/公园 BOSS）

### 开发心得

- **"不追求完美，追求完整"**：Demo 的核心不是每个像素都完美，而是让评委 3 分钟内体验到"从选择主题到通关分享"的完整闭环
- **"用视觉征服第一秒"**：FC 像素风 + 开机仪式动画的设计，让评委在打开 HTML 的第一眼就感受到差异化
- **"AI 是创意放大器，不是替代品"**：TRAE 帮我快速搭建了技术骨架，但"AI 指挥官"的角色设定、"发现笔记"的文案、每个关底场景的情感设计，都来自真实的用户洞察
