## 1. 产品概述

### 1.1 背景与机会

**痛点洞察：** 20-35 岁的都市年轻人每逢周末都面临"周末决策瘫痪"——打开大众点评不知道去哪，刷小红书攻略耗时费力，最终在家躺平后又感到浪费。情侣、朋友聚会更是众口难调，光是"去哪儿"就能讨论半小时。

**机会判断：** 现有本地生活平台解决的是"信息检索"问题，但没有人解决"决策娱乐化"问题。如果将"制定周末计划"本身变成一场游戏，让 AI 充当"任务指挥官"，把城市探索包装成复古 FC/NES 闯关冒险，用户不再需要做攻略——他们只需要接受任务、出发、通关。

**竞品空白：** 小红书/大众点评 = 信息流搜索；去哪儿/携程 = 旅行规划工具；各类打卡 App = 被动签到工具。没有产品将 **AI 生成 + 游戏化叙事 + 线下探索** 三者融合。

### 1.2 产品定义

**一句话描述：** 一款离线可运行的 H5 应用，把你的周末城市探索变成一场 FC 红白机风格的闯关冒险。

**核心体验循环：**

```
告诉 AI 你的城市/心情/预算
        ↓
AI 生成"密令地图"（4-5 个关卡 = 4-5 个地点）
        ↓
按顺序逐关推进（到达地点 → 签到打卡 → 触发事件）
        ↓
途中可能触发隐藏彩蛋（隐藏关卡/限时事件/附近秘境）
        ↓
全部通关 → STAGE CLEAR! 经典动画 + 精美分享卡片
        ↓
分享 → 社交传播 → 新用户进入
```

### 1.3 产品目标

| 维度 | 目标 | 衡量指标 |
|------|------|----------|
| **用户价值** | 消灭周末决策瘫痪，30 秒获得一份有趣的周末计划 | 从打开小程序到获得密令地图 < 60 秒 |
| **情感价值** | 让用户感到"周末过得像一场冒险"而非"又一天刷手机" | 通关后分享率 > 30% |
| **传播价值** | STAGE CLEAR 分享卡片自带传播力 | 分享卡片点击回流率 > 15% |
| **Demo 目标** | 完成核心循环可体验版本，展示 AI + 游戏化创新 | 7/15 前可演示完整流程 |

### 1.4 Demo 范围定义

**Demo 包含（In Scope）：**
- 主题卡选择 → 模拟 AI 对话 → 加载动画 → 展示密令地图
- 3 套预置主题关卡（深圳·情侣/杭州·独居/上海·朋友），可切换体验
- 像素风关卡地图（含 HP 条 / 金币条 / 5 个关卡节点）
- 关卡推进与签到（手动点击签到，模拟"到达"动作）
- "换一个"重新生成当前关卡（含过渡动画）
- 彩蛋触发系统（第 2 关后必触发隐藏关卡）
- STAGE CLEAR 通关动画 + 完整数据统计
- 3 套分享卡片模板（像素复古 / 现代简约 / 文艺清新），可切换并下载 PNG
- 演示模式开关（可跳关、一键通关、切换主题）

**Demo 不含（Out of Scope）：**
- 用户账号体系 / 微信授权登录（评委无门槛进入）
- 真实 AI API 调用（预置数据 + 模拟打字效果保留 AI 概念）
- 真实 GPS 定位签到（手动点击模拟"到达"）
- 社交分享到微信（生成图片供评委下载或截图）
- 好友列表 / 排行榜
- 冒险档案与成就系统（P1 复赛迭代）
- 支付 / 商业化模块

### 1.5 关键假设与风险

| 假设 | 风险 | 验证方式 |
|------|------|----------|
| 预置主题卡足以传达"AI生成"概念 | 评委可能觉得不是真AI | 打字机效果 + "AI生成中"加载动画 + 演示说明强调"Demo阶段预置，正式版接入AI" |
| 手动签到能模拟"到达"的仪式感 | 比真实定位缺少沉浸感 | 加签到过渡动画 + 像素反馈效果 + "演示模式"提示 |
| FC 像素风对目标用户有吸引力 | 部分用户可能觉得"粗糙" | 分享卡片用现代设计补偿，像素风只用于游戏界面 |
| 单文件离线运行够流畅 | 大量像素动画可能卡顿 | 限制动画数量，使用 CSS transform 而非 JS 动画 |

## 2. 目标用户与场景

### 2.1 用户画像

#### 画像 A：小林（25 岁，程序员，有女朋友）

- **周末痛点：** 每周六早上女朋友问"今天去哪"，他打开大众点评刷了 20 分钟，女朋友说"随便"，最后又去了同一家商场。
- **对产品的期待：** 给我一个"任务"，我照着做就行，别让我选。最好有趣一点，能给女朋友一个惊喜。
- **使用场景：** 周六上午 10 点，打开小程序 → 输入"深圳南山，情侣约会，200 块以内" → AI 生成 5 关密令 → 带女朋友出发。
- **传播动机：** 通关后的分享卡片很好看，发朋友圈有面子。

#### 画像 B：阿杰（28 岁，设计师，周末和兄弟聚会）

- **周末痛点：** 四个人的群从周四开始讨论周末干嘛，到周六还没结论。
- **对产品的期待：** 丢一个"密令"到群里，大家一起闯关，比单纯吃饭唱歌有意思。
- **使用场景：** 周五晚上在群里说"我发了个周末密令，明天一起闯" → 周六四人按路线闯关。
- **传播动机：** 通关截图发到群里，@没来的朋友"你错过了"。

#### 画像 C：小鱼（23 岁，运营，独居，喜欢拍照）

- **周末痛点：** 一个人周末不知道去哪，但又不想宅家。想找小众的地方但懒得做攻略。
- **对产品的期待：** AI 能推荐一些她不知道的小店/角落，有惊喜感。
- **使用场景：** 周六下午，输入"广州天河，一个人安静逛逛，100 块" → 获得一条文艺路线 → 一个人慢慢走。
- **传播动机：** 分享卡片 + 自己拍的照片一起发小红书。

### 2.2 核心使用场景

| 场景 | 触发时机 | 用户状态 | 产品角色 |
|------|----------|----------|----------|
| **情侣周末约会** | 周六/周日上午 | 不知道去哪、不想做攻略 | "约会任务指挥官" |
| **朋友聚会破冰** | 周五晚-周六 | 群聊讨论无结论 | "聚会冒险发起者" |
| **独自城市探索** | 周末任意时间 | 想出门但没目的地 | "私人探索向导" |
| **来朋友的城市玩** | 周末/小长假 | 不想去游客景点 | "在地化冒险设计师" |
| **纪念日/生日惊喜** | 特殊日期 | 想策划一个有趣的安排 | "惊喜任务策划师" |

### 2.3 用户旅程地图

```
[发现] 朋友圈看到分享卡片 / 朋友群里看到密令链接
  ↓
[进入] 扫码/点击 → 进入小程序 → 微信授权登录
  ↓
[输入] 看到像素风对话框 → 用自然语言描述需求（城市、心情、预算、人数）
  ↓
[生成] AI "指挥官"回复 → 像素加载动画 → 密令地图展开（4-5 个关卡）
  ↓
[出发] 查看第一关信息 → 点击"出发" → 导航/地图引导前往
  ↓
[闯关] 到达地点 → 签到（定位/手动）→ 解锁下一关 → 途中可能触发彩蛋
  ↓
[通关] 最后一关签到 → STAGE CLEAR! 动画 → 冒险总结 → 生成分享卡片
  ↓
[分享] 保存卡片/分享到朋友圈/发给好友 → 社交裂变
```

## 4. P0 功能详细需求

---

### F-01：主题卡选择

#### 用户故事

> 作为一个周末不知道干嘛的年轻人，我希望快速选一个符合我心情的预置主题，而不是从零描述需求，这样我可以 10 秒内开始一次冒险。

#### 功能描述

首页展示 3 张像素风主题卡片，每张卡片代表一种周末场景：
- **深圳·情侣甜蜜大作战**（推荐）— 浪漫主题、2 人、¥200
- **杭州·独居时光漫步** — 治愈主题、1 人、¥150
- **上海·兄弟聚会闯关** — 美食主题、4 人、¥500

每张卡片含：城市标签、主题名称、缩略插画、核心参数（人数 / 预算 / 风格）。

#### 交互流程

```
[首页 Title Screen]
  ↓
[3张主题卡] 并排展示（移动端纵向）
  ↓
[用户点击主题卡] → 卡片翻转动画 → 进入 F-02
  ↓
[演示模式] 额外提供"随机主题"按钮，可随机抽取
```

#### 验收标准

| 编号 | 验收条件 | 验证方式 |
|------|----------|----------|
| AC-01 | 首页展示 3 张主题卡，视觉风格统一且有差异化色彩 | UI 测试 |
| AC-02 | 点击主题卡有"已选中"反馈（翻转/高亮/缩放） | 交互测试 |
| AC-03 | 演示模式下"随机主题"按钮可循环切换 | 功能测试 |
| AC-04 | 移动端纵向排列，桌面端并排排列 | 响应式测试 |

### F-02：模拟 AI 对话

#### 用户故事

> 作为一个选择了主题的用户，我希望看到 AI 指挥官"现场为我规划"的过程，让我感觉这局游戏是专属为我定制的，而不是套模板。

#### 功能描述

主题卡选定后，进入 AI 对话页。AI "指挥官"以像素 NPC 形象出现，逐字打字给出任务简报。配合加载动画和"AI生成中"提示，营造真实 AI 生成的氛围。Demo 阶段内容是预置的，但通过打字效果保留"AI 在工作"的体验感。

#### 交互流程

```
[主题卡选定] → 进入对话页
  ↓
[AI 指挥官发言]（打字机效果，每字 50ms）：
  "特工，密令已下达。"
  "本次任务：深圳·情侣甜蜜大作战"
  "作战时间：今日 10:00 - 19:00"
  "作战预算：¥200"
  ↓
[加载动画] 像素小人走路动画 + 文字滚动：
  "正在扫描区域情报..."
  "规划潜入路线..."
  "密令生成完毕！"
  ↓
[3秒后] 自动跳转到密令地图（F-03）
```

#### 验收标准

| 编号 | 验收条件 | 验证方式 |
|------|----------|----------|
| AC-01 | AI 指挥官文字逐字显示（打字机效果） | 体验测试 |
| AC-02 | 加载动画至少展示 2 个阶段的文字 | 流程测试 |
| AC-03 | 加载完成后 1-2 秒自动跳转（也可点击跳过） | 交互测试 |
| AC-04 | 对话页可"跳过动画"直接进入地图 | 兜底测试 |
| AC-05 | 不同主题卡的 AI 文案不同 | 内容测试 |

#### 边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户在加载中点击返回 | 提示"指挥官正在生成密令，确认离开？" |
| 加载动画卡死 | 提供"跳过"按钮，超时 10 秒后自动跳转 |

### F-03：像素密令地图

#### 用户故事

> 作为一个收到密令的冒险者，我希望看到一张 FC 红白机风格的关卡地图，能清楚看到今天要去的所有地点和路线，这样我会感到兴奋并知道该从哪里开始。

#### 功能描述

AI 生成密令后，展示一张全屏的像素风格地图。地图以俯视/斜 45 度视角呈现，5 个关卡以 FC 游戏关卡节点的形式排列在地图上，用虚线/路径连接。当前关卡高亮闪烁，已通关关卡显示旗帜，未解锁关卡显示问号/锁。

#### 视觉规范

**配色方案（NES 经典调色板）：**
- 主色调：`#000000`（黑）、`#FCFCFC`（白）
- 强调色：`#F83800`（红）、`#00A800`（绿）、`#0058F8`（蓝）、`#F8B800`（金）
- 地图底色：`#004000`（深绿草地）、`#3CBCFC`（浅蓝水面）

**元素设计：**
- 玩家角色：16x16 像素小精灵，站在当前关卡位置
- 关卡节点：32x32 像素图标（城堡、商店、公园等），按地点类型匹配
- 路径：像素虚线连接各节点，已通关段变为实线
- 地图标题：像素字体显示冒险名称
- HP 条（体力值）：顶部红色血条，初始 100%，每过一关扣 20%（纯装饰）
- 金币条（预算）：顶部金色币条，显示预算剩余
- START 按钮：闪烁的 "PRESS START" 文字，点击开始第一关

#### 动态效果

- **HP 条变化**：每通关一关，HP 减少 20%，同时播放"小人擦汗"动画
- **金币条变化**：每通关一关，金币增加对应估算花费，播放"金币飞散"动画
- **超预算提示**：当金币累计超过预算上限时，金币条变红闪烁
- **彩蛋标记**：触发彩蛋的关卡节点出现 ⭐ 标记

#### 交互说明

| 操作 | 行为 |
|------|------|
| 进入地图 | 播放"地图展开"动画（从中心向四周扩散）+ 8-bit 音效 |
| 点击当前关卡 | 弹出关卡详情面板（地点名、活动、距离、预计花费） |
| 点击未解锁关卡 | 显示 "??? LOCKED" + 像素锁图标 |
| 点击已通关关卡 | 显示通关标记 + 回忆缩略图（如果有拍照） |
| 双指缩放 | 地图放大/缩小（像素风保持） |
| 滑动 | 平移地图视角 |

#### 验收标准

| 编号 | 验收条件 | 验证方式 |
|------|----------|----------|

### F-06：彩蛋触发系统

#### 用户故事

> 作为一个正在闯关的冒险者，我希望在闯关过程中能遇到意料之外的惊喜（隐藏彩蛋），这样我会觉得这次冒险真的像一场有秘密的游戏，而不是简单的导航。

#### 功能描述

在关卡推进过程中，用户可能触发隐藏彩蛋。彩蛋是密令地图上没有标注的惊喜内容，增加探索的不可预测性和趣味性。Demo 阶段保证第 2 关后必触发 1 个隐藏关卡，确保演示效果。

#### 彩蛋类型（Demo 阶段实现）

| 类型 | 触发条件 | 表现形式 |
|------|----------|----------|
| **隐藏关卡** | 第 2 关通关后自动触发 | 地图上突然出现一个未标注的 "?" 节点 |

#### 隐藏关卡详细设计

**触发逻辑：**
- 第 2 关通关后，系统自动插入一个隐藏关卡
- 隐藏关卡是预置数据中的"彩蛋关卡"

**触发表现：**
1. 屏幕突然闪烁 + "警报"音效
2. 弹出像素风弹窗："！发现隐藏密令！"
3. 显示隐藏关卡信息（地点、任务）
4. 用户选择"接受挑战"或"跳过"
5. 接受 → 地图更新，插入隐藏关卡节点（标记为 "?" 图标）
6. 跳过 → 继续原路线，不影响通关判定

#### 验收标准

| 编号 | 验收条件 | 验证方式 |
|------|----------|----------|
| AC-01 | 第 2 关通关后必触发隐藏关卡 | 流程测试 |
| AC-02 | 触发时有明显的视觉/音效提示（屏幕闪烁+警报音） | 体验测试 |
| AC-03 | 用户可选择接受或跳过隐藏关卡 | 交互测试 |
| AC-04 | 接受隐藏关卡后地图正确更新（增加一个 "?" 节点） | 状态测试 |
| AC-05 | 跳过隐藏关卡不影响主线通关判定 | 逻辑测试 |
| AC-06 | 隐藏关卡的通关也会计入冒险总结 | 数据验证 |

#### 边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户快速跳过 | 尊重用户选择，不强制 |
| 演示模式 | 彩蛋默认开启，可一键关闭 |

## 5. 数据埋点方案

### 5.1 埋点原则

Demo 阶段为离线 H5 应用，**无服务端**，所以采用纯前端统计方案：

1. **本地事件队列：** 所有事件记录到 `localStorage` 的 `wm_events` 数组
2. **控制台输出：** 开发模式下事件实时输出到 `console`，便于测试
3. **演示数据导出：** 提供"导出统计数据"按钮，可生成 JSON 文件
4. **关键转化：** 重点追踪核心漏斗的每一步转化率

### 5.2 事件定义表

| 事件ID | 事件名称 | 触发时机 | 关键参数 | 优先级 |
|--------|----------|----------|----------|--------|
| E-001 | `app_launch` | H5 页面打开 | is_new_user, referrer | P0 |
| E-002 | `theme_select` | 主题卡选择 | theme_id, theme_name | P0 |
| E-003 | `dialog_start` | AI 对话界面展示 | theme_id | P0 |
| E-004 | `ai_typing_start` | AI 打字效果开始 | theme_id | P0 |
| E-005 | `ai_typing_complete` | AI 打字效果完成 | duration_ms | P0 |
| E-006 | `map_view` | 密令地图页展示 | theme_id, stage_count | P0 |
| E-007 | `stage_detail_view` | 查看关卡详情 | stage_number, stage_type | P0 |
| E-008 | `checkin_click` | 点击"已到达·签到" | stage_number | P0 |
| E-009 | `checkin_success` | 签到成功 | stage_number, time_spent_sec | P0 |
| E-010 | `stage_clear` | 单关通关动画播放 | stage_number, is_hidden_stage | P0 |
| E-011 | `easter_egg_trigger` | 彩蛋触发 | egg_type, stage_number | P0 |
| E-012 | `easter_egg_accept` | 用户接受彩蛋挑战 | egg_type | P0 |
| E-013 | `easter_egg_skip` | 用户跳过彩蛋 | egg_type | P0 |

### 5.3 核心漏斗定义

```
  → checkin_success (stage 1) → ... → checkin_success (stage N)
  → all_clear → summary_view → share_card_generate → share_card_download
```

**关键转化指标：**

| 转化环节 | 目标转化率 | 意义 |

| 进入地图 → 第一关签到 | > 50% | 用户是否真的出发 |
| 第一关签到 → 全部通关 | > 60% | 路线质量与用户粘性 |
| 全部通关 → 生成分享卡 | > 80% | 分享意愿 |
| 生成分享卡 → 下载PNG | > 60% | 卡片质量 |

### B. 参考竞品分析

| 产品 | 类型 | 共同点 | 差异点 |
|------|------|--------|--------|
| 大众点评 | 本地生活信息流 | 提供地点信息 | 无游戏化、无 AI 生成、被动搜索 |
| 小红书攻略 | UGC 内容社区 | 提供路线灵感 | 需用户自己做攻略、无互动闯关 |
| 高德/百度地图 | 导航工具 | 提供路线导航 | 纯工具属性、无娱乐性 |
| Pokemon GO | AR 游戏 | 线下探索 + 游戏化 | 不是任务规划工具、无 AI 生成 |
| Geocaching | 户外寻宝游戏 | 线下打卡闯关 | 无 AI、非实时生成、社区驱动 |

**周末密令的差异化定位：** 唯一将 **AI 个性化生成 + 游戏化闯关叙事 + 线下城市探索** 三者融合的产品。不是帮你找地方，而是帮你"玩"一个周末。

### C. 技术选型建议（Demo 阶段）

| 层级 | 方案 | 理由 |
|------|----------|------|
| 技术栈 | 纯 HTML + CSS + JS（无框架） | 单文件离线运行，零依赖，评委双击即用 |
| 样式 | CSS 变量 + BEM 命名 | 易维护，无 CSS-in-JS 编译成本 |
| 状态管理 | 原生 JS 对象 + 简单发布订阅 | Demo 规模无需引入框架 |
| 数据存储 | localStorage | 离线持久化，无需后端 |
| 动画 | CSS keyframes + transform | 性能优于 JS 动画 |
| Canvas 绘图 | 浏览器原生 Canvas API | 分享卡片生成用 |
| 资源 | Base64 内联或纯 CSS | 避免外部依赖，单文件可分发 |
| 字体 | Google Fonts (Press Start 2P) | 经典 NES 字体 |
| 部署 | 单文件 HTML，可托管在任意静态服务 | 国内可用：火山引擎、腾讯云 COS |
