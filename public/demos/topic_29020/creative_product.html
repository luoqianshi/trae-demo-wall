// GATE SYSTEM

P0-P4 增强门禁

从 AST 静态分析到跨模型对抗审查，每一层都让代码更接近生产标准。

P0
AST 静态分析

真实代码分析，无 AI 幻觉。基于 AST 的类型注解、注释密度、复杂度评分。

P0
运行时验证

编译检查 + pytest 测试。代码不仅要写得漂亮，还要跑得起来。

P1
依赖真实性

PyPI API 验证包的存在与版本，杜绝 AI 编造不存在的依赖。

P1
语义熵检测

AST 结构变化检测，识别"原地踏步"的无效迭代，避免无尽空转。

P2
跨模型对抗

用不同模型互审代码，结构 vs 合规性双视角对抗，发现单一视角盲区。

P2
意图达成检查

验证代码是否真的实现了原始需求，匹配度量化打分。

P3
证据存储

SQLite + SHA256 不可变审计日志，每一次评审都有迹可循。

P3
API 网关 + RBAC

企业级 API 网关 + 角色权限控制，可直接接入团队工作流。

P4
安全表达式求值

策略 DSL 不再依赖 eval，policy engine 沙箱化执行。

P4
结构化日志

全链路 JSON 日志，配套升级意图检查器与实质性跨审核。

// PHILOSOPHY

五种克制，
胜过一千种堆叠

01 · 文件系统即数据库

不用数据库。所有状态用 YAML + Markdown 持久化，可直接 git 管理。

02 · 决策点驱动

只有 2 个点需要人参与，其余全自动。AI 不喧宾夺主。

03 · 角色切换

Work 角色分析思考，Solo 角色执行落地，对应不同的 prompt 模板。

04 · 完全可追溯

每轮迭代的所有产物保留，3 个月后回看也能复盘当时思路。
