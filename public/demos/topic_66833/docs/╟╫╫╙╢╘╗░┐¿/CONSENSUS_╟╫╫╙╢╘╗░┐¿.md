## 一、需求描述

### 1.1 项目概述
开发一个"亲子对话卡 - AI家庭沟通助手"展示用 Demo，纯 HTML 实现，无需后端，基于 Mock 数据演示核心功能。

### 1.2 产品定位
- **名称**：亲子对话卡
- **slogan**：当家长不知道如何开口时，帮他说出第一句话
- **定位**：不替代心理咨询师，只做"帮家长把话说对"的轻量小工具
- **目标用户**：初高中学生家长（35-50岁）

### 1.3 核心价值
1. 让专业支持"平民化"——普通爸妈也能拿到靠谱的沟通"小抄"
2. 在情绪爆发前"喊停"——几十秒就能拿到冷静、可执行的方案

## 二、验收标准

### 2.1 功能验收标准
| 序号 | 功能点 | 验收标准 |
|------|--------|----------|
| F1 | 首页展示 | 完整呈现产品介绍、核心功能、场景入口，与设计图风格一致 |
| F2 | 场景列表 | 展示 6 个核心场景卡片，支持分类筛选切换 |
| F3 | 场景详情 | 点击场景进入详情页，展示场景描述和对话流程 |
| F4 | 对话展示 | 混合式展示：聊天气泡 + 步骤标签 + 要点提示 |
| F5 | 多轮追问 | 每个场景至少 4 轮对话深度，支持点击展开下一轮 |
| F6 | 追问链路可视化 | 清晰展示当前对话进度/链路 |
| F7 | Mock 数据 | 所有数据本地 JSON，无网络依赖，打开即用 |
| F8 | 交互效果 | 按钮 hover、卡片点击动效、模拟 loading、对话渐入动画 |

### 2.2 视觉验收标准
- ✅ 暖色系主色调（橙红/暖橘色系）
- ✅ 圆角卡片设计
- ✅ 清晰的视觉层次
- ✅ 像聊天一样自然的对话感受
- ✅ 桌面端（1280px+）显示良好

### 2.3 技术验收标准
- ✅ 纯静态页面，双击 index.html 即可运行
- ✅ 无外部依赖（或 CDN 方式引入，不影响本地打开）
- ✅ 代码结构清晰，注释完整
- ✅ 数据与视图分离（Mock 数据独立文件）

## 三、技术实现方案

### 3.1 技术栈
| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 结构 | HTML5 | 语义化标签 |
| 样式 | CSS3 + 自定义变量 | 暖色系主题变量，响应式布局 |
| 交互 | 原生 JavaScript (ES6+) | 无框架依赖 |
| 数据 | JSON 内置 | Mock 数据独立 JS 文件 |
| 图标 | 可选：Font Awesome / SVG 内置 | 优先 SVG 内置减少依赖 |

### 3.2 页面结构（多页跳转）
```
index.html          # 首页 - 产品介绍 + 场景快捷入口
scenes.html         # 场景列表页 - 全部场景 + 分类筛选
scene-detail.html   # 场景详情页 - 对话流程 + 多轮追问
```


## 四、技术约束

### 4.1 必须遵守
1. **零后端依赖**：所有数据本地 Mock，页面纯静态
2. **零构建工具**：不需要 npm、webpack 等，原生 JS 直接运行
3. **可离线运行**：双击 html 文件即可打开，无需服务器
4. **代码风格一致**：统一命名规范、缩进风格

### 4.2 可选增强（时间充裕时）
- 深色模式切换
- 分享功能（纯前端复制链接）
- 收藏场景（localStorage）

## 五、任务边界限制

### 5.1 包含范围 ✅
1. 3 个完整页面（首页/列表/详情）
2. 6 个核心场景 Mock 数据
3. 每个场景 4 轮对话深度
4. 基础交互动画效果
5. 混合式对话展示（气泡 + 步骤 + 要点）

### 5.2 不包含范围 ❌
1. 真实 AI 大模型接口调用
2. 用户登录/注册系统
3. 数据持久化（后端数据库）
4. 完整 20+ 场景（仅 6 个演示）
5. 移动端精细适配（桌面端为主）
6. 多语言支持
