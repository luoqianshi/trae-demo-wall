## 四、需求理解

### 4.1 目标用户
- 主要用户：初高中学生家长（35-50岁）
- 痛点：想和孩子沟通但不知道怎么说，怕说重了怕说轻了
- 使用场景：半夜发现孩子玩手机、拿到不及格试卷、孩子摔门说"不用你管"等情绪激动时刻

### 4.2 核心用户旅程
```
家长遇到沟通困境 → 打开网页 → 选择对应场景 → 查看话术步骤
   → 按步骤尝试沟通 → 孩子回怼 → 点击"孩子这么说怎么办"
   → 获取下一轮策略 → 继续沟通
```

### 4.3 页面结构设想
1. **首页（index.html）** - 产品介绍 + 场景快捷入口 + 功能亮点
2. **场景列表页（scenes.html）** - 所有场景卡片，支持分类筛选
3. **对话详情页（scene-detail.html）** - 单场景对话流程，多轮追问链路可视化

## 六、智能决策策略

基于现有信息，我做出以下默认决策（用户可随时调整）：

1. **页面路由**：采用**多页面结构**（每个页面独立 HTML 文件），好处是简单直观、易维护、可直接打开运行，符合"展示 demo"定位。
2. **对话形式**：采用**聊天气泡式 + 步骤指引结合**的方式，既像聊天一样自然，又有清晰的步骤感。
3. **场景数量**：实现 **6 个核心场景**（沉迷手机、考试失利、拒绝沟通、早恋苗头、厌学情绪、顶撞老师），每个场景 3 轮对话。
4. **视觉风格**：严格遵循设计图的暖色系 + 圆角卡片风格，主色调建议为温暖的橙红色系。
