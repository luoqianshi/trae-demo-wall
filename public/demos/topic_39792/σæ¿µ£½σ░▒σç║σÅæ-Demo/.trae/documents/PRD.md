# 周末就出发 - 产品需求文档

## 1. 产品概述

### 产品名称
**周末就出发**

### 产品定位
一款帮年轻人一键规划周末出游的 iOS 风格 App，主打"懒人出游"概念，让用户无需复杂规划即可享受说走就走的周末旅行。

### 核心价值主张
- **一键生成**：用户只需输入基本信息，系统自动生成完整周末行程
- **精选推荐**：基于用户偏好推荐景点、餐厅、拍照打卡点
- **一键预订**：机票/高铁、酒店可直接在 App 内预订

### 目标用户
- 20-35 岁的年轻上班族
- 喜欢旅行但不愿花时间做复杂规划
- 追求生活品质和出片效果

---

## 2. 设计规范

### iOS 风格要求
- **设备框架**：居中展示 iPhone 手机外壳（圆角 40px+，灵动岛/刘海）
- **状态栏**：顶部显示时间和电池图标
- **圆角规范**：所有卡片、按钮、输入框使用 iOS 风格圆角（12-20px）
- **字体**：SF 风格系统字体（-apple-system, BlinkMacSystemFont）
- **底部 Tab 栏**：iOS 风格底部导航（图标+文字）

### 配色方案
| 用途 | 色值 |
|------|------|
| 主色渐变 | #FF6B35 (橙红) → #FFA500 (橙黄) |
| 背景色 | #F8F9FA (浅灰白) |
| 卡片背景 | #FFFFFF |
| 正文文字 | #1A1A2E (深灰黑) |
| 次要文字 | #6B7280 |
| 成功色 | #10B981 |
| 边框色 | #E5E7EB |

### 页面结构

#### 页面 1：启动/首页
- App Logo 和 Slogan
- 输入区域：
  - 出发时间选择器
  - 预算输入
  - 类型选择（看海/逛城市/亲近自然）
- 「一键生成行程」大按钮

#### 页面 2：行程生成页
- 交通信息卡片（高铁/机票）
- 住宿信息卡片（酒店）
- 第一天/第二天时间轴
- 时间点包含：餐厅、景点、打卡点
- 每个可点击进入「拍照打卡指南」

#### 页面 3：一键预订页
- 预订弹窗（模拟）
- 显示：车次/航班、价格、出发到达时间
- 「确认预订」按钮
- 预订成功提示

#### 页面 4：拍照打卡指南页
- 打卡点详情
- 机位建议
- 构图建议
- 穿搭推荐
- 模拟天气信息

#### 页面 5：我的页面
- 用户头像
- 订单历史入口
- 设置入口

### 底部 Tab 栏
- 首页（house icon）
- 行程（map icon）
- 我的（person icon）

### 交互流程
```
首页输入需求 → 点击生成行程 → 行程详情页
    ↓
查看机票预订 → 弹出预订确认 → 预订成功
    ↓
查看酒店预订 → 弹出预订确认 → 预订成功
    ↓
点击打卡点 → 进入拍照指南页
```

### 打卡点拍照指南

#### 栈桥风景区
- 机位：伸入海中的长堤尽头
- 构图：长堤引导线+回澜阁
- 穿搭：浅色连衣裙/休闲装
- 天气：晴 24°C 傍晚有金色光

#### 圣弥厄尔教堂
- 机位：教堂前广场仰拍
- 构图：对称构图+蓝天背景
- 穿搭：韩系风格/文艺复古
- 天气：晴转多云 22°C

#### 五四广场夜景
- 机位：海边栏杆处
- 构图：五月的风雕塑+灯光
- 穿搭：夜景穿搭/小礼服
- 天气：晴 20°C 夜间凉

#### 第二海水浴场
- 机位：沙滩+礁石组合
- 构图：自然风景+人物
- 穿搭：泳装外套/沙滩风
- 天气：晴 26°C 海风

## 4. 技术实现

### 技术栈
- 单文件 HTML
- 内联 CSS（iOS 风格样式）
- 内联 JavaScript（页面导航、弹窗、交互）

### 关键实现
- CSS 变量管理配色
- Flexbox/Grid 布局
- CSS 动画过渡效果
- 模拟数据直接嵌入 JS
- 页面通过 class 切换显示/隐藏

### 响应式
- iPhone 尺寸固定宽度（375px）
- 居中展示手机外壳
- 支持safari/chrome直接打开
