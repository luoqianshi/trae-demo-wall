# 🗄️ 方案3 - Bases 数据库模板

## 什么是 Bases

Obsidian 1.10+ 版本内置的**数据库核心插件**，类似 Notion 的表格数据库。

**优点**：
- 原生支持，不用装第三方插件
- 和 Obsidian 深度集成
- 性能好，不卡顿

## 开启 Bases

1. 打开 Obsidian 设置 → 核心插件
2. 找到 **数据库（Bases）**
3. 开启开关

## 创建第一个数据库

### 方法1：从文件夹创建
1. 右键点击 `10_项目Projects` 文件夹
2. 选择 **新建数据库 BASE**
3. 设置数据库名称，例如「项目总览」

### 方法2：从空白创建
1. 命令面板 → 输入 `新建数据库`
2. 选择存放位置
3. 自定义字段

## 建议字段配置

### 项目数据库

| 字段名 | 类型 | 选项 |
|--------|------|------|
| 项目名 | 文本 | - |
| 状态 | 单选 | 进行中 / 暂停 / 已完结 |
| 优先级 | 单选 | 高 / 中 / 低 |
| 截止日期 | 日期 | - |
| 阶段 | 单选 | 筹备 / 执行 / 收尾 |
| 负责人 | 文本 | - |
| 标签 | 多选 | 自定义 |
| 项目链接 | 链接 | 关联项目笔记 |

### 任务数据库

| 字段名 | 类型 | 选项 |
|--------|------|------|
| 任务名 | 文本 | - |
| 所属项目 | 链接 | 关联项目 |
| 状态 | 单选 | 待办 / 进行中 / 审核 / 完成 |
| 优先级 | 单选 | 高 / 中 / 低 |
| 截止日期 | 日期 | - |
| 预估工时 | 数字 | - |
| 实际工时 | 数字 | - |
| 任务笔记 | 链接 | 关联任务笔记 |

## 三种视图

### 1. 表格视图
- 最基础的视图
- 适合批量查看和编辑
- 支持排序、筛选、分组

### 2. 看板视图
- 按状态分列
- 拖拽卡片改变状态
- 适合进度管理

### 3. 日历视图
- 按日期展示
- 适合看任务分布
- 排期规划

## 使用技巧

### 关联数据
- 用「链接」字段关联项目和任务
- 点击可以直接跳转到对应笔记

### 筛选器
- 保存常用筛选条件
- 比如「只看高优先级任务」
- 「只看本周到期任务」

### 分组
- 按项目分组查看任务
- 按优先级分组
- 按负责人分组

## 和方案2的对比

| 功能 | Bases（方案3） | Dataview + Kanban（方案2） |
|------|---------------|--------------------------|
| 上手难度 | ⭐ 简单 | ⭐⭐ 需要学一点语法 |
| 可视化 | ⭐⭐ 三种视图 | ⭐⭐⭐ 自定义程度高 |
| 性能 | ⭐⭐⭐ 原生快 | ⭐⭐ 取决于查询复杂度 |
| 灵活性 | ⭐⭐ 受限于插件 | ⭐⭐⭐ 完全自定义 |
| 推荐度 | 新手推荐 | 进阶推荐 |

## 注意事项

- Bases 是 Obsidian 官方功能，持续更新中
- 数据存储在笔记的 YAML frontmatter 中
- 和其他方案兼容，可以同时使用
