# 📊 方案3 - Project Manager 插件配置说明

## 插件介绍

**Projects** 插件（by Marcus Olsson）是 Obsidian 社区最强大的项目管理插件之一，支持：
- 📋 表格视图
- 📊 看板视图
- 📅 日历视图
- 🔥 甘特图视图

## 安装方法

1. 打开 Obsidian 设置 → 社区插件 → 浏览
2. 搜索 **Projects**（by Marcus Olsson）
3. 安装并启用

## 创建第一个项目

1. 左侧边栏点击 **Projects** 图标
2. 点击 **New Project**
3. 填写项目名称和描述
4. 选择项目存放位置（建议放在 `10_项目Projects/` 下）

## 配置字段

建议配置以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 状态 | 单选 | 待办 / 进行中 / 审核 / 已完成 / 暂停 |
| 优先级 | 单选 | 高 / 中 / 低 |
| 截止日期 | 日期 | 任务截止时间 |
| 负责人 | 文本 | 任务负责人 |
| 预估工时 | 数字 | 预估需要多少小时 |
| 实际工时 | 数字 | 实际花费小时数 |
| 阶段 | 单选 | 筹备 / 执行 / 收尾 |
| 标签 | 多选 | 自定义标签 |

## 四种视图

### 1. 表格视图（Table）
- 类似 Excel，批量编辑最方便
- 支持排序、筛选、分组

### 2. 看板视图（Kanban）
- 拖拽卡片改变状态
- 视觉化进度管理

### 3. 日历视图（Calendar）
- 按日期查看任务分布
- 适合排期规划

### 4. 甘特图视图（Gantt）
- 可视化项目时间线
- 查看任务依赖关系
- 识别关键路径

## 任务依赖设置

1. 打开任务笔记
2. 在 YAML frontmatter 中添加：
```yaml
依赖: 
  - "[[任务A]]"
  - "[[任务B]]"
```
3. 在甘特图中可以看到依赖连线

## 进阶技巧

### 工时统计
- 每个任务填写预估工时和实际工时
- 项目结束后可以统计总工时
- 分析预估准确度，不断提升

### 子任务
- 一个大任务可以拆成多个子任务
- 在任务笔记中用 `- [ ]` 待办清单
- 或使用 Projects 的子任务功能

### 模板复用
- 建好一个项目的字段配置后
- 可以导出配置，新建项目时导入

## 注意事项

- Projects 插件数据存在单独的 `.json` 文件中
- 建议定期备份项目配置
- 和 Dataview 可以共存，互不冲突
