## 1. Demo 简介

### 一句话介绍：
FlowDeck 是一套基于 Obsidian 的 PARA 项目管理模板库，让一个人用最低成本（免费 + 本地 + 10 分钟搭建），把散落在各处的项目、任务、日志、周报、习惯全部收拢到一个库里。

### 解决了什么问题：
一个人或者小团队同时管多个项目时，信息散落在 Excel、Word、微信收藏、备忘录、周报里……光是「找到信息」就要浪费大量时间。工具切换、进度不可视、重复劳动、缺乏复盘，这些问题每天都在消耗精力。

### 核心亮点：
- **0 成本**：基于免费的 Obsidian，不花一分钱
- **100% 本地**：数据全部存在自己电脑上，隐私安全
- **3 套方案**：从零插件到专业版，按需选择，可逐步升级
- **10+ 模板**：项目、任务、日志、周报、月报、习惯追踪，开箱即用
- **完整闭环**：收集 → 项目化 → 任务拆解 → 可视化跟踪 → 复盘归档

## 2. 背景 / 动机

### 为什么会想到做这个：
说出来有点好笑，我最开始整理这套东西，纯粹是因为**忘性太大**。

有一次同时推进三个产品上线，任务散落在：
- 微信收藏里存了几条待办
- Excel 里有个甘特图
- 备忘录里写了几句想法
- 飞书里还有几条会议纪要

结果上线前一天才发现有个关键任务漏了——因为它被记在了微信收藏里，我再也没打开过。

那次之后我就下定决心：**必须把所有项目信息收拢到一个地方**。

试过 Notion（国内同步太慢）、试过飞书（太重了个人用浪费）、试过各种待办 App（只管清单不管文档），最后发现 Obsidian + 几个插件 + 一套好的模板，居然完全够用。

于是就有了 FlowDeck。

### 适合谁用：
- 自由职业者 / 独立开发者
- 内容创作者（视频/写作/运营）
- 小团队负责人
- 学生 / 研究者
- 任何同时管理 2 个以上项目的人

## 3. 功能介绍 & 效果展示

### 📁 系统架构

极简两层文件夹结构，基于 PARA 方法论：

```
Obsidian项目管理库/
├── 00_收集箱/           ← 临时想法碎片暂存
├── 10_项目Projects/     ← 所有进行中项目
│   └── 项目名/
│       ├── 01_项目总览.md
│       ├── 02_任务清单/
│       ├── 03_资料素材/
│       └── 04_复盘归档/
├── 20_资源Resources/    ← 模板 + 参考文档
├── 30_归档Archive/      ← 已完结项目
├── 40_日志Daily/        ← 每日日志
├── 50_习惯追踪/         ← 习惯打卡
├── Home.md              ← 零插件版仪表盘
└── Home_Dataview版.md   ← 自动版仪表盘
```

### 🔥 6 大核心功能

**1. 收集箱机制**
- 任何临时想法、需求碎片随手丢进去
- 每天花 5 分钟整理归类
- 不让任何灵感溜走

**2. 一键生成项目**
- 用模板 30 秒创建完整项目页
- 自动包含阶段拆分、任务清单、风险提示、复盘区
- 不用每次从零搭框架

**3. 可视化看板（Kanban）**
- 四列标准流程：待办 → 进行中 → 审核验收 → 已完成
- 拖拽卡片改变任务状态
- 视觉化管理进度

**4. 自动仪表盘（Dataview）**
- 自动汇总所有项目和任务
- 逾期任务自动标红
- 完成率实时统计
- 不用手动更新表格

**5. 日志与复盘体系**
- 每日日志模板：时间记录 + 计划 + 总结
- 周报模板：本周完成 + 问题 + 下周计划
- 月报模板：项目概况 + 关键数据 + 规划
- 养成记录和复盘习惯

**6. 习惯追踪**
- 月度打卡表，6 个习惯 × 31 天
- 每周统计完成率
- 月底反思调整

### 🎯 三套方案，按需选择

| | 方案1：零插件版 | 方案2：可视化看板版 | 方案3：专业全功能版 |
|---|---|---|---|
| 需要插件 | 不需要 | Kanban + Dataview + Templater | Projects / Bases |
| 学习成本 | ⭐ 最低 | ⭐⭐ 中等 | ⭐⭐⭐ 较高 |
| 任务看板 | 手动勾选 | ✅ 拖拽卡片 | ✅ 拖拽 + 甘特图 |
| 自动仪表盘 | 手动更新 | ✅ Dataview自动 | ✅ 数据库视图 |
| 甘特图 | ❌ | ❌ | ✅ |
| 推荐 | 新手 / 轻度 | **大多数人** | 进阶 / 复杂项目 |

## 4. 技术实现 / 创作思路

### 技术栈：
- **载体**：Obsidian（免费、本地、跨平台的 Markdown 知识库）
- **形态**：Markdown 文件 + YAML 元数据 + 文件夹结构
- **可选插件**：Kanban（看板）、Dataview（自动查询）、Templater（模板引擎）

### 设计思路：
整个系统的核心设计原则是「**渐进式复杂度**」——

1. **第一天**：你可以只用「收集箱 + 项目总览 + 手动更新 Home 页」，零插件就能跑
2. **用熟了之后**：装上 Kanban 和 Templater，体验升级一个档次
3. **项目变复杂了**：再上 Projects 插件，甘特图、任务依赖全安排

不会一上来就给你一堆复杂概念把人吓跑，而是从最简单的版本开始，用着用着不够了再升级。

### 为什么选 Obsidian 而不是其他工具：
- **数据是你的**：全部是纯文本 Markdown 文件，10 年后还能打开
- **不依赖网络**：离线也能用，没有服务器挂了的风险
- **生态强大**：社区有上千个插件，想加什么功能都有
- **零成本**：个人使用完全免费

## 5. 经历的困难 & 踩过的坑

### 坑1：到底要多复杂？
最开始我想做一个「大而全」的系统，什么功能都想加进去——工时统计、OKR、财务管理……结果越做越乱，自己都不想用。

后来想明白了：**80% 的人只需要 20% 的功能**。于是大刀阔斧砍掉了很多东西，只保留最核心的项目管理 + 日志复盘，反而清爽好用了。

### 坑2：插件装太多会卡
最开始装了七八个插件，结果 Obsidian 启动变慢了，搜索也卡。

现在的建议是：**从 0 插件开始，缺什么装什么**。不要一开始就把所有插件都装上，很多功能你可能根本用不上。

### 坑3：模板太复杂不想填
最开始的项目模板有十几个字段，每次新建项目要填半天，结果就是——不填了。

后来精简到只保留最核心的几个字段（状态、截止日期、优先级），其他都是可选的，使用频率一下就上来了。
