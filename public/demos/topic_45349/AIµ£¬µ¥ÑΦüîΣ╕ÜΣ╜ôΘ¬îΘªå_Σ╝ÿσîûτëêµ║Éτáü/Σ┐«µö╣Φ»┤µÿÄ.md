# 修改说明

## 1. 推荐系统

- 修复匹配度计算：由错误的 0.6-1.0 区间改为 60-99 区间。
- 推荐结果新增 `scoreBreakdown`，可展示兴趣、科目、性格、避雷和综合指数。
- 结果页新增“推荐依据拆解”，增强可信度。

## 2. 表单体验

- 兴趣和擅长科目限制为 2-4 个。
- 讨厌工作类型至少选择 1 个。
- 达到上限后按钮自动禁用，避免文字说明和实际逻辑不一致。
- Loading 动画完整播放 2.8 秒，再生成推荐结果。

## 3. 浏览职业

- 首页“浏览全部职业”改为先进入浏览模式再跳转。
- 直接访问 `/results` 时允许浏览全部职业，不再强制跳回输入页。

## 4. 沉浸式体验页

- 修复 Hook 条件调用问题。
- 增加职业任务面板：当前任务、场景氛围、决策评分。
- 增加职业装备 HUD：展示每个职业的核心工具。
- 决策题升级为更贴合职业场景的问题。
- 最后一个场景优先使用该职业专属挑战题。
- 结尾页新增关键选择记录，体验感更完整。

## 5. 报告页

- 报告 ID 改为基于生成时间和职业 ID 的稳定编号，不再每次刷新变化。
- 新增 Top 3 职业横向对比：匹配度、核心优势、主要风险。
- 增加打印样式，导出 PDF 时更正式。

## 6. 新增职业

新增 4 个职业：

- 网络安全分析师
- 机器人工程师
- 游戏策划师
- 新媒体策略师

每个职业都补充了匹配权重、能力项、时间线、路线图、入门项目、风险、工作场景、装备、挑战题和人格画像。

## 7. 工程质量

- TypeScript 检查通过。
- ESLint 检查通过。
- 移除生产构建中的开发定位插件配置。
- 增加 `prefers-reduced-motion` 支持。
- 输出压缩包已去除 `node_modules` 和旧构建产物，保持专业交付结构。
