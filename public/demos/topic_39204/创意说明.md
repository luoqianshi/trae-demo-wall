# 创意说明：AI 同声传译助手

## 1. 项目背景与痛点

AI 同声传译助手是一款面向课堂、会议、展会接待和跨国沟通场景的实时双语同传工具。我想解决的核心问题是：用户在跨语言交流中不仅听不懂内容，还很难判断机器翻译结果是否稳定、是否已经被修正、是否可以在会后直接引用。

现有很多实时字幕或翻译工具只展示单一结果，用户看到一句字幕时并不知道它是临时草稿还是最终定稿。对于会议记录者、课堂讲解者和商务沟通者来说，这会带来明显不便：实时沟通时容易误解关键信息，会后整理时又需要重新核对录音、字幕和翻译内容，效率低且不够可信。

## 2. 解决方案与核心功能

本项目的解决方案是设计一款强调“实时可修正、定稿可追溯、移动端可快速启动”的 AI 同声传译助手。产品采用 Draft 草稿字幕和 Final 定稿字幕双层机制：Draft 用于快速显示临时翻译，帮助用户第一时间理解内容；Final 在句子稳定后生成正式版本，用于展示、记录和会后导出。

核心功能包括四部分：第一，场景模板，用户可以选择课堂模式、会议模式、双人沟通模式或外放视频模式，一键匹配收音和字幕策略；第二，会前关键词注入，提前输入人名、品牌名、项目名和专业术语，提高识别与翻译稳定性；第三，可信时间轴，记录每句话的首次出现时间、修订次数、最近更新时间和定稿状态；第四，会后导出，生成最终双语稿、时间戳字幕、修订摘要和待确认清单。

## 3. 技术框架或实现思路

技术实现上，项目可以采用前后端分离思路。前端使用 React 构建主控端、展示端、观察端和记录端界面，并通过 Electron 或网页形式运行，方便在桌面端、移动端或投屏环境中使用。后端使用 FastAPI 提供 WebSocket 实时通信接口，负责接收音频流、推送字幕结果、返回状态消息和处理开始、暂停、停止等控制指令。

实时处理链路可以分为音频采集、语音识别、机器翻译、结果修订、可信度判断和导出整理几个模块。系统先将语音转写为源语言文本，再调用翻译模型生成目标语言字幕，同时根据上下文、关键词和句子稳定程度判断是否需要修订。每次修订都会被记录到时间轴中，最终形成从 Draft 到 Final 的可追溯过程。

## 4. 应用场景与价值

在课堂场景中，老师或学生可以快速开启双语字幕，降低外语课程、公开讲座和国际交流活动的理解门槛。在会议场景中，主控端负责控制同传流程，展示端突出 Final 字幕，记录端沉淀完整时间轴，方便会后整理纪要。在展会和商务沟通场景中，产品可以帮助双方快速理解对方表达，减少因语言障碍造成的沟通停顿。

这个创意的价值不仅是提高翻译效率，更重要的是提升 AI 翻译结果的透明度和可信度。用户可以清楚看到哪些内容只是临时草稿，哪些内容已经定稿，哪些句子需要人工复核。它让实时翻译从“能看见结果”升级为“能信任结果、能追溯过程、能利用成果”。已附带创意产物 HTML 文件 `trae-work-idea.html`，可用于展示完整方案。
