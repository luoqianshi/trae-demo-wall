# AI 同声传译助手 — 后端执行计划

> 更新日期：2026-06-04
> 基于最新技术栈：Python + FastAPI + WebSocket + faster-whisper + DeepSeek API + IndexTTS2

---

## 一、目标

构建实时语音翻译管道，能够：
- 捕获系统音频流，检测语音活动
- 实时语音识别（ASR），流式输出文本
- 调用 DeepSeek API 进行翻译 + 自动纠错
- 推送翻译结果到前端，可选 TTS 语音播报
- 端到端延迟控制在 3~5 秒内

---

## 三、模块分解

### 模块 1：音频捕获与 VAD 模块

| 项 | 说明 |
|:---|:---|
| **Purpose** | 捕获系统音频/麦克风输入，通过 VAD 检测语音活动区间 |
| **Input** | 音频设备（系统音频环回或麦克风） |
| **Output** | 语音片段（bytes），推送到 ASR 模块 |
| **Dependency** | 无 |

**关键实现**：
- `sounddevice.RawInputStream` 捕获音频流（16kHz, 16bit, 单声道）
- `silero-vad` 检测语音起始/结束，过滤静音
- 音频分片：每段 1~2 秒，重叠 0.5 秒保证连续性
- 支持两种模式：实时流式（默认）/ 文件批量

---

### 模块 2：ASR 识别模块

| 项 | 说明 |
|:---|:---|
| **Purpose** | 将音频片段转为文本，流式输出 |
| **Input** | 音频片段（numpy array / bytes） |
| **Output** | 文本片段 + 时间戳 + 置信度 |
| **Dependency** | 模块 1 |

**关键实现**：
- 模型：`faster-whisper` `large-v3-turbo`（比 large-v3 快 8 倍）
- 设备：CPU `int8` 量化（内存 <1GB，适合无 GPU 环境）
- 参数：`beam_size=5`, `best_of=5`, `vad_filter=True`
- 增量输出：通过回调函数实时推送识别结果
- 语言检测：自动检测输入语言

---

### 模块 3：翻译 + 纠错引擎

| 项 | 说明 |
|:---|:---|
| **Purpose** | 调用 DeepSeek API 实时翻译，并基于上下文自动纠错 |
| **Input** | ASR 输出的文本片段 + 历史上下文 |
| **Output** | 翻译后的中文文本 + 修正标记（如有） |
| **Dependency** | 模块 2 |

**关键实现**：
- **模型**：`deepseek-v4-flash`（1M 上下文，流式 SSE）
- **API 格式**：OpenAI 兼容，base_url = `https://api.deepseek.com`
- **Prompt 设计（鲁棒思维链）**：
  ```
  你是一名专业的同声传译员。请将以下英文实时翻译成流畅的中文。

  规则：
  1. 保持口语化、自然流畅
  2. 如遇专业术语，保留英文原文并在括号内给出中文
  3. 根据上下文判断：如果之前的翻译有错误或不准确，请输出修正后的版本
  4. 以 JSON 格式输出：{"translation": "...", "corrections": [{"old": "...", "new": "..."}]}

  历史上下文：{history}
  当前文本：{current_text}
  ```
- **滑动窗口**：维护最近 5~10 句作为上下文
- **流式输出**：SSE 实时返回，降低感知延迟

---

### 模块 4：WebSocket 服务模块

| 项 | 说明 |
|:---|:---|
| **Purpose** | FastAPI WebSocket 服务端，连接前端，推送翻译结果 |
| **Input** | 翻译引擎输出 / 前端控制指令 |
| **Output** | JSON 消息（translation/correction/status/error） |
| **Dependency** | 模块 3 |

**关键实现**：
- FastAPI `WebSocketEndpoint`
- 消息类型：
  - `{"type": "translation", "text": "...", "timestamp": ...}`
  - `{"type": "correction", "oldText": "...", "newText": "...", "index": ...}`
  - `{"type": "status", "state": "running/stopped/error"}`
- 支持多客户端连接
- 自动重连与心跳机制

---

### 模块 5：TTS 语音播报模块（可选）

| 项 | 说明 |
|:---|:---|
| **Purpose** | 将翻译后的中文转为语音播报 |
| **Input** | 中文翻译文本 |
| **Output** | 音频流（播放或保存） |
| **Dependency** | 模块 3 |

**关键实现**：
- **主方案**：`IndexTTS2` 本地部署
  - 模型下载：`IndexTeam/IndexTTS-2`（HuggingFace）
  - 推理：`<200ms`，支持流式
  - 情感：默认中性，可选激动/温柔等
- **备选方案**：`edge-tts`（微软免费 API，无需 GPU）
- 播报策略：每句翻译完成后延迟 0.5 秒播报，避免与演讲重叠

---

### 模块 6：管道编排与状态管理

| 项 | 说明 |
|:---|:---|
| **Purpose** | 协调各模块工作流，管理全局状态 |
| **Input** | 各模块事件 / 用户指令 |
| **Output** | 统一的状态机与错误处理 |
| **Dependency** | 所有模块 |

**关键实现**：
- 异步队列：`asyncio.Queue` 连接各模块
- 状态机：`idle` → `capturing` → `transcribing` → `translating` → `outputting`
- 错误处理：ASR 失败 → 降级到文本提示；API 失败 → 重试 3 次
- 性能监控：记录各环节延迟，输出到日志

## 六、API 设计

### WebSocket 消息协议

**后端 → 前端**

| 类型 | 格式 | 说明 |
|:---|:---|:---|
| translation | `{"type": "translation", "text": "...", "isFinal": false}` | 流式翻译结果 |
| correction | `{"type": "correction", "oldText": "...", "newText": "...", "sentenceIndex": 2}` | 纠错通知 |
| status | `{"type": "status", "state": "running"}` | 状态变更 |
| error | `{"type": "error", "message": "..."}` | 错误信息 |

**前端 → 后端**

| 类型 | 格式 | 说明 |
|:---|:---|:---|
| start | `{"action": "start", "sourceLang": "en", "targetLang": "zh"}` | 启动翻译 |
| stop | `{"action": "stop"}` | 停止翻译 |
| config | `{"action": "config", "apiKey": "...", "ttsEnabled": true}` | 更新配置 |
