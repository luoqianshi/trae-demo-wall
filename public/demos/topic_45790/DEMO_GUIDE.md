# 评委 3 分钟体验指引

> 目标：让评委在 3 分钟内完整体验研复通 ReproPilot 的核心闭环。
> 全程本地运行，无需联网，无需安装。

## 准备

- 解压 `研复通_ReproPilot_初赛Demo.zip` 到任意目录。
- 使用 Chrome / Edge / Firefox 浏览器。
- 建议分辨率 1366×768 及以上。

## 体验路径（按顺序，预计用时）

### 第 1 步｜打开（约 5 秒）

双击解压目录中的 `index.html`。

看到首页：标题「研复通 ReproPilot」与文案「代码能运行，不等于论文复现正确」。

> 首次访问会弹出 4 步引导，可阅读或直接点击「跳过引导」。

### 第 2 步｜载入示例项目（约 10 秒）

点击首页主按钮「**载入 MDI+GRU 示例项目**」。

进入工作台，默认停在「项目总览」页。此时顶栏可信度已显示一个初值（约 58%）。

### 第 3 步｜开始审计（约 15 秒）

在「项目总览」页找到「审计控制」卡片。

点击「**开始审计 / 重新审计**」按钮：
- 按钮进入加载状态（旋转动画对应真实规则引擎计算）；
- 完成后顶部弹出绿色 Toast 提示「已完成审计：通过 X / 12 条规则，可信度 X%」。

### 第 4 步｜查看可信度与高风险问题（约 20 秒）

在审计控制卡片查看：
- 综合可信度（基线约 58%）；
- 失败规则数（基线 5 条左右）；
- 高风险失败数。

下方的「综合复现可信度」进度条与五个评分维度（论文步骤映射 / 数据可追溯性 / 边界与分组科学性 / GRU 请求与结果完整性 / 测试与产物真实性）展示扣分分布。

### 第 5 步｜打开一条风险证据（约 20 秒）

点击左侧导航「**风险证据**」。

页面顶部出现筛选条：全部 / 高风险 / 中风险 / 低风险。点击「高风险」筛选。

点击任意一条风险卡片，右侧抽屉滑出，展示：
- 风险标题与严重度；
- 关联的规则编号（如 R02）；
- 关联的论文步骤（如 P05）；
- 证据文本（具体哪条消息、哪个字段、什么问题）；
- 影响说明。

### 第 6 步｜生成修正计划（约 15 秒）

点击左侧导航「**修正计划**」。

页面顶部出现筛选条：全部 / P0 / P1 / P2。可按优先级筛选任务。

页面顶部有「**生成修正计划**」按钮，点击：
- 按钮进入加载状态；
- 完成后 Toast 提示「已根据 X 条失败规则生成修正计划，共 X 项任务」。

### 第 7 步｜应用示例修复（约 15 秒）

在「修正计划」页点击「**应用示例修复**」按钮：
- 按钮进入加载状态并重新审计；
- 完成后页面顶部出现「修复前后对比」卡片。

### 第 8 步｜查看修复前后对比（约 10 秒）

在「修正计划」页顶部的对比卡片中查看：
- 修复前可信度（约 58%）→ 修复后可信度（约 92%）；
- 修复前风险数（6 项）→ 修复后风险数（0 项高风险）；
- 各项任务的修复状态。

### 第 9 步｜导出审计报告（约 10 秒）

点击左侧导航「**报告导出**」。

分别点击：
- 「**导出 Markdown 报告**」：浏览器下载 `repropilot-report.md`；
- 「**导出 JSON 报告**」：浏览器下载 `repropilot-report.json`。

> 若浏览器拦截下载，页面会弹出红色 Toast 提示「下载被浏览器阻止，请检查下载权限」。

## 完成

至此评委已完整体验研复通的核心闭环。总用时约 2 分钟，留有查看细节的余量。

## 故障排查

| 现象 | 原因 | 解决 |
| ---- | ---- | ---- |
| 点击按钮无反应 | 浏览器过旧 | 升级到 Chrome 90+ / Edge 90+ / Firefox 90+ |
| 下载未触发 | 浏览器拦截 | 允许该站点下载文件 |
| 引导弹窗重复出现 | 清除了 localStorage | 正常现象，再次点击跳过即可 |
| 控制台有 CORS 警告 | 使用了 `file://` 直接打开 | 正常，不影响功能 |

## 不要做的事

- 不要尝试用 `http-server` 之类启动（不必要，且可能与 file:// 行为不同）。
- 不要修改 `js/data.js` 中的内置数据（会破坏基线案例的故意偏差）。
- 不要把 release 包内的文件移到其他目录后还期望相对路径正常（请保持目录结构完整）。
