# 研复通 ReproPilot — AI 科研复现与实验审计助手

> **代码能运行，不等于论文复现正确。**
> 研复通把论文要求、源码实现和实验结果放到同一张可验证地图上。

参赛赛道：**学习工作**（TRAE AI 创造力大赛 · 初赛 Demo）

---

## 一、项目介绍

研复通 ReproPilot 是一款面向本科生、研究生、科研新人和论文复现开发者的**本地科研复现审计助手**。

它解决的痛点不是「帮你总结论文」，而是：**论文方法是否真的在源码里实现、源码虽然能跑但有没有偏离论文流程、实验参数和产物是否一致、当前项目到底复现到了哪一步**。

每个判断都带可追溯证据，每个结论都能回到原消息、源码或实验记录。

> 说明：本初赛版本采用**本地结构化规则引擎**（共 12 条规则 R01–R12），不调用大模型，不会自动理解任意 PDF 或任意源码。内置 MDI+GRU 案例来源于真实科研开发流程，但已脱敏。

## 二、核心功能

1. **载入示例项目**：一键载入内置 MDI+GRU 第一阶段科学缝合案例（基线含故意偏差）。
2. **执行规则审计**：基于 12 条规则对 message_id、字段边界、严格分组、GRU request、测试产物等维度进行结构化核验，输出综合可信度评分与失败规则。
3. **查看风险证据**：每条风险都关联具体规则、相关步骤、证据文本与影响说明，可按高/中/低风险筛选。
4. **生成修正计划**：根据失败规则自动生成优先级（P0/P1/P2）修正任务清单，可按优先级筛选。
5. **应用示例修复**：一键应用示例修复数据，重新审计后展示修复前后对比（可信度提升、高风险清零）。
6. **导出审计报告**：支持导出 Markdown 与 JSON 两种格式，含项目信息、规则结果、风险清单、修正计划与评分维度。
7. **首次使用引导**：4 步引导，可跳过，说明产品定位与本地处理保证。
8. **关于 Demo 真实性说明**：明确说明这是初赛可交互原型，不冒充大模型自动理解任意项目。

## 三、打开方式

**双击 `index.html` 即可运行**，无需启动服务器、无需联网、无需任何依赖。

- 不使用 React / Vue 等框架
- 不依赖外部 CDN
- 不调用外部 API
- 不使用外部字体和远程图片
- 所有资源使用相对路径

> 说明：浏览器端不通过 `fetch` 自动读取本地 JSON，避免 `file://` 环境限制。内置案例数据由 `js/data.js` 提供，`sample-data/` 下的 JSON 供用户手动查看与导入。

## 四、推荐体验流程（3 分钟内完成）

1. 双击 `index.html` 打开首页。
2. 首次访问会出现 4 步引导，可阅读或点击「跳过」。
3. 点击「载入 MDI+GRU 示例项目」进入工作台。
4. 在「项目总览」页点击「开始审计」按钮（按钮会显示加载动画并真实执行规则引擎计算）。
5. 查看综合可信度与失败规则，点击左侧「风险证据」打开一条风险证据详情。
6. 点击左侧「修正计划」查看按优先级排序的任务，点击「应用示例修复」。
7. 在「修正计划」页查看修复前后对比（可信度提升、风险数量下降）。
8. 点击左侧「报告导出」，分别导出 Markdown 与 JSON 报告。

> 详细 3 分钟评委体验指引见 [docs/DEMO_GUIDE.md](docs/DEMO_GUIDE.md)。

## 八、测试方法

本项目使用 Node.js 内置测试运行器（`node --test`），不依赖任何第三方测试框架。

### 运行测试

```bash
cd ReproPilot-Demo
node --test
```

预期输出：`# pass 23  # fail 0`，退出码 `0`。

测试覆盖：
- Schema 校验（缺少字段、非法结构）
- 12 条规则的正反向用例
- 评分计算（基线分数低于修复后分数）
- 修正计划只包含失败规则
- `applyDemoRepairs` 后高风险清零
- `compareAudits` 返回前后对比结构
- Markdown / JSON 报告生成与可重新解析
- `downloadTextFile` 在 Node 环境安全降级

详见 [docs/TEST_REPORT.md](docs/TEST_REPORT.md)。
