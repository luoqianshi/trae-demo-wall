## 1. Demo 简介

### 是什么

**智行通**是一款面向城市通勤人群的**地铁实时智能出行助手小程序**，集成列车到站倒计时、车厢级拥挤度热力图、智能换乘导航、个性化出行提醒等功能，帮助用户实现"进站前就知道该不该跑"的精准出行决策。

### 面向谁

| 用户群体 | 核心需求 |
|---------|---------|
| 上班族通勤族 | 精确到秒的到站时间，避开拥挤车厢，准时打卡 |
| 学生群体 | 规划上课路线，避免因等车迟到 |
| 外地游客/出差人士 | 换乘站内导航，不怕走迷宫 |
| 老年人/特殊群体 | 大字体 + 语音播报，降低使用门槛 |

### 主要功能

**功能一：实时到站倒计时 + 车厢级拥挤度**

首页展示附近站点，点击进入后可看到精确到秒的列车到站倒计时，以及**每节车厢**的拥挤度分布（6 节车厢分别标注空闲/适中/拥挤），还有建议前往哪节车厢的智能提示。

**功能二：全线路拥挤度热力图**

支持列表视图和柱状图视图双模式切换，直观展示整条线路各站点的拥挤度分布。顶部还有统计学概览（站点总数、拥挤站点数、平均拥挤度），帮助用户快速判断出门时机。

**功能三：智能换乘导航**

输入起点和终点（支持快捷站点选择），系统给出多方案对比（时间、换乘次数），选中方案后展示时间线式的详细步骤，包括每段地铁的拥挤度标记、换乘步行指引等。

**功能四：个性化通勤管理**

在"我的"页面可以设置家和公司地址、查看出行统计（本月出行次数、通勤时长、碳减排量）、管理推送通知和车厢温度偏好。

## 2. Demo 创作思路

### 灵感来源

灵感来源于日常通勤的真实体验：每天早高峰在地铁站台上，数以万计的通勤者盯着隧道尽头，不知道下一趟车还有多久。而根据交通运输部数据，2025 年全国轨道交通年客运量已达 **332.4 亿人次**，54 个城市开通了地铁，日均客流量突破 **1.2 亿人次**——这是一个巨大的用户痛点池。

### 想解决的问题

当前通勤者面临的核心痛点：

1. **盲目等待**：站台上缺乏精确到秒的到站倒计时，只能碰运气
2. **拥挤不可预知**：高峰期枢纽站拥堵系数达 1.8（远超国际标准 1.2），用户不知道哪节车厢人少
3. **换乘迷宫**：超 60% 换乘站高峰期拥堵系数 >1.5，43% 的乘客因换乘不便选择绕行
4. **信息碎片化**：到站看高德、拥挤看百度、乘车码开亿通行——多 App 切换成本高

### 为什么做这个方向

虽然高德地图已推出全国拥挤度导航（2025 年 12 月），广州地铁也有"精准出行"功能，但现有产品要么是**大而全的综合地图 App**（打开慢、功能杂），要么是**各城市独立的地铁官方 App**（跨城不通用、信息更新慢）。智行通的差异化在于：

- **小程序形态**：扫码即用，零下载门槛
- **聚焦地铁场景**：不做"又一个地图"，只做地铁出行做到极致
- **车厢级粒度**：比地图 App 的线路级拥挤度更深一层，精确到每节车厢
- **深色科技风 UI**：暗色主题 + 青绿色强调色，适合站台低光环境下的快速浏览

## 4. TRAE 实践过程

### 开发流程

整个 Demo 开发分四个阶段，全部由 TRAE 驱动完成：

**阶段一：市场调研与方案规划**

我先向 TRAE 描述了产品方向，TRAE 自动搜索了 2025-2026 年地铁出行 App 市场现状、竞品功能对比、智慧交通政策趋势，输出了结构化的调研报告，为后续方案设计提供了数据支撑。

**阶段二：创意方案 HTML 报告生成**

基于调研结果，TRAE 按照"创意名称与介绍 → 目标用户及痛点 → 价值与意义"的结构，生成了一份完整的 HTML 创意方案报告，包含配图（AI 生成）、数据可视化统计卡片、竞品对比表格，并引用了 7 个权威数据来源。

**阶段三：UniApp 小程序项目搭建**

我说"基于 HTML 内容生成一个可运行的多端小程序项目 Demo，基于 Vue3 + UniApp + TailwindCSS + Vant 4 UI"，TRAE 自动完成了：

- 项目脚手架搭建（package.json、vite.config.js、tailwind.config.js、pages.json、manifest.json）
- 4 个核心页面（首页、拥挤度、换乘、我的）的完整 Vue 组件编写
- 3 个可复用组件（ArrivalCard、MetroLine、StationCard）
- 全套模拟数据（mock.js）支持所有页面交互
- 深色科技风全局样式（global.scss）
- TabBar 图标生成（Python 脚本生成 PNG）
- 响应式布局和动画效果

**阶段四：参赛帖撰写**

最后，TRAE 根据 Demo 内容和比赛要求，生成了本参赛帖的全部文字内容。
