# AIIDE 服装数字手册配置叠加与融合规范 (v2.3.9-stable)

本规范确立了不同内容画像调性（Category Profile, Quality Profile, Style Profile）在叠加融合时的物理规则与冲突解决契约，为引擎自动化解耦及模块化数据流动提供底层规范支持。

---

## 📐 一、画像调性定义与分层

在全域生成体系中，调性配置信息被拆分为以下四个维度：

1. **结构规则 (Template Contract)**：即 `references/clean_md_template.md` 约定的各章节物理 Markdown 结构骨架。
2. **品类事实 (Category Profiles)**：针对特定服装品类（如 `tops`、`pants`、`outerwear`）的专用物理事实规程（例如：长裤必须展现裤长、裆深、臀围等尺码属性；外套必须包含拉链、袖口防风工艺）。
3. **质量深度 (Quality Profiles)**：定义内容的质量标杆与丰富度（如 `premium` 高奢精装级、`casual` 常规商业级、`lite` 快速分发级），主要控制段落字数下限、专业名词密度和分析图谱维度。
4. **风格表达 (Style Profiles)**：定义文案的修辞调性与情绪张力（如 `commute` 精英通勤风、`casual` 松弛都市休闲风、`street` 潮流街头风、`premium` 高质感留白风）。

---

## ⚡ 二、叠加与合并优先级契约 (Precedence Hierarchy)

当系统同时加载多个画像配置文件并合并生成商品数字手册时，必须遵守以下绝对的优先级递减和强行覆盖链：

```text
结构规则 (Template) ➔ [强行覆盖] ➔ 品类事实 (Category) ➔ [主导填充] ➔ 质量深度 (Quality) ➔ [控制精度] ➔ 风格表达 (Style)
```

### 物理叠加层优先级规则表

| 叠加层级 | 决策实体 | 约束边界与冲突合并行为 |
| :--- | :--- | :--- |
| **P0 强物理约束** | **结构规则 (Template)** | 拥有绝对的物理结构控制权。任何品类或风格所建议的 Markdown 表现形式，若与 `clean_md_template.md` 冲突，**一律强行抛弃并以模板为准**。 |
| **P1 领域事实层** | **品类事实 (Category)** | 决定各章节物理段落的信息必填域。例如，`pants` 品类事实决定了“Ch 8.1”中必须呈现面料拉伸与裆深参数，这一事实**覆盖质量与风格层**的默认文本段落。 |
| **P2 质量粒度层** | **质量深度 (Quality)** | 控制信息密度与字数级别限制。如果风格调性（如极简风格）建议“用两句话概括”，但 `premium` 质量深度规定“核心卖点提炼每条字数必须在 80-120 字之间且有微观技术名词”，**质量约束强行覆盖风格的极简意见**。 |
| **P3 艺术修辞层** | **风格表达 (Style)** | 拥有最终的艺术修辞、形容词替换和语言调色权。在满足 P0-P2 约束的前提下，进行语境润色（例如：将通俗词汇“好搭配”在精英通勤风格中翻译为“兼具微风般的兼容度与利落的线条平衡感”）。 |

## 🔄 三、典型冲突场景融合规程 (Conflict Resolution Algorithms)

### 典型冲突场景一：修辞高频词与信息密度限制的融合
*   **冲突现象**：风格表达 (Style: Street) 建议使用大量潮酷短排比与感叹号，导致内容长度极短，从而触发质量深度 (Quality: Premium) 规定“每个段落必须提供深度原理解析（不得少于60字）”的扣分规则。
*   **融合算法规程**：
    1.  系统提取 Street 风格的核心词库（如“机能”、“撞色”、“先锋”）。
    2.  以 Quality: Premium 定义的结构化句式（“物理结构 + 视觉情绪 + 场景回弹”）为框架骨骼。
    3.  将 Street 风格词汇作为形容词填装进骨骼内，确保在单句字数满足 60 字下限的同时，艺术风格率达到 80%。

### 典型冲突场景二：品类尺寸特有字段在多色系分类中的表彰
*   **冲突现象**：在 Chapter 5.1 颜色展示中，模板要求展示全部商品颜色与对应图片，而 Pants 品类事实要求在尺码表中按色系拆分多行展示，导致部分属性重复呈现。
*   **融合算法规程**：
    1.  Chapter 5.1 的颜色卡片只进行纯色图与 PID 确认，不引入尺码数据。
    2.  品类事实中的多色系尺码字段，必须且仅在 Chapter 8.1 最后的材质尺码表中以多维列格式展示，保持结构与展现绝对分离。

## 四、解耦数据模型规范

配置文件合并策略主要通过 JSON 数据块进行如下形式的融合演进：

```json
{
  "profile_merge_policy_version": "2.3.9-stable",
  "active_skus": ["SKU_EXAMPLE_A", "SKU_EXAMPLE_B"],
  "resolution_rules": {
    "template_override": true,
    "category_fact_dominance": true,
    "quality_word_count_enforcement": "max(style_limit, quality_min)",
    "style_rhetoric_layering": "replace_synonyms_only"
  }
}
```

本规范一经发布，后续所有针对 `category_profiles/` 的开发均在此框架契约下进行。
