# 纯净 Markdown 内容源契约 (Clean MD Contract)

> [!NOTE]
> **适用范围与受众**：本契约是最终手册 Markdown 源文件物理格式的权威技术标准。它主要约束引擎的静态语法校验器（Linter）与 Markdown 编译器，用以保证交付物绝无 HTML/CSS 视觉标签。

## 级别规范：`clean-md-compatible` (v2.3.3.2 交付标准)

为了保障手册内容在 AI 数字人、RAG 向量检索、局部增量更新以及语义审计时的结构纯净度与解析稳定性，特制定本法典。

---

## 🚫 1. 禁用元素 (FORBIDDEN)
在 `clean-md-compatible` 规范下，正文 Parts 源文件和最终 `{SKU}.md` 严禁包含任何视觉样式的 HTML/CSS 干扰：
1.  **禁止任何 HTML 结构性容器**：严禁出现 `<div>`、`<span>`、`<section>`、`<p>`、`<ul>`、`<li>`、`<table>`、`<tr>`、`<td>`、`<th>` 等原生 HTML 块级或行级标签。
2.  **禁止行内或外部 CSS 样式**：严禁在任何元素上附加 `style` 属性或直接引入 `<style>` 标签。
3.  **禁止任何 CSS 类名或标识符**：严禁使用 `class` 或 `id` 属性。
4.  **禁止任何客户端脚本**：严禁出现 `<script>` 标签或包含诸如 `onclick` 等事件处理器。
5.  **禁止任何装饰性或排版型 Emoji**：在正文排版中（如标题前缀、无序列表项目符号）默认禁止使用任何无语义的纯视觉装饰 Emoji。仅在特定话术字段值或特定标签字段值中允许使用必要的语义 Emoji。
6.  **禁止 HTML 图片控制**：严禁使用 `<img>` 标签或 `height`/`width` 等 HTML 属性控制图片大小。图片尺寸必须通过物理预处理（统一补白、缩放为标准分辨率）解决，而非在 Markdown 中写 HTML style。

---

## ✅ 2. 允许元素 (ALLOWED)
契约规定，纯净内容源必须严格基于标准 Markdown 原生语法构建：
1.  **标准标题**：一至四级标准标题（`#` 至 `####`），用于建立章节主干树结构。
2.  **标准段落**：普通无格式文本换行。
3.  **标准 Markdown 表格**：使用 `|` 和 `-` 进行结构对齐的原生表格（`|---|`）。
    - 所有主要信息表（基础信息、人群画像、场景矩阵、情绪矩阵、营销表、尺码表、素材索引）必须使用原生 Markdown 表格。
    - 禁止在 Clean MD 中使用 HTML `<table>` 替代 Markdown 表格。
4.  **标准无序/有序列表**：使用 `-` / `*` 或 `1.` / `2.` 的标准缩进列表。
5.  **标准加粗**：使用双星号 `**加粗文本**`。
6.  **标准图片引用**：原生 Markdown 图片语法：`![图片描述](图片路径)`。
    - *特例*：针对 **8.1 面料与品牌资质** 图片，必须由流水线物理重绘和补白缩放为完全一致的 **`3:4` (900x1200 像素) 标准分辨率物理竖图**。在 Markdown 原生表格中并排引用时，必须满足 `MAX_COLS = 2` 的最大双列折行对齐规则，通过"同比例同分辨率"物理机制在任何渲染器中均平分 `50%:50%` 列宽，确保合格证大图超高清晰度与大小绝对一致。
    - 禁止在 Markdown 中使用 `<img style=...>` 控制图片高度或宽度。
7.  **标准引用块**：使用 `>` 的引用。

---

## 💡 3. 元注释纯净化剥离契约 (Comment Purifying Exception)

在交付的最终大 `{SKU}.md` 纯净手册中，**绝对禁止残留任何** `<!-- pdh:... -->` 隐藏元注释或定位段落注释（打扫战场，100% 物理清空注释杂质，以防干扰 AI 数字人与检索读取）。

但在开发或中间态的零部件文件（即 `.scratch/parts/part_*.md` 零件文件）中，允许且仅允许保留以下隐藏元注释，以便为流水线提供格式追踪与 HTML 视图高奢组件的编译驱动：

### A. 顶部零件格式元注释 (File-level Parts Metadata)
每个 Parts 零部件文件的首行，允许包含以下标记以维持引擎的安全沙箱读取：
```markdown
<!-- pdh:format=clean-md-compatible; version=v2.3.3.2 -->
```
*   **用途**：标识零件已遵循 v2.3.3.2 纯净规范，供三轨校验器智能识别。在 assemble 双端合并组装大文件时，引擎将自动扫清此行。

### B. 章节组件驱动注释 (Component Markers)
在需要增强为高奢视觉卡片的章节（如人群画像、场景矩阵、尺码建议），允许且仅允许在**对应章节标题的前一行**加入白名单注释标记：
```markdown
<!-- pdh:component=audience_card_grid -->
### 2.1 核心人群画像
```
*   **白名单组件值**：
    *   `audience_card_grid`（人群画像 Flex 渐变卡片）
    *   `scene_card_grid`（3x2 场景卡片网格）
    *   `marketing_card_grid`（核心营销与策略卡片）
    *   `livestream_quote_card`（直播与痛点话术卡片）
    *   `qualification_card_grid` (面料资质与洗唛卡片)
    *   `size_table_gradient`（尺码数据表高奢渐变）
    *   `size_guide_cards`（尺码穿搭与辅助建议卡片）
*   **使用约束**：
    1.  注释内容必须独占一行，禁止与正文或标题混写。
    2.  注释内容内部严禁嵌套任何 HTML 标签、CSS 属性、JS 代码。
    3.  注释内容在普通 Markdown 解析下应当隐蔽，绝对不得作为正文内容渲染展示给人类读者。
