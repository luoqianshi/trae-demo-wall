# HTML 渲染与视觉交付契约 (HTML Render Contract)
## 视觉交付标准：`v2.3.3.2-luxury`

为了规范精美 HTML 视图的视觉品质、内网离线安全性及数据事实的严谨度，特制定本法典。

---

## ✅ 1. 允许元素与布局规范 (ALLOWED & ENHANCED)
HTML 渲染器在编译纯净 Markdown 源文件时，可以自由运用高级 CSS 语法，将文本结构在内存中重塑为顶级高奢排版：
1.  **高级单页内联样式**：允许在 `<head>` 中嵌入高品质的全局与局部 `<style>` 样式表。
2.  **弹性布局与栅格网格**：提倡使用 CSS Flexbox 与 CSS Grid 构建自适应的多列布局、画像卡片墙和瀑布流网格。
3.  **色彩与背景美学**：使用系统色表，应用柔和的水平或对偶渐变（如紫蓝深沉渐变 `linear-gradient`）、明亮的毛玻璃磨砂（Glassmorphism）、平滑的高斯模糊及立体微阴影（Box Shadow）。
4.  **TOC 侧边树导航**：必须在页面左侧渲染出高品质、自适应的 Table of Contents 章节导航，使用纯 HTML 锚点链接（`<a href="#id">`）实现点击跳转与位置索引。
5.  **离线字体与可选外网字体**：
    *   **默认必须**使用本地系统字体栈（System Font Stack），确保在完全离线、内网安全隔离的客户端上也能完美呈现极高水准的西文及中文排版。
    *   仅在命令行参数指示 `--allow-remote-fonts` 时，才允许在头部插入远程 Google Fonts 的加载链接。
6.  **打印适配样式**：配置 `@media print` 打印专用 CSS，使人类用户能将 HTML 格式一键完美导出为无噪点、分页合理的 PDF 汇报文件。

---

## 🚫 2. 禁用元素与安全沙箱 (FORBIDDEN & ISOLATED)
为了保证商品交付件在企业内部及客户端环境的绝对安全性，杜绝任何合规性风险，特制定硬性安全红线：
1.  **绝对禁止任何客户端脚本（JS HARD BAN）**：
    *   禁止包含任何 `<script>` 标签（无论是本地脚本还是远程 CDN 外链）。
    *   禁止在任何 HTML 标签上附加内联 JavaScript 事件处理器（如 `onclick`、`onload`、`onmouseover` 等）。
    *   所有的导航、页签切换必须完全基于 HTML/CSS 原生交互（如 CSS `:checked` 或 `href` 锚点）。
2.  **严禁包含不受控的远程链接**：页面内的图片、样式或其他任何资产均应当为相对路径的本地打包资源，禁止拉取非受控域名的未知网络资源。

---

## ⚖️ 3. 事实一致性与“零捏造”原则 (FACT PARITY RULES)
视觉渲染版只是内容源的一面“艺术放大镜镜子”，绝对不准成为新的信息源头：
1.  **禁止无中生有（NO HALLUCINATION）**：HTML 渲染器只能从 `{SKU}.md` 源中解析数据进行卡片填充，**绝对禁止**在模板中硬编码添加该商品不存在的规格参数、检测结论、人群话术或核心 PID 事实。
2.  **禁止遗漏删减（NO OMISSION）**：对于面料纤维配比（如 `粘胶 70%`）、尺码表关键行列数值、认证标准与合格结论，HTML 渲染必须做到 100% 完整展示，绝对禁止为了追求视觉对称美观而擅自删去部分关键行、关键列或警告性声明。
3.  **降级回退原则（GRACEFUL FALLBACK）**：如果纯净 Markdown 内部的表格列名或数据格式无法匹配 `components_schema.json` 规定的输入列名，**绝对禁止导致渲染中断或抛错崩溃**。渲染引擎应当写入警告，并将该区块自动以标准 Markdown 原生样式优雅退化渲染为普通的 HTML 表格，保证页面整体的顺利交付。
