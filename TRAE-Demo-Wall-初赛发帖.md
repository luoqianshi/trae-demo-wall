## 1. Demo 简介

**是什么：** 一个网站（Web App），通过 Python 爬虫自动采集 TRAE 论坛中的参赛作品，将所有网页/前端类作品以卡片式画廊呈现，用户无需下载即可在页面内直接 iframe 预览和体验每一个 Demo。

**面向谁：**
- **参赛者** — 浏览其他人的作品、学习前端实现思路、获取创作灵感
- **TRAE 爱好者** — 了解 TRAE 能做出什么水平的作品
- **评委/组织者** — 高效浏览和评审大量参赛作品

**核心功能：**

| 功能 | 说明 |
|---|---|
| **自动数据采集** | Python 爬虫遍历 Discourse 论坛 JSON API，自动识别 web/前端作品，生成结构化索引 |
| **搜索筛选排序** | 支持关键词搜索、标签分类筛选，按浏览量/点赞数/时间排序 |
| **即时预览体验** | 详情页以 16:9 iframe 沙盒直接运行作品 Demo，支持外部链接和本地部署双模式 |
| **懒加载** | 首屏仅加载轻量索引，滚动时通过 IntersectionObserver 按需分页，每次增量 12 条 |
| **粒子动效背景** | Canvas 2D 粒子系统，80 粒子 + 鼠标吸引 + 距离连线 |
| **打字机品牌标题** | 导航栏品牌文字循环打字机效果，纯 JS setTimeout 实现 |
| **液态玻璃导航** | 滚动感知 `backdrop-filter: blur()` 强度变化 |

## 2. Demo 创作思路

**灵感来源：**

我在参赛过程中发现，TRAE 论坛中已经积累了大量优秀的前端/网页作品，但这些作品散落在不同页面的帖子中，逐一翻阅效率极低。更关键的是，很多作品只提供了文字描述和截图，用户必须手动下载附件、解压、本地打开才能看到实际效果。整个链路太长，大部分人在看完标题后就划走了。

**想解决的问题：**

- **发现难** — 论坛帖子按时间排列，没有分类和搜索，找到感兴趣的作品全靠运气
- **体验难** — 作品需要手动下载附件才能预览，很多用户看完标题就放弃了
- **学习难** — 想参考某个作品的实现方式，但帖子中往往没有提供在线 Demo 链接

**为什么做这个方向：**

这些作品本质都是网页，为什么不把它们直接变成「可体验」的？TRAE Demo Wall 将数百个作品的浏览成本从「逐帖翻阅 + 下载解压」降低到「打开网站 → 筛选 → 点击即体验」。对于参赛者而言，这个平台本质上是一个 **TRAE 前端作品的「活教材」**——每一个作品都可以直接在 iframe 中运行体验，用户不仅能看到最终效果，还能通过查看原帖深入理解创作者的实现思路。这种「看到效果 → 产生兴趣 → 深入学习」的闭环，是静态截图无法提供的。

同时，这个平台本身也是用 TRAE 构建的，从爬虫到前端全链路均由 TRAE 辅助完成，本身就是 TRAE 生产力的最佳证明。

## 4. TRAE 实践过程

整个项目从零到部署，全链路由 TRAE 辅助完成。以下是关键开发步骤：

### Step 1：数据采集管道搭建

使用 TRAE 编写 Python 爬虫，通过 Discourse 论坛 JSON API 自动遍历所有帖子，识别 web/前端类作品，提取标题、作者、标签、浏览量、描述、附件链接等结构化数据，生成双层 JSON（轻量索引 + 分页详情）。
